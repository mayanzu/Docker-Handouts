Part 1

虚拟化概述

从物理机到虚拟机的演进

Part 1.1

什么是虚拟化？

核心定义
虚拟化（Virtualization）是将物理计算资源抽象成逻辑资源的技术，使得多个操作系统和应用可以共享同一套物理硬件资源，实现资源的高效利用和灵活管理。

虚拟化技术的核心思想是解耦——将软件与硬件分离，打破"一台物理服务器只能运行一个操作系统"的传统限制。通过虚拟化，单台物理服务器可以同时运行多个相互隔离的虚拟机。

虚拟化的核心价值

1. 资源利用率提升：打破物理边界，提高硬件利用率

2. 成本降低：减少物理服务器数量，降低运维成本

3. 灵活扩展：快速创建、复制、迁移虚拟机

4. 隔离安全：虚拟机之间相互隔离，提高安全性

Part 1.2

虚拟化发展历史

虚拟化技术的发展经历了多个重要阶段：

时期 里程碑 意义
1960s IBM推出CP/CMS系统 虚拟化技术的起源
1999 VMware Workstation发布 x86平台虚拟化开端
2003 Xen开源项目启动 开源虚拟化的兴起
2005-2006 Intel VT/AMD-V技术 硬件辅助虚拟化成熟
2013 Docker发布 容器技术革命
提示

从2013年Docker发布开始，容器技术逐渐成为虚拟化领域的新热点，与传统虚拟机形成互补关系。

Part 2

虚拟化类型

不同层次的虚拟化技术

Part 2.1

按实现层次分类

1硬件虚拟化

在硬件层面实现虚拟化，如CPU虚拟化（Intel VT-x/AMD-V）、内存虚拟化、I/O虚拟化

2系统虚拟化

通过Hypervisor实现完整操作系统虚拟化，如VMware、KVM、Xen

3桌面虚拟化

将桌面环境虚拟化，用户通过网络访问虚拟桌面，如Citrix、VMware Horizon

4应用虚拟化

将应用程序与底层系统隔离，如Java虚拟机、容器技术

本课程重点
本课程主要关注系统虚拟化（虚拟机）和应用虚拟化（容器）两大类型，理解它们的区别和适用场景。
Part 2.2

虚拟化技术对比

特性 全虚拟化 半虚拟化 硬件辅助虚拟化
实现方式 软件模拟所有硬件 修改客户机内核 CPU硬件支持
性能开销 较高（15-20%） 中等（5-10%） 低（1-5%）
兼容性 无需修改系统 需要修改内核 无需修改系统
代表产品 VMware ESX早期版本 Xen KVM、VMware vSphere
技术说明

- 全虚拟化：Hypervisor完全通过软件模拟硬件，客户机无感知

- 半虚拟化：客户机操作系统需要修改以配合Hypervisor

- 硬件辅助虚拟化：CPU提供虚拟化扩展指令，性能最优

Part 3

虚拟机技术

Hypervisor与虚拟机架构

Part 3.1

Hypervisor架构

Hypervisor定义
Hypervisor（虚拟机监控器）是运行在物理硬件和虚拟机之间的软件层，负责管理物理资源并为虚拟机提供虚拟硬件接口。

1Type 1 裸机型

直接运行在物理硬件上，如VMware ESXi、Microsoft Hyper-V、Xen，性能最优

2Type 2 宿主型

运行在宿主操作系统上，如VMware Workstation、VirtualBox，易于安装使用

架构对比

Type 1直接控制硬件，性能更好，适合生产环境；Type 2依赖宿主系统，安装简便，适合开发和测试。

Part 3.2

虚拟机核心组件

一个完整的虚拟机包含以下核心组件，Hypervisor负责将这些虚拟组件映射到物理资源：

虚拟组件 功能说明 虚拟化方式
vCPU 虚拟中央处理器 时间片轮转调度
vRAM 虚拟内存 影子页表/EPT
vDisk 虚拟磁盘 镜像文件（VMDK/VHD）
vNIC 虚拟网卡 软件模拟/SR-IOV
关键概念

- VMDK：VMware虚拟磁盘格式

- VHD/VHDX：微软虚拟磁盘格式

- QCOW2：QEMU/KVM虚拟磁盘格式

Part 3.3

虚拟机 vs 容器

理解虚拟机与容器的区别是本课程的重要基础。两者都是虚拟化技术，但实现层次和适用场景不同：

对比项 虚拟机（VM） 容器（Container）
虚拟化层次 硬件级虚拟化 操作系统级虚拟化
隔离级别 强隔离（独立内核） 进程级隔离（共享内核）
启动速度 分钟级 秒级
资源占用 较高（GB级） 低（MB级）
性能损耗 5-15% 接近原生
适用场景 完整系统隔离 应用快速部署
Part 4

虚拟化工具实践

VMware Workstation虚拟机操作

Part 4.1

VMware Workstation简介

VMware Workstation
VMware Workstation是VMware公司推出的Type 2 Hypervisor，是业界领先的桌面虚拟化软件。它支持在Windows和Linux宿主系统上同时运行多个操作系统，功能强大且性能优异。
VMware Workstation特点

1. 性能优异：优化的虚拟化引擎，接近原生性能

2. 功能强大：支持3D加速、快照、克隆、虚拟网络编辑等

3. 兼容性好：支持数百种操作系统，包括最新的Windows和Linux发行版

4. 企业级特性：支持vSphere连接、团队开发等高级功能

版本说明

VMware Workstation Pro是商业版本，功能完整；VMware Workstation Player是免费个人版，功能简化但足够学习使用。

Part 4.2

创建虚拟机

创建虚拟机步骤 操作指南 
1. 打开VMware Workstation，点击"文件"-"新建虚拟机"
2. 选择配置类型
   - 典型（推荐）：快速创建，使用默认设置
   - 自定义：手动配置所有参数
3. 选择安装来源
   - 安装程序光盘映像文件（ISO）
   - 稍后安装操作系统
4. 选择客户机操作系统
   - Linux - Ubuntu 64位
5. 命名虚拟机并选择存储位置
6. 指定磁盘容量（建议20GB以上）
   - 将虚拟磁盘存储为单个文件
   - 将虚拟磁盘拆分为多个文件
配置建议

- 内存：建议分配2-4GB，不超过物理内存的50%

- 处理器：根据物理核心数分配（建议2核）

- 磁盘：单个文件性能更好，多个文件便于迁移

Part 4.3

常用功能操作

VMware Workstation常用操作 操作指南 
# 虚拟机电源操作
启动虚拟机：点击"开启此虚拟机"
挂起虚拟机：点击"挂起客户机"（保存状态）
关闭虚拟机：点击"关机"或"关闭客户机"
重启虚拟机：点击"重新启动客户机"

# 快照管理
创建快照：虚拟机 - 快照 - 拍摄快照
恢复快照：虚拟机 - 快照 - 恢复到快照
管理快照：虚拟机 - 快照 - 快照管理器

# 克隆虚拟机
虚拟机 - 管理 - 克隆
- 链接克隆：依赖原虚拟机，节省空间
- 完整克隆：独立副本，占用双倍空间

# 虚拟机设置调整
虚拟机 - 设置
- 可调整内存、处理器、硬盘、网络适配器等
快照功能说明

- 快照：保存虚拟机某一时刻的完整状态

- 用途：系统测试前备份、危险操作前保护

- 注意：快照过多会影响性能，建议定期清理

Part 4.4

虚拟机网络配置

VMware Workstation提供多种网络模式，通过虚拟网络编辑器可以灵活配置：

网络模式 特点 适用场景
桥接模式 虚拟机与宿主机同级，独立IP 服务器部署，需要外部访问
NAT模式 虚拟机可访问外网，通过NAT转换 默认模式，上网浏览
仅主机模式 仅宿主机与虚拟机通信，无外网 封闭测试环境
自定义/LAN区段 虚拟机之间组成独立网络 多机集群测试
配置建议

学习Docker时建议使用桥接模式或NAT模式+端口映射，方便从宿主机访问容器服务。可通过"编辑"-"虚拟网络编辑器"进行高级配置。

Part 5

WSL

Windows Subsystem for Linux

Part 5.1

什么是WSL？

WSL定义
WSL（Windows Subsystem for Linux）是Windows 10/11内置的Linux兼容层，允许开发者在Windows上直接运行Linux命令行工具、应用程序和容器，无需传统虚拟机的开销。

WSL是微软与Canonical合作开发的子系统技术，它通过系统调用转换将Linux系统调用实时转换为Windows NT内核调用，实现了Linux环境在Windows上的原生运行。

WSL版本对比

1. WSL 1：系统调用转换层，无真实Linux内核，文件IO性能受限

2. WSL 2：基于轻量级VM，运行真实Linux内核，完整系统调用兼容性，性能大幅提升

推荐使用WSL 2

WSL 2是默认版本，提供更好的性能、完整的系统调用兼容性，且支持Docker Desktop集成。本课程所有操作均基于WSL 2。

Part 5.2

WSL的优势

1轻量高效

相比传统虚拟机，WSL 2启动仅需数秒，内存占用极低，资源开销小

2无缝集成

与Windows文件系统无缝互通，可在Windows和Linux间自由切换

3Docker原生支持

Docker Desktop可直接使用WSL 2后端，无需额外配置

4开发友好

支持VS Code远程开发，Windows Terminal集成，开发体验极佳

对比项 传统虚拟机 WSL 2
启动时间 30秒-2分钟 1-3秒
内存占用 2GB+ 动态分配
文件访问 共享文件夹/网络 原生访问
GUI应用 完整支持 WSLg支持
Part 5.3

安装WSL

WSL安装命令 PowerShell 
# 以管理员身份打开PowerShell，执行以下命令

# 自动安装WSL（推荐方式）
wsl --install

# 安装完成后重启计算机
# 重启后会自动继续安装Ubuntu

# 查看已安装的发行版
wsl --list --verbose

# 设置默认WSL版本为WSL 2
wsl --set-default-version 2

# 安装特定Linux发行版
wsl --install -d Ubuntu-22.04

# 查看可用的发行版列表
wsl --list --online
系统要求

- Windows 10：版本2004及更高（内部版本19041+）

- Windows 11：所有版本均支持

- 需要在BIOS中启用虚拟化支持（Intel VT-x/AMD-V）

Part 5.4

WSL常用操作

WSL管理命令 PowerShell/CMD 
# 启动默认的Linux发行版
wsl

# 启动指定的发行版
wsl -d Ubuntu-22.04

# 关闭所有WSL发行版
wsl --shutdown

# 终止指定的发行版
wsl -t Ubuntu-22.04

# 导出发行版为tar文件
wsl --export Ubuntu-22.04 D:\backup\ubuntu.tar

# 从tar文件导入发行版
wsl --import CustomUbuntu D:\WSL D:\backup\ubuntu.tar

# 设置默认发行版
wsl --set-default Ubuntu-22.04

# 卸载发行版
wsl --unregister Ubuntu-22.04
文件系统访问

- 在WSL中访问Windows文件：/mnt/c/（C盘）、/mnt/d/（D盘）

- 在Windows中访问WSL文件：在资源管理器输入 \\wsl$\

- 建议将项目文件放在WSL文件系统中以获得最佳性能

Part 5.5

WSL与Docker

WSL 2与Docker Desktop深度集成，是Windows上运行Docker的最佳方案：

集成优势

1. 原生Linux内核：WSL 2运行真实Linux内核，Docker容器在Linux环境中原生运行

2. 性能优化：文件系统性能优于传统Hyper-V方案

3. 资源高效：Docker和WSL共享同一个轻量级VM

4. 开发体验：VS Code可直接连接到WSL中的Docker

Docker Desktop WSL集成配置 操作步骤 
1. 安装WSL 2（已完成）
2. 下载安装Docker Desktop for Windows
3. 打开Docker Desktop设置
4. 选择"Use the WSL 2 based engine"
5. 在Resources - WSL Integration中
   - 启用"Enable integration with my default WSL distro"
   - 为需要的发行版开启集成
6. 点击Apply & Restart

# 验证Docker在WSL中可用
wsl -d Ubuntu-22.04
docker --version
docker run hello-world
最佳实践

建议在WSL中直接安装Docker Engine（不依赖Docker Desktop），可获得更纯粹的Linux Docker体验，适合服务器环境模拟。

动手实验：WSL配置与使用

实验 WSL安装与配置

实验目的

在Windows上安装配置WSL 2，安装Ubuntu系统，并验证Docker集成

实验步骤

1 以管理员身份打开PowerShell，执行 wsl --install 命令

常见问题

- WSL安装失败：确保Windows版本符合要求，已启用虚拟化

- 文件权限问题：Windows文件在WSL中显示777权限是正常现象
