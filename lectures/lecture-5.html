Part 1

容器基础概念

理解容器的本质和特点

Part 1.1

什么是Docker容器？

容器定义
Docker容器是镜像的运行实例，是一个轻量级、独立、可执行的软件包，包含运行应用所需的所有内容：代码、运行时、系统工具、系统库和配置。

容器与镜像的关系类似于对象与类的关系。镜像是静态的模板，容器是镜像的动态实例。容器在镜像的基础上添加了一个可写层，所有运行时的修改都发生在这个可写层中。

1轻量级

共享宿主机内核，启动时间秒级，资源占用少

2隔离性

通过Namespace和Cgroups实现进程、网络、文件系统等隔离

3可移植性

在任何Docker环境中行为一致，实现"一次构建，到处运行"

4可控制

可限制CPU、内存、IO等资源使用，确保系统稳定

容器 vs 虚拟机

容器：轻量级，启动快，资源占用少，共享内核

虚拟机：重量级，启动慢，资源占用多，完整操作系统

- Deleted：容器已删除

容器操作命令

掌握容器的创建、启动、停止等操作

Part 2.1

创建与启动容器

docker create - 创建容器（不启动） Bash 
# 语法：docker create [OPTIONS] IMAGE [COMMAND] [ARG...]
# 作用：创建一个新容器但不启动它，返回容器ID
#       适合需要先配置再启动的场景

# 基础用法 - 创建Ubuntu容器
$ docker create --name my-ubuntu ubuntu:22.04

# 预期输出：
# 容器ID（如：1a2b3c4d5e6f...）

# 参数详解：
# --name "容器名"      为容器指定名称，便于后续管理
#                      不指定则自动生成随机名称
# --env, -e "KEY=VAL"  设置环境变量，可多次使用
#                      示例：-e TZ=Asia/Shanghai
# --volume, -v "宿主路径:容器路径"  挂载数据卷
#                      示例：-v /data/app:/app
# --port, -p "宿主端口:容器端口"   端口映射
#                      示例：-p 8080:80
# --network "网络名"   将容器连接到指定网络
#                      示例：--network my-network
# --restart "策略"     容器退出后的重启策略
#                      no：不重启（默认）
#                      always：总是重启
#                      on-failure：失败时重启
#                      unless-stopped：除非手动停止否则重启

# 示例：创建带完整配置的Ubuntu容器
$ docker create \
  --name ubuntu-app \
  -e TZ=Asia/Shanghai \
  -e LANG=C.UTF-8 \
  -v /data/app:/app \
  --restart unless-stopped \
  ubuntu:22.04

# 预期输出：
# 容器ID

# 创建后需要手动启动
$ docker start ubuntu-app
docker run - 创建并启动容器（最常用） Bash 
# 语法：docker run [OPTIONS] IMAGE [COMMAND] [ARG...]
# 作用：创建容器并立即启动，相当于 create + start

# 基础用法 - 启动Ubuntu容器（交互式）
$ docker run -it --name my-ubuntu ubuntu:22.04 bash

# 预期输出：
# 进入Ubuntu容器的bash终端
root@1a2b3c4d5e6f:/#

# 参数详解：
# --detach, -d         后台运行模式，容器在后台运行并返回容器ID
#                      不加此参数则前台运行，占用当前终端
# --interactive, -i    保持STDIN打开，即使没有附加
#                      通常与-t配合使用（-it），用于交互式操作
# --tty, -t            分配一个伪终端（pseudo-TTY）
#                      与-i配合使用，提供完整的终端体验
# --rm                 容器退出后自动删除
#                      适合临时测试，不会留下停止的容器
# --name "名称"        指定容器名称，便于后续管理
# --env, -e "KEY=VAL"  设置环境变量
# --volume, -v         挂载数据卷
# --port, -p           端口映射（宿主端口:容器端口）
# --network "网络名"   指定网络
# --restart "策略"     重启策略（no/always/on-failure/unless-stopped）
# --cpus "数量"        限制CPU核心数
# --memory, -m "大小"  限制内存使用

# 示例：交互式运行Ubuntu容器（临时测试）
$ docker run -it --rm ubuntu:22.04 bash

# 预期输出：
# 进入Ubuntu容器的bash终端
root@1a2b3c4d5e6f:/# cat /etc/os-release
PRETTY_NAME="Ubuntu 22.04.3 LTS"
...
root@1a2b3c4d5e6f:/# exit
# 退出后容器自动删除

# 示例：后台运行Ubuntu容器
$ docker run -d --name ubuntu-server \
  -e TZ=Asia/Shanghai \
  --restart unless-stopped \
  ubuntu:22.04 sleep infinity

# 预期输出：
# 容器ID

# 示例：带资源限制运行
$ docker run -d --name ubuntu-limited \
  --cpus 0.5 \
  --memory 512m \
  --restart unless-stopped \
  ubuntu:22.04 sleep infinity
运行模式

前台运行：容器在当前终端运行，按Ctrl+C停止

后台运行：使用-d参数，容器在后台运行

交互式运行：使用-it参数，进入容器终端

Part 2.2

查看容器

docker ps - 列出容器 Bash 
# 语法：docker ps [OPTIONS]
# 作用：列出容器信息，默认只显示运行中的容器

# 基础用法 - 列出运行中的容器
$ docker ps

# 预期输出：
CONTAINER ID   IMAGE          COMMAND           CREATED         STATUS         PORTS     NAMES
1a2b3c4d5e6f   ubuntu:22.04   "sleep infinity"  5 minutes ago   Up 5 minutes             my-ubuntu

# 参数详解：
# --all, -a            显示所有容器，包括已停止的
#                      默认只显示运行中的容器
# --quiet, -q          只显示容器ID，适合脚本处理
#                      常用于批量操作：docker rm $(docker ps -aq)
# --filter, -f "条件"  根据条件过滤容器
#                      示例：-f status=running（运行中）
#                            -f name=ubuntu（名称包含ubuntu）
#                            -f ancestor=ubuntu:22.04（基于ubuntu镜像）
# --format "模板"      使用Go模板格式化输出
#                      示例：--format "table {{.Names}}\t{{.Status}}"
# --last, -n "数量"    显示最近创建的n个容器（包括停止的）
# --no-trunc           不截断输出，显示完整信息
# -s, --size           显示容器大小

# 示例：显示所有容器
$ docker ps -a

# 预期输出：
CONTAINER ID   IMAGE          COMMAND           CREATED          STATUS                      PORTS     NAMES
1a2b3c4d5e6f   ubuntu:22.04   "sleep infinity"  5 minutes ago    Up 5 minutes                          my-ubuntu
2b3c4d5e6f7g   ubuntu:22.04   "bash"            10 minutes ago   Exited (0) 8 minutes ago              my-ubuntu-2

# 示例：只显示容器ID（常用于批量操作）
$ docker ps -q

# 预期输出：
1a2b3c4d5e6f

# 示例：显示最近创建的3个容器
$ docker ps -n 3

# 预期输出：
# 最近3个容器的信息

# 示例：过滤特定状态的容器
$ docker ps -f status=exited

# 示例：自定义格式输出
$ docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Image}}"

Part 2.3

停止与删除容器

docker stop - 优雅停止容器 Bash 
# 语法：docker stop [OPTIONS] CONTAINER [CONTAINER...]
# 作用：发送SIGTERM信号，等待容器优雅停止
#       容器内的进程有时间清理资源、保存数据

# 基础用法 - 停止Ubuntu容器
$ docker stop my-ubuntu

# 预期输出：
my-ubuntu

# 参数详解：
# --time, -t "秒数"    停止前的等待时间，默认10秒
#                      Docker先发送SIGTERM，等待指定时间
#                      如果超时仍未停止，则发送SIGKILL强制停止
#                      建议给重要应用更长的停止时间

# 示例：停止多个容器
$ docker stop ubuntu-app1 ubuntu-app2

# 预期输出：
ubuntu-app1
ubuntu-app2

# 示例：给容器更长的停止时间（30秒）
$ docker stop -t 30 ubuntu-database

# 预期输出：
ubuntu-database

# 停止流程说明：
# 1. Docker发送SIGTERM信号给容器主进程
# 2. 主进程收到信号，开始清理工作（如保存数据、关闭连接）
# 3. 如果在-t时间内停止，容器正常退出
# 4. 如果超时，Docker发送SIGKILL强制终止
docker start - 启动已停止的容器 Bash 
# 语法：docker start [OPTIONS] CONTAINER [CONTAINER...]
# 作用：启动已停止的容器，恢复之前的运行状态

# 基础用法 - 启动容器
$ docker start my-ubuntu

# 预期输出：
my-ubuntu

# 参数详解：
# --attach, -a         附加到容器的标准输入/输出/错误
#                      启动后立即看到容器输出
# --interactive, -i    附加到容器的STDIN
#                      与-a配合使用，可以与容器交互
# --checkpoint "名称"  从检查点恢复（实验性功能）
# --checkpoint-dir     检查点目录（实验性功能）

# 示例：启动并附加到容器（查看输出）
$ docker start -a my-ubuntu

# 预期输出：
# 显示容器启动日志

# 示例：启动并进入交互模式
$ docker start -a -i my-ubuntu

# 预期输出：
# 进入容器终端
docker rm - 删除容器 Bash 
# 语法：docker rm [OPTIONS] CONTAINER [CONTAINER...]
# 作用：删除已停止的容器，释放资源

# 基础用法 - 删除容器
$ docker rm my-ubuntu

# 预期输出：
my-ubuntu

# 参数详解：
# --force, -f          强制删除，即使容器正在运行
#                      会先停止容器再删除，慎用
# --volumes, -v        删除容器关联的匿名卷
#                      注意：不会删除命名卷
# --link, -l           删除容器间的链接（已弃用）

# 示例：强制删除正在运行的容器
$ docker rm -f my-ubuntu

# 预期输出：
my-ubuntu

# 示例：删除多个容器
$ docker rm ubuntu-app1 ubuntu-app2

# 预期输出：
ubuntu-app1
ubuntu-app2

# 示例：删除所有停止的容器
$ docker rm $(docker ps -a -q)

# 预期输出：
# 所有停止容器的ID
注意事项

- 删除容器前需先停止容器（除非使用-f参数）

- 使用-f参数强制删除可能导致数据丢失

- 删除容器不会自动删除关联的数据卷，需使用-v参数

Part 2.4

容器交互与执行命令

docker exec - 在容器中执行命令（推荐） Bash 
# 语法：docker exec [OPTIONS] CONTAINER COMMAND [ARG...]
# 作用：在运行中的容器内启动新进程执行命令
#       不影响容器主进程，是进入容器的推荐方式

# 基础用法 - 在Ubuntu容器中执行命令
$ docker exec my-ubuntu ls -la /home

# 预期输出：
# 目录内容列表

# 参数详解：
# --interactive, -i    保持STDIN打开，即使没有附加
#                      用于需要输入的命令
# --tty, -t            分配伪终端（pseudo-TTY）
#                      与-i配合使用（-it），提供完整终端体验
# --user, -u "用户"    指定执行命令的用户（用户名或UID）
#                      示例：-u root 或 -u 0
# --workdir, -w "路径" 指定命令的工作目录
#                      示例：-w /app
# --env, -e "KEY=VAL"  设置环境变量
# --privileged         给命令扩展权限
# --detach, -d         后台运行命令

# 示例：进入Ubuntu容器终端（最常用）
$ docker exec -it my-ubuntu bash

# 预期输出：
root@1a2b3c4d5e6f:/#

# 示例：以root用户身份执行命令
$ docker exec -u root my-ubuntu apt-get update

# 预期输出：
# 更新包列表

# 示例：在指定工作目录执行命令
$ docker exec -w /tmp my-ubuntu touch test.txt

# 预期输出：
# 在/tmp目录创建test.txt文件

# 示例：带环境变量执行命令
$ docker exec -e DEBUG=true my-ubuntu env

# 示例：后台执行命令
$ docker exec -d my-ubuntu /scripts/backup.sh
docker attach 与 docker exec 的区别

docker attach：附加到容器的主进程（PID 1），直接连接到容器的标准输入/输出/错误流

docker exec：在容器内启动一个新的进程，不影响主进程运行

attach 适合查看容器主进程的实时输出，exec 适合在容器内执行命令和调试

docker attach - 附加到容器主进程 Bash 
# 语法：docker attach [OPTIONS] CONTAINER
# 作用：连接到容器主进程的标准输入/输出/错误流，实时查看容器日志

# 附加到运行中的容器（查看实时输出）
$ docker attach my-ubuntu

# 预期输出：
# 直接看到容器主进程的输出内容

# 参数详解：
# --no-stdin=true      不附加标准输入，只查看输出，无法输入命令
#                      适用场景：只想查看日志，不想意外输入影响容器
# --sig-proxy=true     代理信号到容器（默认开启）
#                      开启时：Ctrl+C 会发送 SIGINT 到容器主进程，可能导致容器停止
#                      关闭时：Ctrl+C 只断开连接，容器继续运行

# 示例：只查看输出，不接受输入
$ docker attach --no-stdin my-ubuntu

# 示例：关闭信号代理，Ctrl+C不会停止容器
$ docker attach --sig-proxy=false my-ubuntu

# 重要提示：
# 1. 使用 Ctrl+C 可能会停止容器（当 --sig-proxy=true 时）
# 2. 安全退出（不停止容器）：按 Ctrl+P 然后按 Ctrl+Q
# 3. attach 连接的是容器启动时的主进程，不是新开shell
注意事项

- attach 连接主进程：退出可能影响容器运行状态

- 推荐使用 exec：更安全，不会影响容器主进程

- 安全退出方式：Ctrl+P Ctrl+Q 分离而不停止容器

Part 3

容器资源管理

控制容器的资源使用

Part 3.1

CPU资源限制 --cpus

--cpus 限制CPU核心数 Bash 
# 语法：docker run --cpus <value> IMAGE
# 作用：限制容器可以使用的CPU核心数

# 示例：限制Ubuntu容器使用1个CPU核心
$ docker run -d --name ubuntu-cpu1 --cpus 1 ubuntu:22.04 sleep infinity

# 验证CPU限制
$ docker inspect --format '{{.HostConfig.NanoCpus}}' ubuntu-cpu1
# 输出：1000000000（1个CPU核心，单位纳秒）
说明

--cpus：限制容器可以使用的CPU核心数，可以是小数（如0.5表示半个核心）

Part 3.1

CPU资源限制 --cpu-shares

--cpu-shares 设置CPU权重 Bash 
# 语法：docker run --cpu-shares <value> IMAGE
# 作用：设置CPU权重，当CPU资源竞争时生效（相对值）

# 示例：设置CPU权重（相对值）
$ docker run -d --name ubuntu-high --cpu-shares 1024 ubuntu:22.04 sleep infinity
$ docker run -d --name ubuntu-low --cpu-shares 512 ubuntu:22.04 sleep infinity
# 当CPU资源竞争时，ubuntu-high获得2倍于ubuntu-low的CPU时间
说明

--cpu-shares：设置CPU权重，当CPU资源竞争时生效。默认值为1024，值越大权重越高

Part 3.1

CPU资源限制 --cpuset-cpus

--cpuset-cpus 指定CPU核心 Bash 
# 语法：docker run --cpuset-cpus <value> IMAGE
# 作用：指定容器可以使用的CPU核心（逗号分隔或范围）

# 示例：指定容器使用特定CPU核心（0和1）
$ docker run -d --name ubuntu-pinned --cpuset-cpus 0,1 ubuntu:22.04 sleep infinity

# 示例：使用范围表示法（等同于0,1）
$ docker run -d --name ubuntu-range --cpuset-cpus 0-1 ubuntu:22.04 sleep infinity
说明

--cpuset-cpus：指定容器可以使用的CPU核心。可以是逗号分隔的列表（如"0,1,2"）或范围（如"0-3"）

Part 3.2

内存资源限制 --memory

--memory 硬内存限制 Bash 
# 语法：docker run --memory <value> IMAGE
# 作用：限制容器使用的内存，不能超过此值

# 示例：限制Ubuntu容器使用512MB内存
$ docker run -d --name ubuntu-mem --memory 512m ubuntu:22.04 sleep infinity

# 内存单位：b, k, m, g（字节、KB、MB、GB）

# 验证内存限制
$ docker inspect --format '{{.HostConfig.Memory}}' ubuntu-mem
# 输出：536870912（512MB的字节数）

# 在容器内查看内存限制
$ docker exec ubuntu-mem cat /sys/fs/cgroup/memory/memory.limit_in_bytes
536870912
说明

--memory：硬限制，容器使用的内存不能超过此值。如果容器尝试使用更多内存，会被OOM killer终止

Part 3.2

内存资源限制 --memory-swap

--memory-swap 交换空间限制 Bash 
# 语法：docker run --memory <mem> --memory-swap <swap> IMAGE
# 作用：memory-swap = memory + swap，限制总内存使用

# 示例：限制内存512MB，交换空间1GB
$ docker run -d --name ubuntu-swap \
  --memory 512m --memory-swap 1g ubuntu:22.04 sleep infinity
# swap = 1g - 512m = 512m

# 注意：--memory-swap必须大于--memory
说明

--memory-swap：内存+交换空间的总限制。当物理内存不足时，使用交换空间

Part 3.2

内存资源限制 --memory-reservation

--memory-reservation 软限制 Bash 
# 语法：docker run --memory <mem> --memory-reservation <res> IMAGE
# 作用：设置软限制，内存紧张时触发，不强制执行

# 示例：硬限制512MB，软限制256MB
$ docker run -d --name ubuntu-soft \
  --memory 512m --memory-reservation 256m ubuntu:22.04 sleep infinity
说明

--memory-reservation：软限制，当系统内存不足时会提前警告，不会强制终止容器

Part 3.2

内存资源限制 --oom-kill-disable

--oom-kill-disable 禁用OOM killer Bash 
# 语法：docker run --oom-kill-disable --memory <mem> IMAGE
# 作用：禁用Out-Of-Memory killer，容器内存不足时不会自动终止

# 示例：禁用OOM killer
$ docker run -d --name ubuntu-no-oom \
  --oom-kill-disable --memory 512m ubuntu:22.04 sleep infinity
警告

谨慎使用：禁用OOM killer后，如果容器耗尽内存，可能会影响宿主机上其他进程

Part 3.3

IO资源限制 --blkio-weight

--blkio-weight IO权重 Bash 
# 语法：docker run --blkio-weight <value> IMAGE
# 作用：设置块设备IO权重（10-1000），影响IO调度优先级

# 示例：设置IO权重为500（默认100）
$ docker run -d --name ubuntu-io --blkio-weight 500 ubuntu:22.04 sleep infinity
说明

--blkio-weight：设置相对权重，值越大，容器获得IO时间越多。权重仅在多个容器竞争IO时生效
