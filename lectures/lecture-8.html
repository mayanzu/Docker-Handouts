Part 1

镜像仓库概述

理解镜像仓库的概念和类型

1.1

什么是镜像仓库？

镜像仓库概念
Docker镜像仓库（Registry）是存储和分发镜像的服务系统，类似图书馆存储图书、出版社发行图书，镜像仓库存储和分发Docker镜像。

镜像仓库的工作流程：开发者构建镜像 → 推送到仓库 → 其他机器从仓库拉取镜像。就像我们写完文章后存到云端，其他人可以下载查看一样。

1存储功能

保存镜像的各层数据，支持版本管理

2分发功能

提供镜像上传、下载、Pull和Push

3认证功能

用户认证、权限管理、访问控制

4同步功能

支持镜像在多个仓库间同步

1.2

镜像仓库类型

仓库类型
Docker镜像仓库可分为公共仓库和私有仓库两种类型。
类型 说明 代表 适用场景
公共仓库 对所有人开放，可免费使用 Docker Hub、Quay 开源项目、公共镜像
私有仓库 需要认证，限制访问 Registry、Harbor、阿里云 企业内网、商业项目
选择建议

公共仓库：适用于使用开源镜像和分享开源项目

私有仓库：适用于企业项目、商业镜像、敏感环境

1.3

镜像命名与拉取

镜像命名规则 格式说明 
# 完整镜像名称格式
[registry]/[namespace]/[name]:[tag]

# 示例说明
# registry.docker.io/library/nginx:latest
# |          |          |     |      |
# 仓库地址   命名空间    镜像名  版本标签

# 简化规则：
# - Docker Hub可省略registry地址
# - library命名空间可省略
# - latest标签可省略
镜像拉取示例 Bash 
# 拉取官方镜像（library命名空间）
$ docker pull nginx:latest
$ docker pull ubuntu:22.04

# 拉取用户镜像
$ docker pull mysql/mysql-server:8.0

# 拉取私有仓库镜像
$ docker pull registry.example.com:5000/myapp:v1

# 拉取指定版本
$ docker pull nginx:1.25-alpine
说明

镜像名称由仓库地址、命名空间、镜像名和标签组成

私有仓库搭建

部署企业私有镜像仓库

2.1

部署Docker Registry

Docker Registry
Docker Registry是Docker官方开源的轻量级镜像仓库，可部署在企业内部存储镜像。最简单的部署方式是直接使用Docker官方提供的registry镜像。
快速启动（基础版） Bash 
# 在服务器上运行以下命令，启动一个运行在5000端口的Registry容器
$ docker run -d \
  -p 5000:5000 \
  --restart=always \
  --name my-registry \
  registry:2

# 参数说明：
# -d                 后台运行模式
# -p 5000:5000      端口映射，宿主机5000映射到容器5000
# --restart=always  开机自启
# --name my-registry 容器名称
# registry:2        Registry 2.x版本
挂载数据卷（生产推荐） Bash 
# 默认情况下，镜像存储在容器内部
# 为了防止容器删除后数据丢失，建议将宿主机目录挂载到容器中

$ docker run -d \
  -p 5000:5000 \
  --restart=always \
  --name my-registry \
  -v /mnt/registry:/var/lib/registry \
  registry:2

# -v /mnt/registry:/var/lib/registry
# 将宿主机的/mnt/registry目录映射为容器内的镜像存储目录
部署验证

Registry启动后，可以通过访问 http://localhost:5000/v2/ 验证是否正常运行

预期返回空JSON：{}

2.2

配置非安全仓库

为什么需要配置？
默认情况下，Docker强制要求通过HTTPS访问Registry。如果你是在局域网内测试且未配置SSL证书，需要手动允许HTTP访问。
Linux系统配置 Bash 
# 在客户端机器（推送或拉取镜像的机器）上修改配置文件
# 编辑 /etc/docker/daemon.json

$ sudo tee /etc/docker/daemon.json << 'EOF'
{
  "insecure-registries": ["localhost:5000"]
}
EOF

# 重启Docker服务
$ sudo systemctl restart docker
Windows/macOS配置 GUI 
# Docker Desktop设置步骤：

# 1. 打开Docker Desktop
# 2. 点击设置（Settings）
# 3. 选择 Docker Engine 选项卡
# 4. 在JSON配置中添加：
{
  "insecure-registries": ["localhost:5000"]
}
# 5. 点击 Apply & Restart
安全提示

生产环境建议配置HTTPS证书，不要使用insecure-registries

内网测试环境可以使用HTTP，但要注意网络安全

2.3

使用私有Registry

使用流程
使用私有仓库的逻辑非常简单：打标签 → 推送 → 拉取
1. 创建自定义镜像（安装vim） Bash 
# 创建Dockerfile目录
$ mkdir ~/my-ubuntu && cd ~/my-ubuntu

# 编写Dockerfile（基于ubuntu安装vim）
$ cat > Dockerfile << 'EOF'
FROM ubuntu:22.04

# 设置非交互式环境变量
ENV DEBIAN_FRONTEND=noninteractive

# 安装vim并清理缓存
RUN apt-get update && \
    apt-get install -y --no-install-recommends vim && \
    apt-get clean && \
    rm -rf /var/lib/apt/lists/*

# 设置默认命令
CMD ["vim", "--version"]
EOF

# 构建镜像
$ docker build -t my-ubuntu:v1 .
2. 为本地镜像打标签（Tag） Bash 
# 你需要将镜像名称修改为符合 [Registry地址]/[镜像名]:[标签] 的格式
# 例如，将本地的 my-ubuntu 镜像准备推送到私有仓库：

$ docker tag my-ubuntu:v1 localhost:5000/my-ubuntu:v1

# 查看镜像列表，确认标签已添加
$ docker images | grep my-ubuntu
3. 推送镜像（Push） Bash 
# 推送镜像到私有仓库
$ docker push localhost:5000/my-ubuntu:v1

# 预期输出：
# The push refers to repository [localhost:5000/my-ubuntu]
# v1: digest: sha256:abc123... size: 150MB
4. 拉取镜像（Pull） Bash 
# 在其他配置了"insecure-registries"的机器上执行：

# 先删除本地镜像（测试用）
$ docker rmi my-ubuntu:v1 localhost:5000/my-ubuntu:v1

# 从私有仓库拉取
$ docker pull localhost:5000/my-ubuntu:v1

# 运行容器验证vim是否安装成功
$ docker run --rm localhost:5000/my-ubuntu:v1
2.4

管理与查看

Registry API
Registry提供了RESTful API，可以通过curl命令查看仓库中的镜像信息和标签。
1. 查看仓库中的镜像列表 Bash 
# 查看目前仓库里有哪些镜像
$ curl -X GET http://localhost:5000/v2/_catalog

# 预期输出：
# {"repositories":["my-ubuntu"]}
2. 查看特定镜像的所有标签 Bash 
# 查看my-ubuntu镜像的所有标签
$ curl -X GET http://localhost:5000/v2/my-ubuntu/tags/list

# 预期输出：
# {"name":"my-ubuntu","tags":["v1"]}
3. 查看镜像Manifest信息 Bash 
# 获取镜像的详细元数据
$ curl -X GET \
  -H "Accept: application/vnd.docker.distribution.manifest.v2+json" \
  http://localhost:5000/v2/my-ubuntu/manifests/v1
常用API端点

/v2/ - 检查Registry是否正常运行

/v2/_catalog - 列出所有仓库

/v2/{name}/tags/list - 列出镜像的所有标签

/v2/{name}/manifests/{reference} - 获取镜像Manifest

Part 3

镜像分发与同步

实现镜像的高效分发

3.1

从容器创建镜像

docker commit
docker commit 命令用于将运行中的容器保存为新镜像。当你在一个容器内做了修改（如安装软件、修改配置），可以通过commit将其保存为新的镜像。
基本用法 Bash 
# 基本语法
$ docker commit [选项] 容器ID/名称 新镜像名:标签

# 示例：将正在运行的容器保存为新镜像
$ docker commit my-container my-image:v1

# 常用选项
# -a, --author=""      指定作者信息
# -m, --message=""     添加提交说明
# -p, --pause=true     提交时暂停容器
完整示例：基于Ubuntu安装vim后提交 Bash 
# 1. 启动一个ubuntu容器并进入
$ docker run -it --name my-ubuntu ubuntu:22.04 bash

# 2. 在容器内安装vim（容器内执行）
root@container:/# apt-get update
root@container:/# apt-get install -y vim
root@container:/# exit

# 3. 将修改后的容器提交为新镜像
$ docker commit -a "张三" -m "安装vim编辑器" my-ubuntu my-ubuntu:v2

# 4. 查看新创建的镜像
$ docker images | grep my-ubuntu

# 5. 测试新镜像
$ docker run --rm my-ubuntu:v2 vim --version
使用场景

快速保存：临时调试后保存容器状态

镜像定制：基于基础镜像快速安装软件

问题排查：保存出错现场用于分析

3.2

commit vs Dockerfile

两种方式对比
创建镜像有两种主要方式：docker commit（基于容器）和 Dockerfile（基于脚本）。
对比项 docker commit Dockerfile
使用场景 快速保存、临时调试 标准化构建、持续集成
可重复性 较低，依赖手动操作 高，脚本化可复现
镜像体积 可能较大（包含所有修改） 可控，可多阶段优化
版本管理 不便追踪变更 便于Git版本控制
推荐做法 Bash 
# 方式1：先commit测试，再写Dockerfile（推荐）
# 先用commit快速验证，确认后再编写Dockerfile用于生产

# 方式2：导出容器变更生成Dockerfile
$ docker diff my-container
# 根据diff输出编写对应的Dockerfile指令

# 方式3：使用docker history查看镜像构建历史
$ docker history my-ubuntu:v2
# 根据history反推Dockerfile结构
最佳实践

开发测试阶段可用commit快速迭代

生产环境建议使用Dockerfile确保可复现

3.3

镜像复制工具

docker tag重命名镜像 Bash 
# 将镜像复制到不同仓库（通过tag）
$ docker tag source-image:tag new-registry.com/project/image:tag
$ docker push new-registry.com/project/image:tag
Skopeo工具 Bash 
# 安装skopeo
$ apt install skopeo

# 复制镜像（不依赖Docker daemon）
$ skopeo copy \
  docker://docker.io/library/nginx:latest \
  docker://my-registry.com:5000/nginx:latest

# 查看远程镜像信息
$ skopeo inspect docker://my-registry.com:5000/nginx:latest
阿里云容器镜像服务 Bash 
# 1. 登录阿里云镜像服务
$ docker login --username=your_username registry.cn-hangzhou.aliyuncs.com

# 2. 打标签
$ docker tag myapp:v1 registry.cn-hangzhou.aliyuncs.com/your_namespace/myapp:v1

# 3. 推送
$ docker push registry.cn-hangzhou.aliyuncs.com/your_namespace/myapp:v1
3.4

镜像安全最佳实践

镜像签名 Bash 
# Docker Content Trust（DCT）启用
$ export DOCKER_CONTENT_TRUST=1

# 推送时自动签名
$ docker push your_username/myapp:v1

# 拉取时验证签名
$ docker pull your_username/myapp:v1
镜像扫描 Bash 
# Docker Scout扫描
$ docker scout cves nginx:latest

# Trivy扫描
$ trivy image nginx:latest
安全建议

- 使用官方认证镜像

- 定期更新镜像版本

- 启用镜像签名验证

- 企业内网使用私有仓库
