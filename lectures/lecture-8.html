Part 1

镜像仓库概述

理解镜像仓库的概念和类型

Part 1.1

什么是镜像仓库？

镜像仓库定义
Docker镜像仓库（Registry）是存储和分发镜像的服务系统，类似代码仓库（Git）存储代码，镜像仓库存储Docker镜像。

镜像仓库是Docker生态系统的核心组件，实现了镜像的版本管理、共享分发和存储管理。

1存储功能

保存镜像的各层数据，支持版本管理

2分发功能

提供镜像上传、下载、Pull和Push

3认证功能

用户认证、权限管理、访问控制

4同步功能

支持镜像在多个仓库间同步

Part 1.2

镜像仓库类型

仓库类型
Docker镜像仓库可分为公共仓库和私有仓库两种类型。
类型 说明 代表 适用场景
公共仓库 对所有人开放，可免费使用 Docker Hub、Quay 开源项目、公共镜像
私有仓库 需要认证，限制访问 Registry、Harbor、阿里云 企业内网、商业项目
选择建议

公共仓库：适用于使用开源镜像和分享开源项目

私有仓库：适用于企业项目、商业镜像、敏感环境

Part 1.3

镜像命名与拉取

镜像命名规则 格式说明 
# 完整镜像名称格式
[registry]/[namespace]/[name]:[tag]

# 示例说明
# registry.docker.io/library/nginx:latest
# |          |          |     |      |
# 仓库地址   命名空间    镜像名  版本标签

# 简化规则：
# - Docker Hub可省略registry地址
# - library命名空间可省略
# - latest标签可省略
镜像拉取示例 Bash 
# 拉取官方镜像（library命名空间）
$ docker pull nginx:latest
$ docker pull ubuntu:22.04

# 拉取用户镜像
$ docker pull mysql/mysql-server:8.0

# 拉取私有仓库镜像
$ docker pull registry.example.com:5000/myapp:v1

# 拉取指定版本
$ docker pull nginx:1.25-alpine
说明

镜像名称由仓库地址、命名空间、镜像名和标签组成

Part 2.2

Docker Hub账号与登录

注册Docker Hub账号 Web 
# 访问 Docker Hub官网注册
# https://hub.docker.com

# 注册信息：
# - Username: 用户名（唯一）
# - Email: 邮箱地址

搜索与拉取镜像

Part 2.4

推送镜像到Docker Hub

docker tag - 打标签 Bash 
# 为镜像打标签（格式：用户名/镜像名:标签）
$ docker tag myapp:v1 your_username/myapp:v1

# 查看镜像标签
$ docker images | grep myapp

# 预期输出：
# myapp                v1      abc123...   5 minutes ago   150MB
# your_username/myapp  v1      abc123...   5 minutes ago   150MB
docker push - 推送镜像 Bash 
# 推送镜像到Docker Hub
$ docker push your_username/myapp:v1

# 参数说明：
# your_username    Docker Hub用户名（必须与登录账号一致）
# myapp            镜像名称
# v1               镜像标签（版本号）

# 预期输出：
# The push refers to repository [docker.io/your_username/myapp]
# v1: digest: sha256:abc123...
# Size: 150MB
# latest: digest: sha256:abc123... size: 150MB

# 推送完成后，访问 https://hub.docker.com/repository/docker/your_username/myapp 查看
完整操作流程（WSL） Bash 
# 1. 在当前目录构建镜像
#    -t myapp:v1    指定镜像名和标签
#    .              使用当前目录的Dockerfile
$ docker build -t myapp:v1 .

# 2. 登录Docker Hub（按提示输入用户名和密码）
$ docker login

# 3. 为镜像打标签（格式：用户名/镜像名:标签）
#    注意：用户名必须与登录的Docker Hub账号一致
$ docker tag myapp:v1 your_username/myapp:v1

# 4. 推送镜像到Docker Hub
$ docker push your_username/myapp:v1

# 5. 在其他机器或WSL中拉取镜像
$ docker pull your_username/myapp:v1
Part 3

私有仓库搭建

部署企业私有镜像仓库

Part 3.1

Docker Registry部署

Docker Registry
Docker官方开源的轻量级镜像仓库，可部署在企业内部存储镜像。
快速部署Registry（WSL） Bash 
# 启动Registry容器
$ docker run -d \
  --name registry \
  --hostname registry \
  -p 5000:5000 \
  --restart=always \
  -v registry-data:/var/lib/registry \
  registry:2

# 参数详细说明：
# -d                          后台运行模式，不占用终端
# --name registry             指定容器名称为registry
# --hostname registry         设置容器内部主机名为registry
# -p 5000:5000               将宿主机的5000端口映射到容器的5000端口
#                             Registry默认监听5000端口
# --restart=always           容器退出后自动重启（开机自启）
# -v registry-data:/var/lib/registry \
#                             使用Docker数据卷registry-data持久化存储
#                             镜像数据保存在/var/lib/registry目录
# registry:2                 使用Registry 2版本镜像
验证Registry Bash 
# 查看Registry版本
$ curl http://localhost:5000/v2/

# 预期输出（空JSON）：
# {}

# 查看所有仓库
$ curl http://localhost:5000/v2/_catalog

# 预期输出：
# {"repositories":[]}
Part 3.2

推送镜像到私有仓库

配置 insecure-registries Bash 
# 由于使用HTTP，需要在/etc/docker/daemon.json添加：
{
  "insecure-registries": ["192.168.1.100:5000"]
}

# 重启Docker
$ sudo systemctl restart docker
推送镜像到私有仓库 Bash 
# 1. 拉取镜像作为示例
$ docker pull ubuntu:22.04

# 2. 打标签（指定仓库地址）
$ docker tag ubuntu:22.04 192.168.1.100:5000/my-ubuntu:v1

# 3. 推送到私有仓库
$ docker push 192.168.1.100:5000/my-ubuntu:v1

# 4. 验证推送
$ curl http://192.168.1.100:5000/v2/_catalog

# 预期输出：
# {"repositories":["my-ubuntu"]}
从私有仓库拉取 Bash 
# 拉取私有仓库镜像
$ docker pull 192.168.1.100:5000/my-ubuntu:v1

# 在其他机器上也可以这样拉取
$ docker pull 192.168.1.100:5000/my-ubuntu:v1
Part 3.3

Harbor企业级仓库

Harbor
Harbor是VMware开源的企业级Docker Registry，提供图形界面、权限管理、镜像复制、漏洞扫描等企业功能。
Harbor主要特性 特性说明 
# 1. 基于角色的访问控制（RBAC）
#    - 项目级别权限管理
#    - 用户角色：管理员、开发者、访客

# 2. 镜像复制
#    - 支持跨数据中心同步
#    - 自动同步策略

# 3. 漏洞扫描
#    - 集成Clair扫描器
#    - 安全报告生成

# 4. 图形界面
#    - Web UI管理
#    - 镜像浏览和搜索

# 5. RESTful API
#    - 完整的API支持
#    - 便于集成CI/CD
Harbor安装（Docker Compose方式） Bash 
# 1. 下载Harbor离线安装包
$ wget https://github.com/goharbor/harbor/releases/download/v2.8.0/harbor-offline-installer.tgz

# 2. 解压
$ tar xvf harbor-offline-installer.tgz

# 3. 配置harbor.yml
$ cp harbor.yml.tmpl harbor.yml
$ vi harbor.yml

# 4. 执行安装
$ ./install.sh

# 5. 访问Harbor Web界面
# http://192.168.1.100
Harbor配置要点

hostname：Harbor服务器地址

harbor_admin_password：管理员初始密码

database_password：数据库密码

Part 4.1

镜像导出与导入

docker save - 导出镜像（WSL） Bash 
# 导出单个镜像为tar文件
$ docker save nginx:latest -o nginx.tar

# 参数说明：
# nginx:latest    要导出的镜像名和标签
# -o nginx.tar    指定输出文件名（.tar格式）

# 导出多个镜像到同一个tar文件
$ docker save nginx:latest redis:alpine mysql:8.0 -o images.tar

# 查看tar文件内容（不实际导入）
$ docker load -i nginx.tar --list

# 重要：导出的tar文件包含镜像的所有层，建议压缩后传输
docker load - 导入镜像（WSL） Bash 
# 从tar文件导入镜像
$ docker load -i nginx.tar

# 参数说明：
# -i nginx.tar    指定要导入的tar文件（-i等于--input）

# 导入并查看结果
$ docker load -i nginx.tar | grep "Loaded image"

# 查看导入的镜像
$ docker images | grep nginx

# 预期输出：
# REPOSITORY   TAG      IMAGE ID       CREATED        SIZE
# nginx        latest   abc123...     5 minutes ago  150MB

Part 4.2

镜像压缩与传输

压缩导出 Bash 
# 导出并压缩（推荐用于网络传输）
$ docker save nginx:latest | gzip > nginx.tar.gz

# 查看压缩后的文件大小
$ ls -lh nginx.tar.gz

# 解压导入
$ gunzip -c nginx.tar.gz | docker load
远程传输 Bash 
# 方法1：使用scp传输
$ docker save nginx:latest | gzip | ssh user@remote "docker load"

# 方法2：使用rsync传输
$ docker save nginx:latest -o nginx.tar
$ rsync -avz nginx.tar user@remote:/path/

# 方法3：搭建本地仓库（推荐）

镜像镜像复制

docker tag 重命名镜像 Bash 
# 将镜像复制到不同仓库（通过tag）
$ docker tag source-image:tag new-registry.com/project/image:tag

Skopeo工具 Bash

# 3. 推送

# 4. 其他机器拉取
