Part 1.1

为什么容器数据会丢失？

容器文件系统特性
Docker容器的文件系统具有临时性和分层结构。当容器被删除时，容器内的所有数据都会丢失。
模拟数据丢失场景（Ubuntu容器） Bash 
# 启动一个Ubuntu容器，在容器中创建重要数据
$ docker run -it --name test-container ubuntu:22.04 bash

# 在容器中创建测试文件
root@abc123def456:/# mkdir -p /app/data
root@abc123def456:/# echo "这是重要数据" > /app/data/info.txt
root@abc123def456:/# ls -la /app/data/

# 退出容器并删除
root@abc123def456:/# exit
$ docker rm test-container

# 重新创建容器尝试访问数据
$ docker run -it --name new-container ubuntu:22.04 bash
root@xyz789abc012:/# cat /app/data/info.txt
cat: /app/data/info.txt: No such file or directory
# 数据已经丢失！
核心问题

✓ 容器删除后数据无法恢复

✓ 容器层数据读写性能较差（多层叠加）

✓ 多个容器无法共享同一份数据

Part 1.2

什么是Docker数据卷？

数据卷（Volume）
Docker数据卷是一个可供容器使用的特殊目录，它绕过容器文件系统，直接挂载到宿主机的文件系统中。数据卷的生命周期独立于容器，可以跨多个容器共享和重用。
数据卷工作原理 图示 
宿主机文件系统                        Ubuntu容器
┌────────────────────┐            ┌────────────────────┐
│ /var/lib/docker/   │            │ /app/data          │
│ volumes/my-data/   │◄──────────►│ (挂载点)           │
│ _data/             │   挂载     │                    │
│  - info.txt        │            │  - info.txt        │
│  - logs/           │            │  - logs/           │
└────────────────────┘            └────────────────────┘
     数据持久化保存                    容器内访问

1独立生命周期

删除容器不影响数据卷

2高性能读写

绕过容器层，直接与宿主机交互

3跨容器共享

可同时挂载给多个容器

4双向同步

修改立即生效

Part 1.3

数据卷的存储位置

Linux系统数据卷默认位置

数据卷默认存储在 /var/lib/docker/volumes/ 目录下

查看Ubuntu容器的数据卷信息 Bash 
# 创建一个带数据卷的Ubuntu容器
$ docker run -d --name ubuntu-data -v my-data:/data ubuntu:22.04 sleep 3600

# 查看数据卷详情
$ docker volume inspect my-data
[
    {
        "CreatedAt": "2024-01-15T10:30:00Z",
        "Driver": "local",
        "Mountpoint": "/var/lib/docker/volumes/my-data/_data",
        "Name": "my-data",
        "Scope": "local"
    }
]

# 在宿主机上查看数据卷内容
$ ls -la /var/lib/docker/volumes/my-data/_data/

# 在容器中写入数据
$ docker exec ubuntu-data sh -c 'echo "Hello from container" > /data/test.txt'

# 在宿主机上验证
$ cat /var/lib/docker/volumes/my-data/_data/test.txt
Hello from container
Windows/Mac系统说明

• Windows：C:\ProgramData\docker\volumes\

• Mac：通过Linux虚拟机存储

• 建议：使用docker命令查看

Part 1.4

三种存储方式对比

Docker提供了三种不同的数据存储方式：

存储类型说明数据持久性适用场景
数据卷（Volume）Docker管理的特殊目录持久，独立于容器生产数据库、日志
绑定挂载（Bind Mount）宿主机任意目录持久，依赖宿主机配置文件、开发源码
tmpfs挂载内存中的文件系统临时，容器停止即丢失敏感数据、临时缓存
容器存储层容器的可写层随容器删除而丢失临时测试数据
选择建议

• 生产数据库（MySQL）→ 使用数据卷

• 配置文件（nginx配置）→ 使用绑定挂载

• 敏感信息（密钥）→ 使用tmpfs挂载

Part 2.1

docker volume create - 创建数据卷

docker volume create命令详解 Bash 
# 语法：docker volume create [OPTIONS] [VOLUME_NAME]

# 1. 创建命名数据卷（最常用的方式）
$ docker volume create my-nginx-html

# 2. 创建带标签的数据卷（便于管理）
$ docker volume create \
  --label env=production \
  --label app=nginx \
  nginx-production-data

# 3. 查看创建的数据卷
$ docker volume ls
DRIVER    VOLUME NAME
local     my-nginx-html
local     nginx-production-data

# 4. 查看数据卷详情
$ docker volume inspect my-nginx-html
命名数据卷 vs 匿名数据卷

• 命名数据卷：有明确名称，便于管理和识别

• 匿名数据卷：自动生成随机ID，难以管理

• 推荐：始终使用命名数据卷

Part 2.2

docker volume ls/inspect/rm - 查看和管理

查看数据卷 Bash 
# 列出所有数据卷
$ docker volume ls

# 列出数据卷（过滤）
$ docker volume ls -f name=nginx

# 查看数据卷详细信息
$ docker volume inspect my-nginx-html

# 使用Go模板格式化输出
$ docker volume inspect --format '{{.Mountpoint}}' my-nginx-html
/var/lib/docker/volumes/my-nginx-html/_data
删除数据卷 Bash 
# 删除指定数据卷（必须是未使用的）
$ docker volume rm my-nginx-html

# 清理所有未使用的数据卷（谨慎使用！）
$ docker volume prune
WARNING! This will remove all local volumes not used by at least one container.
Are you sure you want to continue? [y/N] y

# 查看清理结果
$ docker volume ls
删除注意事项

• 无法删除正在被容器使用的数据卷

• 删除容器不会自动删除关联的数据卷

Part 2.3

docker run -v 挂载数据卷详解

docker run -v 是最常用的数据卷挂载方式：

语法：-v volume-name:container-path（推荐） Bash 
# 使用已存在的数据卷
$ docker run -d --name nginx-web \
  -v my-nginx-html:/usr/share/nginx/html \
  nginx:alpine

# 自动创建数据卷（如果不存在）
$ docker run -d --name nginx-web \
  -v nginx-html:/usr/share/nginx/html \
  nginx:alpine

# 参数说明：
# -v 数据卷名称:容器挂载点
#   数据卷名称：可以是已存在的或自动创建的
#   容器挂载点：容器内的目标路径
语法：-v /host/path:/container/path（绑定挂载） Bash 
# 绑定挂载宿主机目录
$ docker run -d --name nginx-web \
  -v /home/user/nginx/html:/usr/share/nginx/html \
  nginx:alpine

# 自定义nginx配置
$ docker run -d --name nginx-web \
  -v /home/user/nginx/conf:/etc/nginx/conf.d \
  -v /home/user/nginx/logs:/var/log/nginx \
  nginx:alpine
Part 2.4

docker run --mount 挂载详解

--mount 是更明确的挂载方式，语法更清晰，推荐在生产环境使用：

--mount 挂载数据卷 Bash 
# 基本语法：--mount type=volume,source=卷名,target=容器路径

# 使用数据卷挂载
$ docker run -d --name nginx-web \
  --mount type=volume,source=nginx-html,target=/usr/share/nginx/html \
  nginx:alpine

# 只读挂载（readonly）
$ docker run -d --name nginx-web \
  --mount type=volume,source=nginx-html,target=/usr/share/nginx/html,readonly \
  nginx:alpine

# 同时挂载多个数据卷
$ docker run -d --name nginx-web \
  --mount type=volume,source=nginx-html,target=/usr/share/nginx/html \
  --mount type=volume,source=nginx-logs,target=/var/log/nginx \
  nginx:alpine
对比：-v 和 --mount 对比 
# -v 语法（简洁）
$ docker run -d --name web -v nginx-data:/app nginx:alpine

# --mount 语法（明确，推荐）
$ docker run -d --name web --mount type=volume,source=nginx-data,target=/app nginx:alpine

# 为什么推荐 --mount？
# 1. 语法更清晰，明确指定 type、source、target
# 2. 错误提示更详细
# 3. 支持更多选项（readonly、volume-opt等）

查看容器挂载信息

查看Ubuntu容器挂载信息 Bash 
# 创建带多个数据卷的Ubuntu容器
$ docker run -d --name ubuntu-app \
  -v nginx-html:/var/www/html \
  -v nginx-logs:/var/log/app \

# 查看容器所有信息
$ docker inspect ubuntu-app

# 只查看挂载信息（Mounts部分）
$ docker inspect --format '{{json .Mounts}}' ubuntu-app | jq
[
  {
    "Type": "volume",
    "Name": "nginx-html",
    "Source": "/var/lib/docker/volumes/nginx-html/_data",
    "Destination": "/var/www/html",
    "RW": true
  }
]
字段说明 解释 
Type: volume          # 挂载类型（volume/bind/tmpfs）
Name: nginx-html      # 数据卷名称
Source: /var/lib/... # 宿主机上的路径（Mountpoint）
Destination: /var/www/html  # 容器内的挂载点
RW: true              # 读写权限（true=可写，false=只读）

挂载类型详解

深入理解三种挂载方式的特点和使用场景

Part 3.1

绑定挂载（Bind Mounts）详解

绑定挂载
绑定挂载将宿主机上的任意目录或文件挂载到容器中。容器可以完全访问宿主机指定路径的内容。
绑定挂载nginx配置文件（Ubuntu容器示例） Bash 
# 创建宿主机上的配置目录
$ mkdir -p ~/nginx-config ~/nginx-html ~/nginx-logs

# 创建自定义nginx配置文件
$ cat > ~/nginx-config/default.conf << 'EOF'
server {
    listen 80;
    server_name localhost;
    root /usr/share/nginx/html;
    
    location / {
        try_files $uri $uri/ =404;
    }
    
    location /health {
        return 200 "OK";
        add_header Content-Type text/plain;
    }
}
EOF

# 创建网页文件
$ echo "
Custom Website via Bind Mount
" > ~/nginx-html/index.html

# 启动nginx使用绑定挂载
$ docker run -d --name nginx-bind \
  -p 8080:80 \
  -v ~/nginx-config:/etc/nginx/conf.d:ro \
  -v ~/nginx-html:/usr/share/nginx/html:ro \
  -v ~/nginx-logs:/var/log/nginx \
  nginx:alpine

# 测试
$ curl http://localhost:8080
$ curl http://localhost:8080/health
使用场景

• 开发环境：挂载源代码，修改实时生效

• 配置文件：使用宿主机编辑器修改配置

• 日志收集：宿主机直接访问容器日志

Part 3.2

tmpfs挂载详解

tmpfs挂载
tmpfs挂载将数据存储在宿主机的内存中，而不是磁盘上。读写速度极快，但容器停止后数据会丢失。
tmpfs挂载示例（Ubuntu容器） Bash 
# 创建带tmpfs挂载的Ubuntu容器
$ docker run -d --name ubuntu-tmpfs \
  --tmpfs /app/secrets \
  --tmpfs /app/cache \
  ubuntu:22.04 sleep 3600

# 使用--mount语法指定更多选项
$ docker run -d --name ubuntu-tmpfs2 \
  --mount type=tmpfs,target=/app/secrets,tmpfs-size=512m,tmpfs-mode=0770 \
  ubuntu:22.04 sleep 3600

# 参数说明：
# tmpfs-size: 限制内存大小（默认无限制）
# tmpfs-mode: 文件权限模式（默认1777）

# 验证tmpfs挂载
$ docker exec ubuntu-tmpfs df -h | grep tmpfs
tmpfs  512M     0  512M   0% /app/secrets
tmpfs  512M     0  512M   0% /app/cache

# 写入数据到tmpfs
$ docker exec ubuntu-tmpfs sh -c 'echo "sensitive data" > /app/secrets/token.txt'
$ docker exec ubuntu-tmpfs cat /app/secrets/token.txt

# 重启容器后数据丢失
$ docker restart ubuntu-tmpfs
$ docker exec ubuntu-tmpfs cat /app/secrets/token.txt
cat: /app/secrets/token.txt: No such file or directory

1高性能

基于内存，读写速度极快

2数据安全

容器停止后自动清空

3内存占用

会占用宿主机内存

Part 4.1

实战：nginx数据持久化完整案例

生产环境nginx部署（Ubuntu环境） Bash 
# 步骤1：创建必要的数据卷
$ docker volume create nginx-web-data
$ docker volume create nginx-config
$ docker volume create nginx-logs

# 步骤2：初始化配置文件（通过Ubuntu容器）
$ docker run --rm -v nginx-config:/config ubuntu:22.04 sh -c 'cat > /config/default.conf << "EOF"
server {
    listen 80;
    server_name localhost;
    root /usr/share/nginx/html;
    access_log /var/log/nginx/access.log;
    error_log /var/log/nginx/error.log;
    location / { index index.html index.htm; }
}
EOF'

# 步骤3：初始化网页内容
$ docker run --rm -v nginx-web-data:/html ubuntu:22.04 sh -c 'cat > /html/index.html << "EOF"
Production Nginx

Welcome to Production Nginx

EOF'

# 步骤4：启动生产nginx容器
$ docker run -d --name production-nginx \
  -p 80:80 \
  -v nginx-web-data:/usr/share/nginx/html:ro \
  -v nginx-config:/etc/nginx/conf.d:ro \
  -v nginx-logs:/var/log/nginx \
  --restart unless-stopped \
  nginx:alpine

# 步骤5：验证
$ curl http://localhost
生产环境要点

✓ 使用命名数据卷便于管理

✓ 配置文件和网页目录只读挂载（:ro）

✓ 设置restart unless-stopped自动重启

Part 4.2

数据卷权限管理

数据卷权限问题是一个常见的坑，需要注意UID/GID匹配：

权限问题排查（Ubuntu容器示例） Bash 
# 问题：nginx无法写入日志
$ docker run -d --name nginx-perm -v nginx-logs:/var/log/nginx nginx:alpine
$ docker logs nginx-perm
# 错误：Permission denied

# 排查方法1：查看容器内进程用户
$ docker exec nginx-perm id
uid=101(nginx) gid=101(nginx) groups=101(nginx)

# 排查方法2：查看数据卷宿主机权限
$ ls -la /var/lib/docker/volumes/nginx-logs/_data/
drwxr-xr-x 2 root root 999 root 4096 Jan 15 10:30 .

# 解决方案1：修改宿主机权限
$ sudo chown -R 101:101 /var/lib/docker/volumes/nginx-logs/_data

# 解决方案2：使用Z标签（SELinux）
$ docker run -d --name nginx-selinux \
  -v nginx-logs:/var/log/nginx:Z \
  nginx:alpine

# 解决方案3：使用ro只读挂载日志目录
$ docker run -d --name nginx-ro \
  -v nginx-web:/usr/share/nginx/html:ro \
  nginx:alpine
常见容器用户UID参考 Bash 
# nginx容器
$ docker run --rm nginx:alpine id
uid=101(nginx) gid=101(nginx)

# Ubuntu容器
$ docker run --rm ubuntu:22.04 id
uid=0(root) gid=0(root)

# MySQL容器
$ docker run --rm mysql:8.0 id
uid=999(mysql) gid=999(mysql)
Part 4.3

数据卷备份与恢复

数据安全至关重要，定期备份数据卷是生产环境的必要操作：

备份数据卷（使用Ubuntu容器） Bash 
# 创建测试数据
$ docker run -d --name nginx-backup \
  -v nginx-backup-data:/data \
  nginx:alpine
$ docker exec nginx-backup sh -c 'echo "Important data $(date)" > /data/backup.txt'

# 方法1：备份到宿主机
$ docker run --rm -v nginx-backup-data:/data -v $(pwd):/backup ubuntu:22.04 \
  tar czf /backup/nginx-backup-$(date +%Y%m%d).tar.gz -C /data .

# 验证备份
$ ls -lh nginx-backup-*.tar.gz

# 方法2：备份到指定目录
$ mkdir -p ~/backups
$ docker run --rm \
  -v nginx-backup-data:/data \
  -v ~/backups:/backup \
  ubuntu:22.04 \
  tar czf /backup/nginx-backup-latest.tar.gz -C /data .
恢复数据卷 Bash 
# 步骤1：创建新数据卷
$ docker volume create nginx-restore-data

# 步骤2：从备份恢复
$ docker run --rm \
  -v nginx-restore-data:/data \
  -v ~/backups:/backup \
  ubuntu:22.04 \
  tar xzf /backup/nginx-backup-latest.tar.gz -C /data

# 步骤3：验证恢复
$ docker run -d --name nginx-restore \
  -v nginx-restore-data:/usr/share/nginx/html \
  nginx:alpine
$ docker exec nginx-restore cat /usr/share/nginx/html/backup.txt
Part 4.4

数据卷使用最佳实践总结

推荐做法

  • 使用命名数据卷而非匿名卷
  • 不需要写入的数据使用只读挂载（:ro）
  • 为数据卷设置合理的备份策略
  • 定期使用 docker volume prune 清理
  • 监控数据卷的空间使用情况
  • 生产环境使用专门的存储驱动
  • 注意数据卷的权限和SELinux问题

避免做法

  • 在容器内直接存储重要数据
  • 使用绑定挂载访问系统敏感目录
  • 多个容器写入同一数据卷时未考虑并发
  • 忽略数据卷的权限问题
  • 生产环境使用匿名卷
  • 删除容器前未备份重要数据
  • 使用 docker volume prune 前未确认
检查数据卷使用情况 Bash 
# 查看Docker存储使用
$ docker system df

动手实验：nginx+配置文件管理

实验 nginx自定义配置与多容器数据共享

实验目的

掌握nginx配置文件的绑定挂载、多个容器共享数据卷、自定义网页

实验步骤

1创建宿主机目录结构
2编写自定义nginx配置
3创建网页内容

实验命令 Bash 
# 步骤1：创建宿主机目录结构
$ mkdir -p ~/nginx-project/{html,config,logs}
$ cd ~/nginx-project

# 步骤2：编写nginx配置文件
$ cat > config/default.conf << 'EOF'
server {
    listen 80;
    server_name localhost;
    root /usr/share/nginx/html;
    location / { index index.html; }
    location /status { stub_status on; access_log off; }
    location /api { return 200 '{"status":"ok"}'; add_header Content-Type application/json; }
}
EOF

# 步骤3：创建网页内容
$ cat > html/index.html << 'EOF'

Nginx Custom Config

Welcome to Custom Nginx


Configuration managed via bind mount


  • Server Status
  • API


EOF

# 步骤4：启动nginx容器
$ docker run -d --name my-nginx-config \
  -p 8080:80 \
  -v $(pwd)/config:/etc/nginx/conf.d:ro \
  -v $(pwd)/html:/usr/share/nginx/html:ro \
  -v $(pwd)/logs:/var/log/nginx \
  nginx:alpine

# 步骤5：测试
$ curl http://localhost:8080
$ curl http://localhost:8080/status
$ curl http://localhost:8080/api

# 步骤6：修改配置验证热更新
$ cat > config/default.conf << 'EOF'
server {
    listen 80;
    server_name localhost;
    root /usr/share/nginx/html;
    location / { index index.html; }
    location /status { stub_status on; access_log off; }
    location /api { return 200 '{"version":"2.0","status":"healthy"}'; add_header Content-Type application/json; }
    location /time { return 200 "{\"time\":\"$(date -Iseconds)\"}"; add_header Content-Type application/json; }
}
EOF

# 重新加载nginx配置
$ docker exec my-nginx-config nginx -s reload

# 验证更新
$ curl http://localhost:8080/api
$ curl http://localhost:8080/time

# 步骤7：清理
$ docker stop my-nginx-config && docker rm my-nginx-config
$ rm -rf ~/nginx-project
Part 4.5

常见问题与解决方案

问题1：权限被拒绝 问题 
# 症状：容器无法写入数据卷
$ docker logs my-nginx
# [alert] open() failed (13: Permission denied)

# 解决方案：使用Z标签
$ docker stop my-nginx && docker rm my-nginx
$ docker run -d --name my-nginx \
  -v nginx-logs:/var/log/nginx:Z \
  nginx:alpine
问题2：数据卷空间不足 问题 
# 排查
$ docker system df -v | grep -A 5 "Local Volumes"

# 解决方案1：清理未使用的卷
$ docker volume prune -f

# 解决方案2：扩大Docker存储空间
# Docker Desktop -> Settings -> Resources -> Disk image size
问题3：Windows路径问题 问题 
# 错误：invalid mount path

# 解决方案1：使用绝对路径
docker run -v //c/users/project:/app nginx:alpine

# 解决方案2：使用PowerShell变量
docker run -v ${PWD}:/app nginx:alpine

# 解决方案3：Docker Desktop设置共享驱动器
# Settings -> Resources -> File Sharing
