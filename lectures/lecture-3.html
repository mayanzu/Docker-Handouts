Part 1

容器技术概述

从轻量级虚拟化到云原生

Part 1.1

什么是容器？

容器定义
容器是一种操作系统级虚拟化技术，它将应用程序及其依赖打包在一起，在隔离的环境中运行，共享宿主机的操作系统内核，但拥有独立的文件系统、进程空间和网络接口。

可以把容器想象成一个标准化的集装箱。就像集装箱统一了货物的包装标准，容器统一了应用程序的打包和运行标准。无论在开发环境还是生产环境，容器都能确保应用以相同的方式运行。

容器的核心特性

1. 轻量级：共享宿主机内核，启动速度快，资源占用少

2. 可移植：一次构建，到处运行，环境一致性保证

3. 隔离性：进程、文件系统、网络相互隔离

4. 标准化：统一的打包、分发、运行标准

Part 1.1.1

为什么容器能隔离环境？

作为开发者，你是否遇到过这些问题？

开发中的常见痛点

- 项目A需要Python 3.8，项目B需要Python 3.11，怎么办？

- 本地开发正常，部署到服务器就报错？

- 两个项目都要用80端口，冲突了？

- 新同事入职，配环境花了一整天？

容器的隔离机制完美解决了这些问题！每个容器就像一个独立的"小房间"，互不干扰。

隔离内容 实际好处
文件系统隔离 每个容器有独立的文件系统，Python 3.8和3.11可以共存
网络隔离 每个容器有独立的端口空间，都能用80端口
进程隔离 容器内的进程互不可见，互不影响
用户隔离 容器内的root不是宿主机的root，更安全
一句话理解

容器让每个应用都以为自己独占一台服务器，实际上它们共享同一台服务器的资源。

Part 1.1.2

为什么容器能节约资源？

传统虚拟机和容器的资源占用对比：

对比项 虚拟机 容器
启动时间 几分钟 几秒钟
内存占用 几GB（含完整操作系统） 几十MB（仅应用本身）
磁盘占用 几十GB 几百MB
单机数量 10-20个 100-1000个
为什么差这么多？

1. 共享内核：容器不需要单独的操作系统，直接用宿主机的Linux内核

2. 按需分配：容器只占用应用需要的资源，不像虚拟机要预留整块内存

3. 快速启停：秒级启动，不用的容器可以随时停掉，释放资源

实际案例：一台4核8G服务器 text 
虚拟机方案：
- 每个虚拟机分配 1核2G
- 最多运行 3-4 个虚拟机
- 大量资源浪费在虚拟机系统上

容器方案：
- 每个容器按需分配（如 0.5核512M）
- 可以运行 20-50 个容器
- 资源利用率提升 5-10 倍
Part 1.1.3

为什么容器让开发更省事？

镜像分层：一次构建，处处运行

Docker镜像采用分层设计，就像盖楼一样：

- 底层：Ubuntu基础镜像（别人已经做好了）

- 中层：安装Python、依赖库（可以复用）

- 顶层：你的应用代码（每次更新只改这层）

分层带来的好处 text 
场景：你有10个Python项目

传统方式：
- 每个项目都要安装一次Python环境
- 重复下载、重复安装
- 总共需要 10 × 500MB = 5GB

容器方式：
- 基础Python镜像只存一份
- 10个项目共享同一层
- 总共只需要 500MB + 10×10MB ≈ 600MB

节省 90% 的存储空间！
开发者的一天

以前：

- 早上配环境 → 中午终于跑起来 → 下午发现版本不对 → 晚上加班重装

现在：

- 早上 docker run → 秒级启动 → 开始写代码 → 准时下班

一句话总结

容器把"配环境"这件烦心事变成了"一条命令搞定"，让开发者专注于写代码，而不是折腾环境。

容器让大规模服务管理变得简单可行

Part 1.2

容器 vs 虚拟机

容器和虚拟机都是虚拟化技术，但实现层次和设计理念不同：

对比项 虚拟机（VM） 容器（Container）
架构层次 硬件级虚拟化 操作系统级虚拟化
操作系统 每个VM独立内核 共享宿主机内核
启动时间 分钟级 秒级/毫秒级
资源占用 GB级 MB级
性能损耗 5-15% 接近原生
隔离强度 强隔离 进程级隔离
选择建议

需要强隔离和运行不同操作系统时选择虚拟机；需要快速启动、高密度部署和微服务架构时选择容器。

Part 1.3

容器技术的发展历程

时间 里程碑 意义
1979 Unix chroot 最早的容器雏形，隔离文件系统
2000 FreeBSD Jails 实现进程和网络隔离
2006 Google Process Containers 资源限制和隔离
2008 LXC发布 Linux容器标准化
2013 Docker发布 容器技术大众化
2015 Kubernetes 1.0发布 容器编排时代开启
2015 OCI成立 容器标准化组织
关键技术演进

- chroot：文件系统隔离的起点

- cgroups：Google贡献的资源限制技术

- Namespace：Linux内核提供的隔离机制

- UnionFS：分层文件系统，镜像的基础

Part 2

Docker简介

容器技术的集大成者

Part 2.1

什么是Docker？

Docker定义
Docker是一个开源的容器化平台，它使开发者能够将应用程序及其依赖打包到一个轻量级、可移植的容器中，然后在任何环境中一致地运行。Docker简化了容器的创建、部署和管理过程。

Docker公司最初由Solomon Hykes于2013年创立，最初是dotCloud公司的内部项目。Docker开源后迅速成为容器技术的事实标准，推动了整个容器生态系统的发展。

Docker的核心价值

1. Build Once, Run Anywhere：一次构建，到处运行

2. 环境一致性：开发、测试、生产环境完全一致

3. 快速交付：加速应用开发和部署流程

4. 资源高效：一台服务器可运行数百个容器

Part 2.2

Docker生态系统

Docker拥有丰富的生态系统，涵盖了容器生命周期的各个阶段：

1Docker Engine

Docker的核心运行时，负责容器的创建、运行和管理

2Docker Hub

官方的镜像仓库，提供海量预构建镜像

3Docker Compose

多容器编排工具，通过YAML定义应用栈

4Dockerfile

镜像构建脚本，定义镜像制作步骤

云原生生态

- Kubernetes：容器编排平台，大规模容器管理

- Prometheus：容器监控和告警

- Jenkins/GitLab CI：容器化CI/CD流水线

Part 3

Docker架构

理解Docker的工作原理

Part 3.1

Docker核心组件

1Docker Daemon

dockerd，后台服务进程，监听Docker API请求，管理镜像、容器、网络和存储

2Docker Client

docker命令行工具，用户与Docker交互的接口，通过REST API与Daemon通信

3Docker Registry

镜像仓库，存储和分发Docker镜像，如Docker Hub、私有仓库

4containerd

容器运行时，管理容器生命周期，Docker 18.09+的核心组件

架构演进

Docker早期使用LXC作为容器运行时，后来开发了自己的libcontainer，再后来将运行时拆分为runc和containerd，遵循OCI标准。

Part 3.2

Docker工作流程

Docker典型工作流程 流程说明 
1. 编写Dockerfile定义应用环境
   FROM ubuntu:22.04
   RUN apt-get update && apt-get install -y python3
   COPY app.py /app/
   CMD ["python3", "/app/app.py"]

2. 构建镜像
   docker build -t myapp:1.0 .

3. 推送镜像到仓库（可选）
   docker push myregistry/myapp:1.0

4. 拉取并运行容器
   docker pull myregistry/myapp:1.0
   docker run -d -p 8080:80 myapp:1.0

5. 管理容器生命周期
   docker ps          # 查看运行中的容器
   docker stop/start  # 停止/启动容器
   docker rm          # 删除容器
理解要点

Docker采用客户端-服务器架构，客户端发送命令，Daemon执行实际操作。镜像采用分层存储，容器是镜像的运行实例。

Part 3.3

Docker底层技术

Docker基于Linux内核的以下关键技术实现容器隔离：

技术 功能 作用
Namespace 命名空间隔离 隔离进程、网络、文件系统等
cgroups 控制组 限制资源使用（CPU、内存、IO）
UnionFS 联合文件系统 镜像分层、写时复制
iptables 网络过滤 容器网络隔离和端口映射
Namespace类型

- PID Namespace：进程ID隔离

- Network Namespace：网络隔离

- Mount Namespace：文件系统挂载点隔离

- UTS Namespace：主机名隔离

- IPC Namespace：进程间通信隔离

Docker安装与配置

搭建Docker运行环境

Part 4.1

在Ubuntu上安装Docker

Docker安装步骤（Ubuntu） Bash 
# 1. 更新包索引
sudo apt-get update

# 2. 安装必要的依赖包
sudo apt-get install -y apt-transport-https ca-certificates curl gnupg lsb-release

# 3. 添加Docker官方GPG密钥
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

# 4. 设置Docker稳定版仓库
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 5. 更新包索引并安装Docker
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 6. 验证安装
sudo docker --version
sudo docker run hello-world
安装后配置

# 将当前用户加入docker组，免sudo使用docker

sudo usermod -aG docker $USER

newgrp docker # 刷新组权限

Part 4.2

在CentOS上安装Docker

Docker安装步骤（CentOS） Bash 
# 1. 卸载旧版本（如有）
sudo yum remove docker docker-client docker-client-latest docker-common docker-latest docker-latest-logrotate docker-logrotate docker-engine

# 2. 安装yum工具
sudo yum install -y yum-utils

# 3. 添加Docker仓库
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# 4. 安装Docker
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 5. 启动Docker服务
sudo systemctl start docker
sudo systemctl enable docker  # 开机自启

# 6. 验证安装
sudo docker --version
sudo docker run hello-world

Part 4.3

Docker配置镜像加速

配置镜像加速器可以显著提高镜像下载速度，以下是常用国内镜像源：

配置阿里云镜像加速 Bash 
# 1. 创建/编辑daemon配置文件
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://your-id.mirror.aliyuncs.com",
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ]
}
EOF

# 2. 重启Docker服务
sudo systemctl daemon-reload
sudo systemctl restart docker

# 3. 验证配置
docker info | grep -A 5 "Registry Mirrors"

docker run -d nginx                    # -d: 后台运行
docker run -it ubuntu /bin/bash        # -it: 交互式运行

# docker stop - 停止运行中的容器（常用）
docker stop container_id
docker stop my-nginx

# docker start - 启动已停止的容器（常用）
docker start container_id
docker start my-nginx

# docker restart - 重启容器
docker restart my-nginx

# docker rm - 删除容器（常用）
docker rm container_id
docker rm -f my-nginx                  # -f: 强制删除运行中的容器
run命令常用参数

- -d：后台运行（detached）

- -p：端口映射（如 -p 8080:80）

- -v：挂载卷（如 -v /host:/container）

- --name：指定容器名称

- -e：设置环境变量

- --rm：容器停止后自动删除

- --restart：自动重启策略

Part 5.3.2

交互与调试命令

docker exec - 在容器内执行命令 Bash 
# docker exec - 在运行中的容器内执行命令（常用）
docker exec -it container_id /bin/bash  # 进入容器获得bash shell
docker exec my-nginx nginx -t           # 测试nginx配置

# docker logs - 查看容器日志（常用）
docker logs container_id                # 查看全部日志
docker logs -f my-nginx                 # -f: 持续跟踪日志
docker logs --tail 100 my-nginx         # 查看最后100行日志

# docker inspect - 查看容器/镜像详细信息
docker inspect container_id             # 查看容器详情
docker inspect nginx                    # 查看镜像详情
容器进入与退出 Bash 
# docker run -it - 创建并进入交互式容器
$ docker run -it ubuntu:22.04 /bin/bash
root@3a2b1c4d5e6f:/#     # 已进入容器

# 退出容器的方法：
# 方法1：exit 命令 - 退出并停止容器
root@3a2b1c4d5e6f:/# exit

# 方法2：Ctrl + D - 快捷键退出（等效于exit）

# 方法3：Ctrl + P + Q - 分离容器（不停止容器）
root@3a2b1c4d5e6f:/#    # 按 Ctrl+P，然后按 Q

# docker exec -it - 进入运行中的容器（推荐）
$ docker exec -it my-nginx /bin/bash
root@3a2b1c4d5e6f:/#     # 已进入容器

# 在容器内执行命令
root@3a2b1c4d5e6f:/# ls -la /etc/nginx/
root@3a2b1c4d5e6f:/# ps aux

# 退出（exec进入的，exit不会停止容器）
root@3a2b1c4d5e6f:/# exit
进入容器的两种方式对比

1. docker run -it：创建新容器并进入，适合一次性调试

2. docker exec -it：进入已运行的容器，推荐日常使用

3. docker attach：附加到主进程，exit会停止容器，不推荐

退出容器的三种方式

1. exit：退出shell，前台容器会停止

2. Ctrl + D：等效于exit命令

3. Ctrl + P + Q：分离容器，容器继续运行

部署方式对比总结

对比项 传统部署 Docker部署
部署时间 5-10分钟 30秒内
命令数量 10+条命令 1条命令
依赖冲突 可能存在库版本冲突 完全隔离

5. 版本灵活：轻松切换版本，支持多版本并行运行
