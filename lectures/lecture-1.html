Part 1

Linux系统概述

了解Linux的前世今生

Part 1.1

什么是Linux？

核心定义
Linux是一种自由和开放源码的类UNIX操作系统内核，由林纳斯·托瓦兹于1991年首次发布。它是一个多用户、多任务的操作系统。

Linux操作系统广泛应用于服务器、超级计算机、嵌入式设备等领域。在容器技术领域，Linux更是不可或缺的基础平台，Docker等容器技术正是基于Linux内核特性实现的。

Linux特点

1. 开源免费：源代码公开，可自由使用和修改

2. 稳定可靠：可连续运行数年无需重启

3. 安全高效：权限机制完善，资源占用低

4. 可定制性强：可根据需求定制系统功能

Part 1.2

Linux发行版

Linux发行版是基于Linux内核的完整操作系统，包含内核、系统库、工具软件等。常见的发行版如下：

发行版 特点 适用场景
Ubuntu 用户友好，社区活跃 桌面、开发、服务器
CentOS 稳定，企业级支持 服务器、生产环境
Debian 稳定，软件包丰富 服务器、开发
Fedora 新技术前沿 开发、测试
提示

本课程推荐使用 Ubuntu 20.04/22.04 或 CentOS 7/8 作为学习环境，这也是Docker官方推荐的基础镜像。

Part 2

常用命令

掌握Linux操作的核心技能

Part 2.1

目录操作命令

目录操作示例 Bash 
# 显示当前目录
pwd

# 列出目录内容
ls
ls -l    # 详细信息
ls -a    # 显示隐藏文件
ls -lh   # 人性化显示文件大小

# 切换目录
cd /home         # 切换到/home目录
cd ~             # 切换到用户主目录
cd ..            # 切换到上级目录
cd -             # 切换到上一个目录

# 创建目录
mkdir dir1              # 创建单个目录
mkdir -p dir1/dir2      # 递归创建多级目录

# 删除空目录
rmdir dir1
命令说明

- pwd：Print Working Directory，显示当前工作目录

- ls：List，列出目录内容

- cd：Change Directory，切换目录

- mkdir：Make Directory，创建目录

Part 2.2

文件操作命令

文件操作示例 Bash 
# 创建空文件
touch file.txt

# 复制文件
cp file1.txt file2.txt       # 复制文件
cp -r dir1 dir2              # 递归复制目录

# 移动/重命名文件
mv file1.txt file2.txt       # 重命名
mv file.txt /tmp/            # 移动文件

# 删除文件
rm file.txt                  # 删除文件
rm -r dir1                   # 递归删除目录
rm -rf dir1                  # 强制递归删除（谨慎使用）

# 查看文件内容
cat file.txt                 # 显示全部内容
head -n 10 file.txt          # 显示前10行
tail -n 10 file.txt          # 显示后10行
tail -f log.txt              # 实时追踪文件末尾
警告

rm -rf 命令非常危险，会强制删除指定目录及其所有内容，且无法恢复。执行前务必确认路径正确！

Part 2.3

文件查找与内容搜索

查找与搜索示例 Bash 
# 查找文件
find /home -name "*.txt"     # 按名称查找
find /home -type f           # 查找所有文件
find /home -type d           # 查找所有目录
find /home -size +100M       # 查找大于100M的文件

# 搜索文件内容
grep "error" log.txt         # 在文件中搜索关键词
grep -r "error" /var/log/    # 递归搜索目录
grep -i "error" log.txt      # 忽略大小写
grep -n "error" log.txt      # 显示行号

# 组合使用
find . -name "*.log" | xargs grep "error"

1find命令

用于查找文件，支持按名称、类型、大小、时间等条件搜索

2grep命令

用于搜索文本内容，支持正则表达式，是日志分析的利器

Part 3

文件系统

理解Linux目录结构

Part 3.1

Linux目录结构

Linux采用树形目录结构，所有文件和目录都从根目录 / 开始。了解标准目录结构对于系统管理至关重要。

目录 说明 重要程度
/ 根目录，所有文件的起点 核心
/home 用户主目录 核心
/etc 系统配置文件 核心
/var 可变数据（日志、缓存等） 重要
/usr 用户程序和数据 重要
/tmp 临时文件 一般
/root root用户主目录 重要
Part 3.2

路径概念

绝对路径
从根目录 / 开始的完整路径，例如：/home/user/documents/file.txt
相对路径
相对于当前工作目录的路径，例如：./documents/file.txt 或 ../config/app.conf
特殊符号

- . 表示当前目录

- .. 表示上级目录

- ~ 表示当前用户的主目录

- - 表示上一个工作目录

最佳实践

在脚本中使用绝对路径可以避免因工作目录变化导致的错误。

Part 4

权限管理

保障系统安全的核心机制

Part 4.1

权限概念

Linux采用多用户权限机制，每个文件和目录都有明确的权限设置，控制不同用户对资源的访问。

1读权限 r

可以查看文件内容，列出目录内容。数值为 4

2写权限 w

可以修改文件内容，在目录中创建/删除文件。数值为 2

3执行权限 x

可以执行文件（脚本、程序），进入目录。数值为 1

权限分组

- 所有者 Owner：文件的创建者

- 所属组 Group：文件所属的用户组

- 其他用户 Others：其他所有用户

Part 4.2

权限查看与修改

权限操作示例 Bash 
# 查看权限
ls -l file.txt
# 输出示例：-rwxr-xr-- 1 user group 1024 Jan 1 10:00 file.txt
# 解释：-文件类型 rwx所有者权限 r-x组权限 r--其他用户权限

# 数字方式修改权限
chmod 755 script.sh    # rwxr-xr-x
chmod 644 file.txt     # rw-r--r--
chmod 700 private/     # rwx------

# 符号方式修改权限
chmod u+x script.sh    # 给所有者添加执行权限
chmod g-w file.txt     # 移除组的写权限
chmod o=r file.txt     # 设置其他用户只读
chmod a+x script.sh    # 所有人添加执行权限

# 修改所有者
chown user file.txt           # 修改所有者
chown user:group file.txt     # 修改所有者和所属组
chgrp group file.txt          # 仅修改所属组
Part 4.3

权限计算方法

权限数值通过相加计算：r=4, w=2, x=1，将需要的权限数值相加即可。

权限组合 计算方式 数值 含义
rwx 4+2+1 7 读+写+执行
r-x 4+0+1 5 读+执行
rw- 4+2+0 6 读+写
r-- 4+0+0 4 只读
常用权限设置

- 755：目录或可执行程序（所有者全权限，其他人可读可执行）

- 644：普通文件（所有者可读写，其他人只读）

- 700：私有目录（仅所有者可访问）

- 600：敏感文件（仅所有者可读写）
