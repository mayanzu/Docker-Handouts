Part 1.1

什么是Linux？

核心定义
Linux是一种自由和开放源码的类UNIX操作系统内核，由林纳斯·托瓦兹于1991年首次发布。它是一个多用户、多任务的操作系统。

Linux操作系统广泛应用于服务器、超级计算机、嵌入式设备等领域。在容器技术领域，Linux更是不可或缺的基础平台，Docker等容器技术正是基于Linux内核特性实现的。

Linux特点

1. 开源免费：源代码公开，可自由使用和修改

2. 稳定可靠：可连续运行数年无需重启

3. 安全高效：权限机制完善，资源占用低

4. 可定制性强：可根据需求定制系统功能

Part 1.2

Linux发行版

Linux发行版是基于Linux内核的完整操作系统，包含内核、系统库、工具软件等。常见的发行版如下：

发行版 特点 适用场景
Ubuntu 用户友好，社区活跃 桌面、开发、服务器
CentOS 稳定，企业级支持 服务器、生产环境
Debian 稳定，软件包丰富 服务器、开发
Fedora 新技术前沿 开发、测试
提示

本课程推荐使用 Ubuntu 20.04/22.04 或 CentOS 7/8 作为学习环境，这也是Docker官方推荐的基础镜像。

Linux与Windows的区别

理解Linux与Windows的区别有助于更好地掌握Linux系统特性和容器技术基础。

对比项 Linux Windows
内核类型 单内核（Monolithic） 混合内核（Hybrid）
文件系统 EXT4、XFS、Btrfs NTFS、FAT32
路径分隔符 /（正斜杠） \（反斜杠）
文件名大小写 区分大小写 不区分大小写
用户界面 命令行为主，GUI可选 GUI为主
软件安装 包管理器（apt/yum） 安装包（.exe/.msi）
权限模型 rwx权限位 + 用户组 ACL访问控制列表
容器技术为什么选择Linux？

1. 内核特性：Linux内核原生支持Namespaces、CGroups等容器核心技术

2. 轻量级：Linux容器共享宿主机内核，启动快、资源占用少

3. 开源生态：丰富的开源工具和社区支持

4. 服务器主导：绝大多数服务器运行Linux系统

Part 2.5

文本处理三剑客

Linux中有三个强大的文本处理工具，被称为文本处理三剑客：grep（搜索）、sed（编辑）、awk（分析）。

sed命令 - 流编辑器 Bash 
# sed基本用法
sed 's/old/new/' file.txt           # 替换每行第一个匹配
sed 's/old/new/g' file.txt          # 替换所有匹配
sed -i 's/old/new/' file.txt       # 直接修改文件

# 删除行
sed '/pattern/d' file.txt           # 删除匹配的行
sed '1d' file.txt                   # 删除第一行
sed '1,5d' file.txt                # 删除1-5行

# 显示特定行
sed -n '1p' file.txt               # 显示第一行
sed -n '1,5p' file.txt             # 显示1-5行
sed -n '/pattern/p' file.txt        # 显示匹配的行

# 常用示例
sed 's/localhost/127.0.0.1/' hosts.txt
sed -n '/^#/!p' file.txt           # 显示非注释行
awk命令 - 文本分析 Bash 
# awk基本用法
awk '{print}' file.txt              # 打印所有行
awk '{print $1}' file.txt          # 打印第一列
awk '{print $1, $3}' file.txt      # 打印第一和第三列

# 字段分隔符
awk -F':' '{print $1}' /etc/passwd # 以冒号为分隔符
awk -F':' '{print $1, $NF}' /etc/passwd  # 用户名和最后一列

# 条件过滤
awk '/pattern/' file.txt            # 打印匹配的行
awk '$1 > 100' file.txt            # 打印第一列大于100的行
awk 'NR==1' file.txt               # 打印第一行

# 内置变量
# NR: 行号, NF: 字段数, $0: 整行, $1-$n: 各字段

# 常用示例
awk -F':' '{print "用户:" $1 ", shell:" $7}' /etc/passwd
awk '{sum+=$1} END {print sum}' numbers.txt  # 求和
三剑客对比

- grep：文本搜索，适合查找匹配行

- sed：流编辑器，适合批量替换、删除

- awk：文本分析工具，适合数据处理和报表生成

Part 3

文件系统

理解Linux目录结构

Part 3.1

Linux目录结构

Linux采用树形目录结构，所有文件和目录都从根目录 / 开始。了解标准目录结构对于系统管理至关重要。

目录 说明 重要程度
/ 根目录，所有文件的起点 核心
/home 用户主目录 核心
/etc 系统配置文件 核心
/var 可变数据（日志、缓存等） 重要
/usr 用户程序和数据 重要
/tmp 临时文件 一般
/root root用户主目录 重要
Part 3.2

路径概念

绝对路径
从根目录 / 开始的完整路径，例如：/home/user/documents/file.txt
相对路径
相对于当前工作目录的路径，例如：./documents/file.txt 或 ../config/app.conf
特殊符号

- . 表示当前目录

- .. 表示上级目录

- ~ 表示当前用户的主目录

- - 表示上一个工作目录

最佳实践

在脚本中使用绝对路径可以避免因工作目录变化导致的错误。

Part 4

权限管理

保障系统安全的核心机制

Part 4.1

权限概念

Linux采用多用户权限机制，每个文件和目录都有明确的权限设置，控制不同用户对资源的访问。

1读权限 r

可以查看文件内容，列出目录内容。数值为 4

2写权限 w

可以修改文件内容，在目录中创建/删除文件。数值为 2

3执行权限 x

可以执行文件（脚本、程序），进入目录。数值为 1

权限分组

- 所有者 Owner：文件的创建者

- 所属组 Group：文件所属的用户组

- 其他用户 Others：其他所有用户

Part 4.2

用户与用户组详解

所有者 Owner
文件的创建者，拥有对该文件的最高控制权。所有者可以修改文件的权限设置，也可以将文件转让给其他用户。
所有者相关操作 Bash 
# 查看文件所有者
$ ls -l file.txt
-rw-r--r-- 1 alice developers 1024 Jan 15 10:00 file.txt
#            ^^^^^ 所有者是alice

# 查看当前用户
$ whoami
alice

# 查看用户信息
$ id alice
uid=1000(alice) gid=1000(alice) groups=1000(alice),27(sudo),100(users)

# 修改文件所有者（需要root权限）
$ sudo chown bob file.txt
$ ls -l file.txt
-rw-r--r-- 1 bob developers 1024 Jan 15 10:00 file.txt

# 同时修改所有者和所属组
$ sudo chown bob:admin file.txt
所属组 Group
文件所属的用户组。组内的所有成员都拥有该文件的组权限。用户组是Linux实现权限共享的重要机制。
用户组相关操作 Bash 
# 查看所有用户组
$ cat /etc/group
root:x:0:
developers:x:1001:alice,bob,charlie
admin:x:1002:

# 查看当前用户所属的组
$ groups
alice sudo developers

# 查看指定用户所属的组
$ groups bob
bob : bob admin developers

# 创建新用户组（需要root权限）
$ sudo groupadd project-team

# 将用户添加到组
$ sudo usermod -aG project-team alice

# 修改文件所属组
$ sudo chgrp developers file.txt

# 查看组成员
$ getent group developers
developers:x:1001:alice,bob,charlie
Part 4.3

三种权限身份的区别

理解所有者、所属组和其他用户的区别是掌握Linux权限管理的关键。下面通过具体场景说明三者的关系：

身份 定义 权限范围 典型场景
所有者 u 文件的创建者 拥有最高权限，可修改权限设置 个人文件、配置文件
所属组 g 文件关联的用户组 组内成员共享组权限 团队协作、项目文件
其他用户 o 非所有者、非组成员 权限最受限 公开文件、系统文件
权限身份示例 Bash 
# 创建项目文件
$ touch project.txt
$ ls -l project.txt
-rw-r--r-- 1 alice alice 0 Jan 15 10:00 project.txt

# 将文件所属组改为开发组
$ sudo chgrp developers project.txt
$ ls -l project.txt
-rw-r--r-- 1 alice developers 0 Jan 15 10:00 project.txt
#            ^^^^^ 所有者    ^^^^^^^^^ 所属组

# 设置权限：所有者读写，组成员读写，其他人只读
$ chmod 664 project.txt
$ ls -l project.txt
-rw-rw-r-- 1 alice developers 0 Jan 15 10:00 project.txt
#  || || ||
#  || || |+-- 其他用户：只读(r--)
#  || || +--- 其他用户：只读(r--)
#  || |+----- 所属组：读写(rw-)
#  || +------ 所属组：读写(rw-)
#  |+-------- 所有者：读写(rw-)
#  +--------- 所有者：读写(rw-)

# 权限验证
# alice（所有者）：可以读写
# bob（developers组成员）：可以读写
# charlie（其他用户）：只能读
权限继承规则

- 用户首先匹配所有者身份，匹配成功则使用所有者权限

- 若不是所有者，检查是否在所属组中，在组内则使用组权限

- 若既不是所有者也不在组内，则使用其他用户权限

Part 4.4

权限查看与修改

权限操作示例 Bash 
# 查看权限
ls -l file.txt
# 输出示例：-rwxr-xr-- 1 user group 1024 Jan 1 10:00 file.txt
# 解释：-文件类型 rwx所有者权限 r-x组权限 r--其他用户权限

# 数字方式修改权限
chmod 755 script.sh    # rwxr-xr-x
chmod 644 file.txt     # rw-r--r--
chmod 700 private/     # rwx------

# 符号方式修改权限
chmod u+x script.sh    # 给所有者添加执行权限
chmod g-w file.txt     # 移除组的写权限
chmod o=r file.txt     # 设置其他用户只读
chmod a+x script.sh    # 所有人添加执行权限

# 修改所有者
chown user file.txt           # 修改所有者
chown user:group file.txt     # 修改所有者和所属组
chgrp group file.txt          # 仅修改所属组
Part 4.5

权限计算方法

权限数值通过相加计算：r=4, w=2, x=1，将需要的权限数值相加即可。

权限组合 计算方式 数值 含义
rwx 4+2+1 7 读+写+执行
r-x 4+0+1 5 读+执行
rw- 4+2+0 6 读+写
r-- 4+0+0 4 只读
常用权限设置

- 755：目录或可执行程序（所有者全权限，其他人可读可执行）

- 644：普通文件（所有者可读写，其他人只读）

- 700：私有目录（仅所有者可访问）

- 600：敏感文件（仅所有者可读写）

Part 4.6

特殊权限

除了基本的rwx权限外，Linux还支持三种特殊权限，用于实现更精细的权限控制。

特殊权限 符号 数值 作用
SUID s（所有者执行位） 4 执行时获得文件所有者权限
SGID s（组执行位） 2 执行时获得文件所属组权限
Sticky Bit t（其他执行位） 1 目录中只能删除自己的文件
特殊权限示例 Bash 
# SUID示例：passwd命令
$ ls -l /usr/bin/passwd
-rwsr-xr-x 1 root root 59640 Mar  3  2022 /usr/bin/passwd
#  ^ SUID位，普通用户执行时获得root权限

# 设置SUID
$ chmod u+s program
$ chmod 4755 program

# SGID示例：共享目录
$ mkdir /shared
$ chmod 2775 /shared
# 新建文件自动继承目录的组

# Sticky Bit示例：/tmp目录
$ ls -ld /tmp
drwxrwxrwt 15 root root 4096 Jan 15 10:00 /tmp
#             ^ Sticky Bit，用户只能删除自己的文件

# 设置Sticky Bit
$ chmod +t /shared
$ chmod 1777 /shared

# 查看特殊权限
$ stat file.txt
  File: file.txt
  Access: (0755/-rwxr-xr-x)  # 0755中的0表示无特殊权限
  Access: (4755/-rwsr-xr-x)  # 4755表示有SUID
安全警告

- SUID程序存在安全风险，谨慎设置

- 定期检查系统中的SUID程序：find / -perm -4000 2>/dev/null

- Sticky Bit常用于临时目录，防止用户互相删除文件
