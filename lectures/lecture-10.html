Part 1

Docker Compose概述

理解多容器编排的需求

1.1

什么是Docker Compose？

Docker Compose定义
Docker Compose是一个用于定义和运行多容器Docker应用的工具。通过一个YAML文件，用户可以定义服务、网络、数据卷等资源，然后通过一条命令启动所有服务。

想象一个典型的Web应用：前端Nginx、后端Python/Node.js、数据库MySQL、缓存Redis。如果分别启动4个容器，需要4条docker run命令，而且还要处理它们之间的依赖关系。Compose就是来解决这个问题的。

1一键启动

通过一条命令启动所有服务，无需手动操作

2环境隔离

每个项目有独立的环境，互不干扰

3配置复用

配置文件版本化，方便团队协作

4资源管理

统一管理网络、数据卷等资源

1.2

Compose安装与基本用法

安装说明
Docker Desktop for Windows/Mac已内置Compose。如果使用Linux或Docker Engine，需要单独安装Compose。
验证安装 Bash 
# 检查Docker Compose版本
$ docker compose version

# 预期输出：
# Docker Compose version v2.20.0

# 旧版本命令（docker-compose）也可用
$ docker-compose version
基本命令 Bash 
# 启动所有服务（后台运行）
$ docker compose up -d

# 启动并实时查看日志
$ docker compose up

# 停止所有服务
$ docker compose down

# 查看服务状态
$ docker compose ps

# 查看日志
$ docker compose logs [service-name]

# 进入容器调试
$ docker compose exec [service-name] bash
注意

新版本使用docker compose（空格），旧版本使用docker-compose（连字符）。两者功能相同。

Part 2

Compose文件语法

掌握docker-compose.yml配置

2.1

docker-compose.yml基础结构

Compose文件格式
docker-compose.yml是一个YAML格式的配置文件，用于定义服务、服务网络、数据卷等资源。文件名固定为docker-compose.yml或docker-compose.yaml。

YAML是一种人类可读的数据序列化格式，结构通过缩进来表示。Compose文件定义了"服务"（运行的容器）、"网络"（容器间通信）、"数据卷"（持久化存储）。

docker-compose.yml基本结构 YAML 
# 版本声明（可选，但建议指定）
version: "3.8"

# 服务定义
services:
  # 服务名称（自定义）
  web:
    image: nginx:alpine
    ports:
      - "8080:80"
    networks:
      - frontend

  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: secret
    volumes:
      - db-data:/var/lib/mysql
    networks:
      - backend

# 网络定义
networks:
  frontend:
  backend:

# 数据卷定义
volumes:
  db-data:
核心概念

- services：定义要运行的容器，每个服务对应一个容器

- networks：定义容器间通信的网络

- volumes：定义持久化存储

2.2

服务定义详解

服务定义原理
服务（service）定义了一个容器运行时配置。当执行docker compose up时，Compose会为每个服务启动一个容器。服务名称就是容器名称的前缀。

服务定义中包含：使用哪个镜像、暴露哪些端口、挂载哪些数据卷、设置哪些环境变量等。这些配置与docker run命令的参数一一对应。

服务定义示例 YAML 
services:
  web:
    # 镜像（可以使用已有的镜像）
    image: nginx:alpine

    # 容器名称（可选）
    container_name: my-web

    # 端口映射
    ports:
      - "8080:80"
      - "8443:443"

    # 环境变量
    environment:
      - NODE_ENV=production
      - DB_HOST=db

    # 数据卷挂载
    volumes:
      - ./html:/usr/share/nginx/html
      - nginx-logs:/var/log/nginx

    # 依赖服务（启动顺序）
    depends_on:
      - api
      - db

    # 网络连接
    networks:
      - frontend
      - backend

    # 重启策略
    restart: unless-stopped

  api:
    image: python:3.11-slim
    command: python app.py
    working_dir: /app
    volumes:
      - ./app:/app
    depends_on:
      - db
      - redis
    networks:
      - backend
2.3

构建镜像与Dockerfile

build配置原理
除了使用已有的镜像，Compose还可以从Dockerfile构建镜像。当服务需要定制化配置时，可以指定build上下文，Compose会自动构建镜像。

使用build时，Compose会先执行docker build，然后用构建好的镜像启动容器。这对于需要根据代码变化重新构建的场景非常有用。

使用Dockerfile构建 YAML 
services:
  web:
    # 指定构建上下文（包含Dockerfile的目录）
    build:
      context: ./webapp
      dockerfile: Dockerfile

    # 也可以指定Dockerfile文件名
    build:
      context: .
      dockerfile: Dockerfile.web

    # 构建参数（对应ARG）
    build:
      context: .
      args:
        VERSION: 1.0
        NODE_ENV: production

    # 镜像名称（构建后打标签）
    image: my-webapp:1.0
完整的Web应用示例 YAML 
services:
  web:
    build: ./web
    ports:
      - "8080:80"
    depends_on:
      - api
    networks:
      - app-net

  api:
    build: ./api
    environment:
      - DB_HOST=db
      - REDIS_HOST=redis
    depends_on:
      - db
      - redis
    networks:
      - app-net

  db:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: secret
      MYSQL_DATABASE: myapp
    volumes:
      - db-data:/var/lib/mysql
    networks:
      - app-net

  redis:
    image: redis:alpine
    networks:
      - app-net

networks:
  app-net:

volumes:
  db-data:

3.1

LEMP栈部署实践

LEMP栈是指Linux + Nginx + MySQL + PHP的Web运行环境。使用Compose可以一键部署完整的LEMP环境。

LEMP栈 docker-compose.yml YAML 
version: "3.8"

services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
      - ./html:/var/www/html
    depends_on:
      - php
    networks:
      - app-net

  php:
    image: php:8-fpm-alpine
    volumes:
      - ./html:/var/www/html
    networks:
      - app-net

  mysql:
    image: mysql:8.0
    environment:
      MYSQL_ROOT_PASSWORD: rootsecret
      MYSQL_DATABASE: myapp
      MYSQL_USER: appuser
      MYSQL_PASSWORD: appsecret
    volumes:
      - mysql-data:/var/lib/mysql
    networks:
      - app-net

  phpmyadmin:
    image: phpmyadmin:latest
    ports:
      - "8080:80"
    environment:
      PMA_HOST: mysql
    depends_on:
      - mysql
    networks:
      - app-net

networks:
  app-net:

volumes:
  mysql-data:

Part 4

服务依赖与扩展

高级编排功能

4.1

服务依赖与启动顺序

depends_on原理
depends_on用于声明服务间的依赖关系。Compose会确保被依赖的服务先启动，后依赖的服务后启动。但它不等待依赖服务"就绪"，只等待其"启动"。

depends_on解决的问题是启动顺序，但不会等待数据库完全就绪（接受连接）才启动应用。对于需要等待服务真正就绪的场景，需要配合健康检查（healthcheck）使用。

服务依赖示例 YAML 
services:
  web:
    depends_on:
      - api
      - db
    # api和db会先于web启动

  api:
    depends_on:
      - db
      - redis
    # redis和db先于api启动

  db:
    image: mysql:8.0
    # db最先启动

  redis:
    image: redis:alpine
    # redis第二启动
带健康检查的依赖 YAML 
services:
  db:
    image: mysql:8.0
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 5s
      timeout: 3s
      retries: 3

  web:
    depends_on:
      db:
        condition: service_healthy
    # 只有当db健康检查通过后，web才会启动
4.2

服务扩展与副本

服务扩展原理
Compose支持水平扩展，即为一个服务启动多个容器副本（replicas）。这可以实现负载均衡和高可用。

当扩展服务时，Compose会启动多个容器，它们共享同一个网络和数据卷（如果是挂载的宿主机目录，则共享同一份数据）。副本数由--scale参数指定。

服务扩展命令 Bash 
# 扩展web服务到3个副本
$ docker compose up -d --scale web=3

# 注意：如果多个容器绑定同一端口，需要使用ports配置中的随机端口
# 或者使用nginx作为负载均衡器

# 查看扩展后的服务状态
$ docker compose ps
负载均衡配置 YAML 
services:
  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro

  web:
    image: myweb:latest
    # 扩展到3个副本
    deploy:
      replicas: 3
    # 或者使用命令行：--scale web=3

# nginx.conf中配置负载均衡
# upstream backend {
#     server web:5000;  # Compose会自动解析到所有副本
# }
4.3

网络与数据卷管理

网络隔离原理
Compose会自动为项目创建一个默认网络，所有服务通过服务名相互访问。通过自定义网络，可以实现服务间的隔离。

在同一网络中的容器可以通过容器名互相访问（DNS解析）。不同网络的容器不能直接通信，必须通过网关或端口映射。

自定义网络配置 YAML 
services:
  frontend:
    networks:
      - web-net
    # frontend只能访问web-net中的服务

  backend:
    networks:
      - web-net
      - db-net
    # backend可以同时访问两个网络

  database:
    networks:
      - db-net
    # database只能被db-net中的服务访问

networks:
  web-net:
    driver: bridge
  db-net:
    driver: bridge
    internal: true  # 完全隔离，不允许外部访问
数据卷配置 YAML 
services:
  db:
    image: mysql:8.0
    volumes:
      # 命名数据卷（推荐）
      - mysql-data:/var/lib/mysql
      # 宿主机目录挂载
      - ./data:/var/lib/mysql
      # 只读挂载
      - ./config:/etc/mysql/conf.d:ro

volumes:
  mysql-data:
    # 可以指定驱动和选项
    driver: local
