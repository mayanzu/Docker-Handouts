Part 1.1

Docker网络组成

Docker网络
Docker网络是容器间通信和容器与外部通信的基础，通过Docker网络驱动实现各种网络功能。

Docker使用Linux网络命名空间实现容器网络隔离，通过veth设备对连接容器和网络驱动。

Docker网络架构 架构图

|                                                        |
|  +------------------------------------------------+   |
|  |                 docker0 网桥                     |   |
|  |              IP: 172.17.0.1/16                 |   |
|  +-----------------------+------------------------+   |
|                          |                              |
|           +--------------+--------------+              |
|           |              |              |              |
|     +-----v----+   +-----v----+   +-----v----+         |
|     | veth xxx |   | veth yyy |   | veth zzz |         |
|     +-----+----+   +-----+----+   +-----+----+         |
|           |              |              |              |
|     +-----+----+   +-----+----+   +-----+----+         |
|     | Container|   | Container|   | Container|         |
|     | eth0    |   | eth0     |   | eth0     |         |

Part 1.2

Docker网络驱动

驱动类型 说明 适用场景
bridge 默认模式，容器连接到虚拟网桥 独立应用容器
host 容器直接使用宿主机网络 高性能场景
overlay 跨主机容器通信（Swarm） 集群环境

none 禁用容器网络 离线计算

Part 2

网络模式详解

各种网络模式配置和使用

Part 2.1

Bridge网络模式

Bridge网络
Bridge是Docker的默认网络模式。当容器启动时未指定网络模式，Docker会创建虚拟网桥（docker0），容器通过该网桥与外部通信。
Bridge网络示例 Bash 
# 启动Ubuntu容器（默认使用bridge网络）
$ docker run -d --name ubuntu-bridge ubuntu:22.04 sleep infinity

# 查看容器IP地址
$ docker inspect --format '{{.NetworkSettings.Networks.bridge.IPAddress}}' ubuntu-bridge
172.17.0.2

# 从容器内测试网络连接
$ docker exec ubuntu-bridge ping -c 3 baidu.com
Bridge网络特点 说明 
# 特点：
# - 每个容器有独立的IP地址
# - 容器可以访问外部网络（通过NAT）
# - 外部访问容器需要端口映射
# - 同一bridge网络的容器可通过IP通信
# - 默认bridge网络不支持DNS解析（容器名不通）
Bridge网络限制

默认bridge网络不支持通过容器名通信，需要使用自定义bridge网络才能实现DNS解析

Part 2.2

Host网络模式

Host网络
Host模式移除容器与宿主机之间的网络隔离，容器直接使用宿主机的网络命名空间。
Host网络示例 Bash 
# 启动容器使用host网络
$ docker run -d --name nginx-host --network host nginx:alpine

# 验证网络（容器与宿主机共享网络）
$ docker exec nginx-host ip addr

# 访问服务（直接使用宿主机端口，无需映射）
# Nginx默认监听80端口，在宿主机直接访问http://localhost即可
Host网络特点 说明 
# 优点：
# - 性能最佳（无NAT转换）
# - 网络配置简单

# 缺点：
# - 端口冲突风险（多容器不能绑定同一端口）
# - 网络隔离性差
# - 安全性降低
使用建议

适用于对网络性能要求高且端口不冲突的场景

Part 2.3

None网络模式

None网络
None模式完全禁用容器的网络功能。容器拥有独立的网络命名空间，但不配置任何网络接口（除loopback外）。
None网络示例 Bash 
# 启动无网络Ubuntu容器
$ docker run -d --name ubuntu-none --network none ubuntu:22.04 sleep infinity

# 查看容器网络
$ docker inspect --format '{{.NetworkSettings.Networks}}' ubuntu-none
# 输出：map[]（空网络）

# 在容器内查看网络接口
$ docker exec ubuntu-none ip addr
# 注意：没有eth0，只有loopback
None网络特点 说明 
# 适用场景：
# - 批处理任务
# - 离线计算
# - 安全隔离场景
# - 不需要网络的后台任务
Part 2.4

Container网络模式

Container网络
Container模式与其他容器共享网络命名空间。新容器不会创建自己的网络接口，而是与指定容器共享同一网络栈。
Container网络示例 Bash 
# 启动第一个容器（作为网络基础）
$ docker run -d --name ubuntu-base ubuntu:22.04 sleep infinity

# 查看基础容器的IP
$ docker inspect --format '{{.NetworkSettings.Networks.bridge.IPAddress}}' ubuntu-base
172.17.0.2

# 启动新容器，共享基础容器的网络
$ docker run -d --name ubuntu-share --network container:ubuntu-base ubuntu:22.04 sleep infinity

# 验证：两个容器IP相同
$ docker inspect --format '{{.NetworkSettings.Networks.bridge.IPAddress}}' ubuntu-share
172.17.0.2

# 在新容器内测试
$ docker exec ubuntu-share ping -c 3 ubuntu-base
# 可以通信（因为共享网络命名空间）
适用场景

Sidecar模式：多容器共享网络，如Sidecar代理

紧密耦合应用：如主应用和日志收集器共享网络

实现容器网络互联

Part 3.1

自定义Bridge网络

自定义Bridge网络
自定义Bridge网络与默认Bridge的区别是支持DNS解析，容器可以通过容器名相互访问。
创建和使用自定义网络 Bash 
# 创建自定义bridge网络
$ docker network create --driver bridge my-network

# 启动容器并连接到自定义网络
$ docker run -d \
  --name ubuntu-web \
  --network my-network \
  ubuntu:22.04 sleep infinity

# 参数说明：
# -d                          后台运行模式
# --name ubuntu-web           容器名称
# --network my-network        连接到my-network网络
# ubuntu:22.04                使用Ubuntu 22.04镜像
# sleep infinity              保持容器运行

$ docker run -d \
  --name ubuntu-db \
  --network my-network \
  ubuntu:22.04 sleep infinity

# 查看容器IP地址
$ docker inspect --format '{{.NetworkSettings.Networks.my-network.IPAddress}}' ubuntu-web
192.168.100.2

# 容器间通过名称通信（DNS解析）
$ docker exec ubuntu-web ping -c 3 ubuntu-db
# PING ubuntu-db (192.168.100.3) 56(84) bytes of data.
指定子网和网关 Bash 
# 创建时指定子网和网关
$ docker network create \
  --driver bridge \
  --subnet 192.168.100.0/24 \
  --gateway 192.168.100.1 \
  my-network

# 参数说明：
# --driver bridge      使用bridge驱动（默认）
# --subnet 192.168.100.0/24 \
#                     指定子网地址和掩码
# --gateway 192.168.100.1 \
#                     指定网关地址

  my-subnet

# 创建带IP地址的网络
$ docker network create \
  --driver bridge \
  --subnet 192.168.200.0/24 \
  --ip-range 192.168.200.0/24 \
  my-network
Part 3.2

端口映射

端口映射
端口映射是将宿主机的端口与容器的端口进行关联，使外部流量可以通过宿主机端口访问容器内服务。
端口映射语法与示例（WSL） Bash 
# 语法：-p <host-port>:<container-port>[:<protocol>]

# 基本端口映射（将宿主机的8080端口映射到容器的80端口）
$ docker run -d \
  --name nginx-web \
  -p 8080:80 \
  nginx
# 访问：http://localhost:8080 或 http://宿主机IP:8080

# 参数说明：
# -p 8080:80    将宿主机8080端口映射到容器80端口
#              外部访问宿主机IP:8080即可访问容器内的Nginx

# 映射多个端口（HTTP和HTTPS）
$ docker run -d \
  --name nginx-multi \
  -p 8080:80 \
  -p 8443:443 \
  nginx

# 指定协议（tcp/udp）
$ docker run -d \
  --name dns-server \
  -p 53:53/udp \
  redis:alpine
# /udp指定使用UDP协议（默认是TCP）

# 绑定到特定IP（仅本地访问）
$ docker run -d \
  --name nginx-local \
  -p 127.0.0.1:8080:80 \
  nginx
# 只能通过 localhost:8080 访问，外部无法访问

# 映射到所有IP（默认行为）
$ docker run -d \
  --name nginx-all \
  -p 0.0.0.0:8080:80 \
  nginx
# 可以通过任何网络接口访问
查看端口映射 Bash 
# 查看端口映射
$ docker port nginx-web
80/tcp -> 0.0.0.0:8080

# 查看所有映射端口
$ docker ps --format "table {{.Names}}\t{{.Ports}}"
NAMES        PORTS
nginx-web    0.0.0.0:8080->80/tcp
说明

端口映射工作流程：外部请求 → 宿主机IP:端口 → NAT转换 → 容器IP:端口

Part 3.3

容器多网络连接

docker network connect - 连接网络（WSL） Bash 
# 创建两个隔离的网络
$ docker network create frontend-network
$ docker network create backend-network

# 启动应用服务器，连接前端网络
$ docker run -d \
  --name app-server \
  --network frontend-network \
  ubuntu:22.04 sleep infinity

# 将运行中的容器连接到后端网络
$ docker network connect backend-network app-server

# 参数说明：
# docker network connect    连接容器到网络
# backend-network           要连接的网络名
# app-server               要连接的容器名

# 验证网络连接
$ docker inspect --format '{{keys .NetworkSettings.Networks}}' app-server
[backend-network frontend-network]

# 验证网络连接
$ docker inspect --format '{{keys .NetworkSettings.Networks}}' app-server
[backend-network frontend-network]
断开网络连接 Bash 
# 断开网络连接
$ docker network disconnect backend-network app-server

# 查看当前网络
$ docker inspect --format '{{keys .NetworkSettings.Networks}}' app-server
[frontend-network]
说明

一个容器可以同时连接多个网络，每个网络分配独立的IP地址

Part 4.2

网络创建与删除

docker network create - 创建网络（WSL） Bash 
# 创建bridge网络（使用默认配置）
$ docker network create my-network

# 参数说明：
# docker network create    创建新网络命令
# my-network              网络名称（自定义）

# 创建带配置的bridge网络
$ docker network create \
  --driver bridge \
  --subnet 192.168.100.0/24 \
  --gateway 192.168.100.1 \
  --attachable \
  my-network

# 参数详细说明：
# --driver bridge          网络驱动类型（bridge/host/overlay）
# --subnet 192.168.100.0/24 \
#                         指定子网地址和CIDR掩码（24位=256个IP）
# --gateway 192.168.100.1 \
#                         指定网关地址
# --attachable            允许容器手动连接到此网络
#                         （使用docker network connect）
# my-network              网络名称（在此示例中）
docker network rm/prune - 删除网络 Bash 
# 删除网络
$ docker network rm my-network

# 清理未使用的网络
$ docker network prune

# 强制删除（如果网络在使用中）
$ docker network rm -f my-network
Part 4.3

网络故障排查

常用排查命令 Bash 
# 1. 查看容器网络配置
$ docker inspect --format '{{json .NetworkSettings}}' <container> | jq

# 2. 查看容器IP地址
$ docker inspect --format '{{.NetworkSettings.Networks.bridge.IPAddress}}' <container>

# 3. 查看端口映射
$ docker port <container>

# 4. 查看网络列表
$ docker network ls

# 5. 在容器内测试网络
$ docker exec <container> ping -c 3 8.8.8.8
$ docker exec <container> curl http://example.com
常见问题与解决 Bash 
# 问题1：容器无法访问外网
# 解决：检查Docker服务状态
$ systemctl status docker

# 问题2：无法通过端口访问容器
# 解决：检查端口映射和防火墙
$ docker port <container>
$ curl http://localhost:<port>

# 问题3：容器间无法通信
# 解决：确保容器在同一网络中
$ docker network inspect <network>

# 问题4：容器名无法解析
# 解决：使用自定义网络
$ docker network create my-network
$ docker run --network my-network ...
调试流程

1. 确认网络模式：bridge/host/custom

2. 检查IP配置：确认容器IP正确

3. 测试连通性：先ping IP，再ping域名

4. 检查端口映射：确认-p参数正确
