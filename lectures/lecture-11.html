Part 1

监控概述

理解容器监控的重要性

1.1

为什么需要监控？

容器监控定义
容器监控是对容器运行状态的持续观察和记录。通过监控，可以及时发现容器故障、性能瓶颈、资源不足等问题，实现主动运维。

想象一个大型商场：监控摄像头（指标采集）24小时记录情况，保安中心（监控系统）实时分析数据，报警系统（告警机制）在异常时通知相关人员。容器监控就是这套系统的数字化版本。

1故障预警

提前发现资源不足、进程异常等问题

2性能优化

了解资源使用情况，合理分配资源

3容量规划

根据历史数据规划扩容和缩容

4故障排查

通过历史数据还原故障现场

1.2

关键监控指标

容器关键指标
容器监控主要关注四类指标：CPU使用率、内存使用率、网络流量、磁盘I/O。这些指标反映了容器的运行状态和性能表现。
指标类型 说明 过高可能原因
CPU使用率 容器使用的CPU资源占比 死循环、计算密集型任务
内存使用率 容器使用的内存大小 内存泄漏、配置不当
网络流量 入站/出站带宽使用 异常流量、攻击
磁盘I/O 读写速度和IOPS 日志过多、频繁写入
docker stats查看实时指标 Bash 
# 查看所有容器实时资源使用
$ docker stats

# 预期输出：
# CONTAINER ID   NAME            CPU %   MEM USAGE / LIMIT     MEM %   NET I/O
# abc123...      nginx-web       0.32%   50.5MiB / 1.952GiB   2.53%   1.5MB / 2.3MB
# def456...      mysql-db        1.28%   320MiB / 1.952GiB    16.39%  5.2MB / 8.1MB

# 只查看特定容器
$ docker stats nginx-web mysql-db

# 显示详细信息（包含网络、磁盘）
$ docker stats --no-stream
Part 2

Portainer可视化

图形化容器管理工具

2.1

Portainer概述

Portainer定义
Portainer是一个轻量级的Docker图形化管理工具，通过Web界面管理容器、镜像、网络、数据卷等资源。适合中小规模Docker环境的管理。

Portainer就像Docker的"控制面板"，把原本需要命令行操作的功能变成了可视化界面。即使不熟悉Linux命令，也可以通过图形界面管理Docker环境。

1容器管理

创建、启动、停止、删除容器

2镜像管理

查看、拉取、删除镜像

3网络管理

查看、创建网络

4数据卷管理

查看、创建、删除数据卷

2.2

Portainer部署与访问

快速部署Portainer Bash 
# 1. 创建Portainer数据卷
$ docker volume create portainer_data

# 2. 启动Portainer容器
$ docker run -d \
  --name portainer \
  --hostname portainer \
  -p 9000:9000 \
  -p 8000:8000 \
  --restart=always \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v portainer_data:/data \
  portainer/portainer

# 参数说明：
# -p 9000:9000    Web管理界面端口
# -p 8000:8000    端口ainer Edge代理
# -v /var/run/docker.sock   Docker socket（核心配置）
# -v portainer_data         持久化数据
访问Portainer Bash 
# 访问Web界面
http://localhost:9000

# 首次访问：
# 1. 设置管理员账号密码
# 2. 选择"Local"管理本地Docker
# 3. 点击"Connect"连接
注意事项

- 如果使用Docker Desktop，Portainer已内置

- 生产环境建议配置HTTPS和认证

- 挂载/var/run/docker.sock后，Portainer有Docker完整权限

Part 3

Prometheus监控

时序数据库监控系统

3.1

Prometheus概述

Prometheus定义
Prometheus是一个开源的监控和告警系统，由SoundCloud开发，现已成为CNCF的毕业项目。它通过HTTP协议定期采集指标数据，存储为时序数据库。

Prometheus的工作方式类似于"定时问卷调查"：它定期向各个目标发送HTTP请求（/metrics端点），收集监控数据并存储。当指标超过阈值时，触发告警通知管理员。

1多维度数据模型

指标由指标名和标签集定义

2PromQL查询语言

强大的数据查询和聚合能力

3Pull模式

服务端主动拉取数据，无需客户端配置

4告警管理

支持基于PromQL的告警规则

3.2

cAdvisor监控容器

cAdvisor定义
cAdvisor（Container Advisor）是Google开发的容器监控工具，专门用于采集容器资源使用信息。它原生支持Docker，可以监控本地所有容器。
部署cAdvisor Bash 
# 启动cAdvisor
$ docker run -d \
  --name cadvisor \
  --hostname cadvisor \
  -p 8080:8080 \
  --restart=always \
  --mount type=bind,source=/,target=/rootfs:ro \
  --mount type=bind,source=/var/run,target=/var/run:ro \
  --mount type=bind,source=/sys,target=/sys:ro \
  --mount type=bind,source=/var/lib/docker/,target=/var/lib/docker:ro \
  google/cadvisor:latest

# 参数说明：
# -p 8080:8080       Web界面端口
# /rootfs            挂载根文件系统（读取容器信息）
# /var/run           Docker socket
# /sys               系统信息
# /var/lib/docker    Docker数据目录
访问cAdvisor Bash 
# Web界面
http://localhost:8080

# 查看容器指标（Prometheus格式）
http://localhost:8080/metrics

部署Prometheus

prometheus.yml配置文件 YAML 
global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  - job_name: 'cadvisor'
    static_configs:
      - targets: ['cadvisor:8080']
启动Prometheus Bash 
# 创建配置文件目录
$ mkdir -p /opt/prometheus

# 创建prometheus.yml
$ cat > /opt/prometheus/prometheus.yml << 'EOF'
global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: 'prometheus'
    static_configs:
      - targets: ['localhost:9090']

  - job_name: 'cadvisor'
    static_configs:
      - targets: ['cadvisor:8080']
EOF

# 启动Prometheus
$ docker run -d \
  --name prometheus \
  --hostname prometheus \
  -p 9090:9090 \
  --restart=always \
  -v /opt/prometheus/prometheus.yml:/etc/prometheus/prometheus.yml \
  prom/prometheus:latest
访问Prometheus Bash 
# Web界面
http://localhost:9090

# 查看监控目标状态
Status -> Targets

Grafana可视化

仪表板和可视化工具

4.1

Grafana概述

Grafana定义
Grafana是一个开源的度量分析和可视化平台。它可以从多种数据源（如Prometheus、InfluxDB）读取数据，提供丰富的图表类型和告警功能。

Grafana就像一个"仪表盘制造工厂"：它从监控系统（数据源）读取原始数据，然后根据需求定制各种仪表盘（图表），并可以设置警戒线（告警）。

1多数据源支持

Prometheus、InfluxDB、Elasticsearch等

2丰富的图表

折线图、仪表盘、热力图等

3告警功能

支持邮件、钉钉等多种告警方式

4模板变量

支持动态仪表盘

4.3

创建监控仪表盘

创建仪表盘步骤 操作步骤 
# 1. 创建新仪表盘
#    点击左侧"+" -> "Create" -> "Dashboard"

# 2. 添加面板
#    点击"Add new panel"

# 3. 配置查询（使用PromQL）
#    在"Metrics"输入框中输入Prometheus查询语句

# 示例查询：
# CPU使用率
# rate(container_cpu_usage_seconds_total{name!=""}[5m])

# 内存使用量
# container_memory_usage_bytes{name!=""}

# 网络流量
# rate(container_network_receive_bytes_total[5m])
常用PromQL查询 PromQL 
# 容器CPU使用率（百分比）
100 * rate(container_cpu_usage_seconds_total[5m]) / machine_cpu_cores

# 容器内存使用率
100 * container_memory_usage_bytes / container_spec_memory_limit_bytes

# 容器网络接收速率（字节/秒）
rate(container_network_receive_bytes_total[5m])

# 容器网络发送速率（字节/秒）
rate(container_network_transmit_bytes_total[5m])

# 容器文件系统读取速率
rate(container_fs_reads_bytes_total[5m])
导入现成仪表盘 操作步骤 
# Grafana支持导入现成的仪表盘
# 1. 点击左侧"+" -> "Import"
# 2. 输入Grafana Dashboard ID或上传JSON文件

# 推荐仪表盘：
# - Docker Monitoring (ID: 193)
# - Docker and System Monitoring (ID: 10619)
# - cAdvisor Prometheus (ID: 11949)

# 导入后会自动创建相关图表和面板

docker-compose.yml监控栈 YAML 
version: "3.8"

services:
  prometheus:
    image: prom/prometheus:latest
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
    networks:
      - monitor-net

  cadvisor:
    image: google/cadvisor:latest
    ports:
      - "8080:8080"
    volumes:
      - /:/rootfs:ro
      - /var/run:/var/run:ro
      - /sys:/sys:ro
      - /var/lib/docker/:/var/lib/docker:ro
    networks:
      - monitor-net

  grafana:
    image: grafana/grafana:latest
    ports:
      - "3000:3000"
    volumes:
      - grafana-data:/var/lib/grafana
    networks:
      - monitor-net

networks:
  monitor-net:

volumes:
  grafana-data:
