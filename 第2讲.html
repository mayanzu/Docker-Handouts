容器管理命令

创建、运行、停止容器

docker run [选项] 镜像名 [命令]

创建并运行容器（最核心的命令）

💡 命令核心功能：

从指定镜像创建一个新容器，并启动它。这是 Docker 最常用、最核心的命令，几乎所有容器操作都从它开始。

常用参数详解：
-d 或 --detach

后台运行容器，并返回容器ID

语法：docker run -d [其他参数] 镜像名
适用场景：需要容器长期运行，不占用终端窗口
Terminal
$ docker run -d nginx

a1b2c3d4e5f67890... (容器ID)

-it （-i + -t 的组合）

-i：保持标准输入打开；-t：分配一个伪终端

语法：docker run -it [其他参数] 镜像名 [命令]
适用场景：需要交互式操作容器，如进入命令行界面
Terminal
$ docker run -it ubuntu:20.04 bash

root@abcdef123456:/# _

-p 或 --publish

端口映射，将容器内的端口映射到宿主机

语法：docker run -p [宿主机端口]:[容器端口] [其他参数] 镜像名
适用场景：需要从外部访问容器内的服务（如Web服务器）
Terminal
$ docker run -d -p 8080:80 nginx

b3c4d5e6f7g890a1... (容器ID)

💡 访问：在浏览器输入 http://localhost:8080 即可访问容器内的 Nginx

-v 或 --volume

挂载宿主机目录或文件到容器内

语法：docker run -v [宿主机路径]:[容器路径] [其他参数] 镜像名
适用场景：需要持久化数据或共享文件
Terminal
$ docker run -d -v /host/html:/usr/share/nginx/html -p 8080:80 nginx

c4d5e6f7g890a1b2... (容器ID)

--name

为容器指定一个自定义名称

语法：docker run --name [容器名] [其他参数] 镜像名
适用场景：便于识别和管理容器
Terminal
$ docker run -d --name my-nginx -p 8080:80 nginx

d5e6f7g890a1b2c3... (容器ID)

--rm

容器停止后自动删除

语法：docker run --rm [其他参数] 镜像名
适用场景：临时测试或一次性任务，避免产生无用容器
Terminal
$ docker run --rm -it ubuntu:20.04 echo "Hello Docker"

Hello Docker

--network

指定容器使用的网络

语法：docker run --network [网络名] [其他参数] 镜像名
适用场景：需要多个容器之间通信
Terminal
$ docker run -d --name my-nginx --network my-net -p 8080:80 nginx

e6f7g890a1b2c3d4... (容器ID)

🎯 综合使用示例：
Terminal
# 后台运行 Nginx，指定名称、端口映射、挂载目录并使用自定义网络
$ docker run -d --name my-web -p 8080:80 -v /host/html:/usr/share/nginx/html --network my-net nginx

f7g890a1b2c3d4e5... (容器ID)

⚠️ 注意事项：
  • • 参数顺序很重要，一般建议先写选项，再写镜像名，最后写命令
  • • 多个短参数可以合并，如 -d -it 可以写成 -dit
  • • 端口映射时，宿主机端口不能被占用，否则会启动失败
  • • 挂载目录时，宿主机目录必须存在，否则会被当作文件挂载
  • • 使用 --rm 参数时，容器停止后会立即删除，无法恢复

docker ps [选项]

查看容器状态（默认显示运行中的容器）

💡 命令核心功能：

列出 Docker 容器的状态信息，是监控和管理容器的重要工具。通过不同的参数组合，可以查看不同状态的容器并获取详细信息。

📊 输出结果解析：
列名 含义 示例
CONTAINER ID 容器的唯一标识符 abc123def456
IMAGE 创建容器使用的镜像 nginx
COMMAND 容器启动时执行的命令 "nginx"
CREATED 容器创建的时间 2 minutes ago
STATUS 容器当前的状态 Up 2 minutes
PORTS 端口映射信息 0.0.0.0:8080->80/tcp
NAMES 容器的名称 my-nginx
常用参数详解：
-a 或 --all

显示所有容器（包括运行中、停止、暂停等状态）

语法：docker ps -a
适用场景：需要查看所有容器的完整状态，包括已停止的容器
Terminal
$ docker ps -a

CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES
abc123def456 nginx "nginx" 2 minutes ago Up 2 minutes 0.0.0.0:8080->80/tcp my-nginx
def789abc123 ubuntu "bash" 5 minutes ago Exited (0) 3 minutes ago my-ubuntu

-l 或 --latest

显示最近创建的容器

语法：docker ps -l
适用场景：需要查看刚刚创建的容器信息
Terminal
$ docker ps -l

CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES
abc123def456 nginx "nginx" 1 minute ago Up 1 minute 0.0.0.0:8080->80/tcp my-nginx

-n 或 --last

显示最近创建的 n 个容器

语法：docker ps -n [数量]
适用场景：需要查看最近创建的多个容器
Terminal
$ docker ps -n 3

CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES
abc123def456 nginx "nginx" 2 minutes ago Up 2 minutes 0.0.0.0:8080->80/tcp my-nginx
def789abc123 ubuntu "bash" 5 minutes ago Exited (0) 3 minutes ago my-ubuntu
ghi012def345 alpine "sh" 10 minutes ago Exited (0) 8 minutes ago my-alpine

-q 或 --quiet

只显示容器ID，不显示其他信息

语法：docker ps -q
适用场景：与其他命令结合使用，如批量操作容器
Terminal
$ docker ps -q

abc123def456
ghi789jkl012

💡 实用组合：docker stop $(docker ps -q) - 停止所有运行中的容器
--size 或 -s

显示容器的磁盘使用情况

语法：docker ps --size
适用场景：需要查看容器占用的磁盘空间
Terminal
$ docker ps --size

CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES SIZE
abc123def456 nginx "nginx" 2 minutes ago Up 2 minutes 0.0.0.0:8080->80/tcp my-nginx 2.04kB (virtual 133MB)

-f 或 --filter

根据条件过滤容器

语法：docker ps -f [过滤条件]
适用场景：需要根据特定条件筛选容器，如状态、名称、镜像等
Terminal
$ docker ps -f status=running

CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES
abc123def456 nginx "nginx" 2 minutes ago Up 2 minutes 0.0.0.0:8080->80/tcp my-nginx

💡 常用过滤条件：status=running/exited, name=my-*, image=nginx
🎯 综合使用示例：
Terminal
# 查看最近创建的3个容器，包括所有状态，并只显示容器ID
$ docker ps -n 3 -a -q

abc123def456
def789abc123
ghi012def345

# 查看所有运行中的nginx容器，并显示磁盘使用情况
$ docker ps -f image=nginx -f status=running --size

CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES SIZE
abc123def456 nginx "nginx" 2 minutes ago Up 2 minutes 0.0.0.0:8080->80/tcp my-nginx 2.04kB (virtual 133MB)

⚠️ 注意事项：
  • • 默认情况下，docker ps 只显示运行中的容器
  • • 使用 -q 参数可以快速获取容器ID，便于与其他命令结合使用
  • • --size 参数会增加命令执行时间，因为需要计算容器的磁盘使用情况
  • • 可以同时使用多个过滤条件，如 docker ps -f status=running -f name=my-*
  • • 容器名称是唯一的，而容器ID是全局唯一的

docker stop/start/restart [容器名/ID]

停止、启动、重启容器

💡 命令核心功能：

管理容器的运行状态，包括停止运行中的容器、启动已停止的容器以及重启容器。

docker stop

停止运行中的容器

语法：docker stop [容器名/ID]
工作原理：向容器发送 SIGTERM 信号，等待 10 秒后发送 SIGKILL 信号强制终止
Terminal
$ docker stop my-nginx

my-nginx

docker start

启动已停止的容器

语法：docker start [容器名/ID]
适用场景：需要重新启动之前停止的容器，保留原有配置
Terminal
$ docker start my-nginx

my-nginx

docker restart

重启容器（先停止，再启动）

语法：docker restart [容器名/ID]
适用场景：需要重新加载容器配置或重启服务
Terminal
$ docker restart my-nginx

my-nginx

🎯 批量操作示例：
Terminal
# 停止所有运行中的容器
$ docker stop $(docker ps -q)

my-nginx
my-web
my-db

# 启动所有已停止的容器
$ docker start $(docker ps -a -q -f status=exited)

my-nginx
my-web
my-db

⚠️ 注意事项：
  • • docker stop 会等待容器优雅停止，默认等待 10 秒
  • • 可以使用 docker kill 命令强制终止容器（不推荐，可能导致数据丢失）
  • • 重启容器会保留容器的配置和数据，但会重新执行启动命令
  • • 批量操作时要小心，确保只操作目标容器

docker rm [选项] [容器名/ID]

删除容器

💡 命令核心功能：

删除不再需要的容器，释放系统资源。默认只能删除已停止的容器，通过参数可以强制删除运行中的容器。

-f 或 --force

强制删除容器，包括运行中的容器

语法：docker rm -f [容器名/ID]
适用场景：需要立即删除容器，不管其状态如何
Terminal
$ docker rm -f my-nginx

my-nginx

-v 或 --volumes

删除容器的同时，删除与容器关联的匿名卷

语法：docker rm -v [容器名/ID]
适用场景：需要彻底清理容器及其相关资源
Terminal
$ docker rm -v my-nginx

my-nginx

🎯 批量删除示例：
Terminal
# 删除所有已停止的容器
$ docker rm $(docker ps -a -q -f status=exited)

my-nginx
my-web
my-db

# 强制删除所有容器（慎用！）
$ docker rm -f $(docker ps -a -q)

my-nginx
my-web
my-db

⚠️ 注意事项：
  • • 默认情况下，docker rm 只能删除已停止的容器
  • • 使用 -f 参数会强制终止并删除容器，可能导致数据丢失
  • • -v 参数只会删除匿名卷，不会删除命名卷
  • • 批量删除前，建议先使用 docker ps -a 确认要删除的容器

容器操作命令

进入容器、查看日志

docker exec [选项] [容器名/ID] [命令]

在运行中的容器内执行命令

💡 命令核心功能：

在已经运行的容器内部执行命令，是与容器交互的重要方式。可以执行单个命令，也可以进入容器进行交互式操作。

-i 或 --interactive

保持标准输入打开，允许与命令交互

语法：docker exec -i [容器名/ID] [命令]
适用场景：需要向命令输入数据的情况
-t 或 --tty

分配一个伪终端，提供交互式终端体验

语法：docker exec -t [容器名/ID] [命令]
适用场景：需要终端界面的情况
-it

-i 和 -t 的组合，提供完整的交互式终端

语法：docker exec -it [容器名/ID] [命令]
适用场景：需要进入容器进行交互式操作
Terminal
$ docker exec -it my-nginx bash

root@abcdef123456:/# _

🎯 使用示例：
Terminal
# 在容器内执行单个命令
$ docker exec my-nginx ls /usr/share/nginx/html

index.html 50x.html

# 进入容器的 bash 终端
$ docker exec -it my-nginx bash

root@abcdef123456:/#

# 在容器内执行命令并将结果输出到宿主机文件
$ docker exec my-nginx cat /usr/share/nginx/html/index.html > index.html
⚠️ 注意事项：
  • • docker exec 只能在运行中的容器内执行命令
  • • 使用 -it 参数组合可以获得最佳的交互式体验
  • • 执行命令的用户默认是容器的 root 用户，除非容器内有其他用户
  • • 可以通过 --user 参数指定执行命令的用户

docker logs [选项] [容器名/ID]

查看容器的日志输出

💡 命令核心功能：

查看容器的日志信息，是调试和监控容器运行状态的重要工具。可以查看历史日志，也可以实时跟踪日志输出。

-f 或 --follow

实时跟踪日志输出，类似于 tail -f 命令

语法：docker logs -f [容器名/ID]
适用场景：需要实时监控容器的运行状态和日志输出
Terminal
$ docker logs -f my-nginx

/docker-entrypoint.sh: Configuration complete
2023/10/10 10:00:01 [notice] 1#1: nginx/1.25.2
2023/10/10 10:00:01 [notice] 1#1: start worker processes
2023/10/10 10:05:00 [access] 172.17.0.1 - - [10/Oct/2023:10:05:00 +0000] "GET / HTTP/1.1" 200 612 "-" "Mozilla/5.0..."

-t 或 --timestamps

在日志中显示时间戳

语法：docker logs -t [容器名/ID]
适用场景：需要精确了解日志产生的时间
--tail

只显示最后 N 行日志

语法：docker logs --tail N [容器名/ID]
适用场景：只需要查看最新的日志信息
Terminal
$ docker logs --tail 5 my-nginx

2023/10/10 10:05:00 [access] 172.17.0.1 - - [10/Oct/2023:10:05:00 +0000] "GET / HTTP/1.1" 200 612 "-" "Mozilla/5.0..."
2023/10/10 10:05:01 [access] 172.17.0.1 - - [10/Oct/2023:10:05:01 +0000] "GET /favicon.ico HTTP/1.1" 404 555 "-" "Mozilla/5.0..."

🎯 使用示例：
Terminal
# 查看容器的所有日志
$ docker logs my-nginx
# 实时跟踪日志，并显示时间戳
$ docker logs -f -t my-nginx
# 查看容器最后10行日志
$ docker logs --tail 10 my-nginx
⚠️ 注意事项：
  • • docker logs 只能查看容器的标准输出和标准错误输出
  • • 使用 -f 参数时，可以按 Ctrl+C 停止跟踪
  • • 容器被删除后，其日志也会被删除，除非使用了日志驱动进行持久化
  • • 对于长时间运行的容器，日志可能会很大，建议使用 --tail 参数查看最新日志

🎯 Docker 命令练习任务

按照以下步骤练习，巩固所学命令：

任务 1：基础环境检查

  1. 运行 docker version 检查安装
  2. 运行 docker info 查看系统信息
  3. 运行 docker system df 查看磁盘使用

任务 2：镜像操作练习

  1. 运行 docker pull nginx 下载 Nginx 镜像
  2. 运行 docker images 查看下载的镜像
  3. 运行 docker pull ubuntu:20.04 下载 Ubuntu 镜像
  4. 再次运行 docker images 确认有两个镜像

任务 3：容器操作练习

  1. 运行 docker run -d --name test-web -p 8888:80 nginx
  2. 运行 docker ps 查看运行中的容器
  3. 在浏览器访问 http://localhost:8888 确认 Nginx 工作
  4. 运行 docker logs test-web 查看容器日志
  5. 运行 docker exec -it test-web bash 进入容器
  6. 在容器内运行 ls /usr/share/nginx/html 查看网页文件
  7. 输入 exit 退出容器
  8. 运行 docker stop test-web 停止容器
  9. 运行 docker ps -a 查看所有容器（包括停止的）
  10. 运行 docker start test-web 重新启动容器
  11. 运行 docker stop test-web 再次停止容器
  12. 运行 docker rm test-web 删除容器

任务 4：清理练习

  1. 运行 docker ps -a 确认没有运行中的容器
  2. 运行 docker images 查看所有镜像
  3. 运行 docker system prune 清理未使用的资源
  4. 再次运行 docker system df 查看清理后的磁盘使用

练习提示：

  • 每个命令执行后，观察输出是否符合预期
  • 练习过程中，可以随时使用 docker ps 和 docker images 检查状态
  • 如果遇到问题，可以使用 docker --help 查看帮助
  • 建议多练习几遍，直到熟练掌握这些命令

  • 谁访问都能看到

💡 关键理解：

Web 服务器就是一个永不停歇的文件管理员，专门负责把网页文件发送给来访问的浏览器。没有它，网站就是一堆"锁在抽屉里"的文件，别人根本看不到。

认识 Nginx：最流行的 Web 服务器

Nginx 是什么？

Nginx（读音：Engine-X）是一个非常强大的 Web 服务器软件，全世界超过 30% 的网站都在使用它。

包括淘宝、京东、B站等大型网站都用 Nginx！

🎯 Nginx 的核心工作流程：

1

监听端口（等待客人）

Nginx 会默认监听 80 端口，等待浏览器来访问

2

接收请求（客人来了）

浏览器访问网站时，Nginx 接收到请求

3

找到文件（找图纸）

Nginx 去文件夹找对应的 HTML 文件

4

返回网页（发出去）

把网页内容发送给浏览器，你就看到网页了

✨ 为什么我们用 Nginx？

  • • 简单：安装即用，不需要复杂配置
  • • 稳定：可以连续运行几个月不出问题
  • • 快速：处理请求的速度非常快
  • • 流行：大量教程和文档，遇到问题容易解决

5. 核心概念：端口映射 (Port Mapping)

这是竞赛中最重要的考点之一。一定要理解透彻！

为什么需要端口映射？

🔒 Docker 容器的隔离机制

还记得我们说过，Docker 容器就像一个密封的盒子吗？这个盒子有一个很重要的特性：完全隔离。

隔离意味着：

  • 容器里的文件，外面看不到
  • 容器里的端口，外面访问不了
  • 容器里跑的 Nginx，外面的浏览器根本连不上

🏗️ 建筑工地的比喻

没有端口映射：

🚧 工地围起来了（容器隔离）

📋 里面有图纸（网页文件）

🚪 但是大门锁着（没有对外开放）

❌ 外面的人进不去拿图纸

有端口映射：

🚧 工地还是围起来的（还是隔离）

📋 图纸还在里面（文件还在容器里）

🚪 但开了一扇小窗口（端口映射）

✅ 外面的人可以通过窗口拿图纸！

🎯 核心理解：

端口映射 = 在容器的墙上开个洞，让外面的人可以通过这个洞访问容器里的服务。 没有这个洞，容器里的 Nginx 跑得再好，外面也访问不到！

什么是端口映射？

🔌 端口映射的本质：建立连接通道

端口映射就是在你的电脑和容器之间建立一条专用通道，让数据可以在两者之间流通。

1

浏览器访问电脑的 8080 端口

用户在浏览器输入 localhost:8080

2

Docker 自动转发到容器的 80 端口

这是端口映射的魔法时刻

3

容器里的 Nginx 接收请求并返回网页

用户看到了网页内容

💡 记住这个比喻：

端口映射就像给密封箱子装了一根管道。外面的水（请求）可以通过管道流进去，里面的水（响应）也可以流出来。 没有这根管道，箱子就是完全封闭的！

📊 端口映射可视化图解

你的电脑 (宿主机)
公网IP / localhost
端口: 8080
(外面的人访问这里)
-p 8080:80 ← 这就是端口映射！ （建立通道）
Docker 容器
Nginx 默认监听
端口: 80
(Nginx 在这里工作)

🔍 详细解释： -p 8080:80 的意思是： "把我电脑的 8080 端口（左边）连接到容器的 80 端口（右边）。 当有人访问我电脑的 8080 端口时，Docker 会自动把请求转发给容器的 80 端口，Nginx 就能收到请求并返回网页了。"

常见疑问解答

❓ 为什么左边是 8080，右边是 80？

左边（8080）是你电脑的端口，可以随便改（8888、9000都行）。
右边（80）是容器里 Nginx 的端口，这个不能随便改，因为 Nginx 默认就监听 80 端口。

❓ 可以用其他端口吗？

可以！比如：
-p 9000:80 浏览器访问 localhost:9000
-p 3000:80 浏览器访问 localhost:3000
左边的端口号随你喜欢，右边的 80 一般不变。

❓ 不写 -p 参数会怎样？

容器能启动，但你访问不了！就像工地围起来了，但没开门窗，外面的人进不去。

现在开始实际操作

有了前面的理解，现在来执行带端口映射的命令：

Terminal

# 这次我们加上 -p 参数 (-d 表示后台运行)

$ docker run -d -p 8080:80 --name my-web nginx

f7a8b9c0d1e2... (出现长串ID说明成功)

📝 命令详解：

  • • docker run - 创建并启动容器
  • • -d - 后台运行
  • • -p 8080:80 - 端口映射！电脑的8080 → 容器的80
  • • --name my-web - 给容器起名叫 my-web
  • • nginx - 使用 nginx 镜像

🎉 见证奇迹的时刻

打开浏览器，访问：http://localhost:8080

立即点击访问

如果你看到 "Welcome to nginx!" 的页面，恭喜你，你搭建了第一个服务器！

6. 进阶操作：潜入容器改网页

默认的欢迎页面太丑了。我们进去把它改掉。

🗝️ 关键命令：docker exec

docker exec -it [容器名] bash

步骤 1：启动 Nginx 容器（带端口映射）

这一步我们要让 Nginx 在后台运行，并且打开 8080 端口

💡 提示： 这条命令有点长，但不用担心！我们一部分一部分来理解。

Terminal
$ docker run -d -p 8080:80 --name my-website nginx

a1b2c3d4e5f6789... (一长串ID)

命令详解（每个部分的作用）：

  • docker run - 创建并启动一个容器
  • -d - 在后台运行（daemon 模式），不占用终端
  • -p 8080:80 - 端口映射：把电脑的 8080 端口连接到容器的 80 端口
  • --name my-website - 给容器起个名字叫 "my-website"，方便后面操作
  • nginx - 使用 nginx 镜像

如何确认成功？

运行下面这个命令查看容器状态：

Terminal
$ docker ps

CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES
a1b2c3d4e5f6 nginx ... 5 sec ago Up 5 sec 0.0.0.0:8080->80/tcp my-website

✅ 看到 my-website 出现在列表里，STATUS 显示 "Up"，就说明成功了！

步骤 2：在浏览器里查看默认网页

现在 Nginx 已经在运行了，我们来看看它长什么样

🖱️ 操作步骤：

  1. 打开你的浏览器（Chrome、Edge、Firefox 都可以）
  2. 在地址栏输入：localhost:8080
  3. 按回车键

👀 你应该看到：

Welcome to nginx!

If you see this page, the nginx web server is successfully installed and working...

🤔 为什么是 localhost:8080？

  • localhost = 你自己的电脑（本地）
  • 8080 = 我们刚才用 -p 8080:80 打开的端口号
  • 合起来就是："访问我电脑的 8080 端口"

步骤 3：进入容器内部

现在我们要"穿越"进容器里面，去修改网页文件

🔑 关键理解：

Docker 容器就像一个虚拟的小电脑。docker exec 命令就像一扇传送门，让我们进去操作这个"小电脑"。

Terminal

# 执行这条命令进入容器

$ docker exec -it my-website bash

root@a1b2c3d4e5f6:/# _

# 👆 看到提示符变成 root@... 就说明成功进入容器了！

命令详解：

  • docker exec - 在运行中的容器里执行命令
  • -it - 进入交互模式（可以像正常使用终端一样操作）
  • my-website - 容器的名字（刚才我们起的）
  • bash - 启动一个命令行终端

重要提示

进入容器后，你的终端提示符会变化。这是正常的！说明你现在在容器"里面"了。

步骤 4：找到网页文件的位置

Nginx 把网页文件放在一个固定的文件夹里，我们需要先进入这个文件夹

📁 小知识：

Nginx 的网页文件默认放在 /usr/share/nginx/html 这个文件夹里。记住这个路径很重要！

Terminal (在容器内)

# 进入网页文件夹

root@...:/# cd /usr/share/nginx/html

# 看看文件夹里有什么文件

root@...:/usr/share/nginx/html# ls

50x.html index.html

# 👆 看到了！index.html 就是我们要修改的文件

命令说明：

  • cd - change directory（改变目录），就是"进入某个文件夹"
  • ls - list（列出），就是"看看这里有什么文件"
  • index.html - 网站的首页文件（浏览器访问网站时第一个加载的文件）

步骤 5：修改网页内容

现在是最关键的一步：把默认网页改成你自己的内容！

⚠️ 注意：

容器里没有安装文本编辑器（nano、vim 都没有），所以我们用 echo 命令直接写入内容。

Terminal (在容器内)

# 方法1: 简单版（一行代码）

// 先设置UTF-8编码，避免中文乱码

root@...:/usr/share/nginx/html# export LANG=UTF-8
root@...:/usr/share/nginx/html# echo "<h1>欢迎来到我的网站！</h1>" > index.html

# 方法2: 完整版（多行代码，更漂亮）

root@...:/usr/share/nginx/html# cat > index.html << 'EOF'
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>我的第一个网页</title>
    <style>
        body { font-family: Arial; text-align: center; padding: 50px; }
        h1 { color: #2563eb; }
    </style>
</head>
<body>
    <h1>🎉 欢迎来到 [你的名字] 的网站！</h1>
    <p>这是我用 Docker 创建的第一个网页</p>
    <p>日期：2025-01-20</p>
</body>
</html>
EOF

# 👆 记得修改 [你的名字] 为你自己的名字！

方法1 说明（简单）

echo "内容" > 文件 的意思是：把引号里的内容写入文件。

优点：简单快速
缺点：只有一行

方法2 说明（完整）

cat > 文件 << 'EOF' 可以输入多行内容，直到遇到 EOF 为止。

优点：可以写完整的 HTML
缺点：稍微复杂一点

确认修改成功

运行这个命令查看文件内容：

Terminal
root@...:/usr/share/nginx/html# cat index.html

<h1>欢迎来到我的网站！</h1>

✅ 看到你刚才写的内容，就说明修改成功了！

❌ 问题1：浏览器打不开 localhost:8080

可能原因：

  • 容器没有运行。运行 docker ps 检查
  • 端口被占用。尝试换一个端口，比如 -p 9090:80
  • Docker Desktop 没有启动。打开 Docker Desktop 程序

❌ 问题2：刷新后还是显示 "Welcome to nginx!"

解决方法：

  • 按 Ctrl + Shift + R 强制刷新（清除缓存）
  • 确认你修改的是 index.html 文件
  • 检查文件内容：在容器内运行 cat index.html

❌ 问题3：进不去容器（docker exec 报错）

解决方法：

  • 检查容器名字是否正确：docker ps 查看
  • 确保容器正在运行（STATUS 显示 "Up"）
  • 如果容器停止了，运行 docker start my-website 启动
