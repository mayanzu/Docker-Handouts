Part 1

容器技术产生背景

为什么我们需要容器技术？

Part 1.1

经典场景：开发人员的噩梦

痛点场景
"在我机器上是好的啊！"

这是开发人员最常说的一句话，也是运维人员最怕听到的一句话

在传统软件开发中，应用在开发环境运行正常，但部署到生产环境后却出现各种问题。 这种"环境不一致"的问题每年浪费企业大量时间和资源。

问题本质
软件运行依赖于特定的运行环境，包括操作系统版本、运行时版本、依赖库版本、配置文件等。当这些环境因素在开发和生产环境之间存在差异时，就会导致"在我机器上能跑"的问题。
Part 1.1

经典场景：开发人员的噩梦

某互联网公司的Web应用部署问题
配置项 开发环境 生产环境 问题
Python版本 3.11 3.8 版本不兼容
Web框架 Flask 3.0 Flask 2.0 API变更
数据库 PostgreSQL MySQL SQL语法差异
操作系统 macOS CentOS 路径/权限差异
问题根源

开发环境和生产环境的软件栈不一致是导致部署失败的主要原因。

这种不一致性包括：操作系统差异、运行时版本差异、依赖库版本差异、配置差异等。

影响：项目间相互干扰

名词解释：环境

环境指软件运行所需的完整条件，包括操作系统、运行时（如Python、Java）、依赖库、配置文件等。就像植物生长需要土壤、阳光、水分一样，软件运行也需要特定的"环境"。

环境配置的复杂性来源

1. 操作系统差异：不同OS有不同的文件系统、权限模型、网络配置

2. 版本依赖：项目A需要Python 3.8，项目B需要Python 3.11

3. 库冲突：不同项目依赖同一库的不同版本

4. 配置差异：开发/测试/生产环境的配置参数不同

Part 1.3

传统方案一：详细文档

方案描述

编写详细的环境配置文档，指导手动安装

优点
成本低，易于实施 灵活性高，可定制
缺点
文档容易过时 人工操作容易出错 不同人理解可能有偏差 无法保证一致性
名词解释：部署文档

记录如何安装和配置软件环境的说明文件。就像家电的使用说明书，告诉你如何操作。但文档的问题是静态的，而环境是动态变化的。

Part 1.3

传统方案一：详细文档

方案描述

将整个操作系统打包成虚拟机镜像，保证环境完全一致

优点
环境完全隔离 可以打包所有依赖 保证一致性
缺点
镜像体积巨大（5-20GB） 启动慢（分钟级） 资源消耗高 难以大规模部署
名词解释：虚拟机

虚拟机（Virtual Machine）是在一台物理电脑上模拟出的另一台完整电脑。它有自己的操作系统，就像在电脑里又装了一台电脑。每个虚拟机都需要完整的操作系统，这就是它"重"的原因。

Part 1.3

传统方案一：详细文档

思考

有没有一种技术，既能像虚拟机一样保证环境隔离，又能像普通进程一样轻量快速？

答案就是——容器技术！

虚拟机：太重、太慢
→
容器：轻量、快速
核心洞察

虚拟机之所以"重"，是因为每个虚拟机都包含完整的操作系统（1-2GB）。

如果我们能共享宿主机内核，只打包应用和依赖，就能实现轻量级隔离！

Part 2.1

Docker是什么？

Docker
Docker是一个开源的应用容器引擎，让开发者可以将应用程序及其依赖打包到一个可移植的容器中，然后发布到任何操作系统上运行。
名词解释：容器

容器（Container）是一种轻量级的虚拟化技术。可以把它想象成一个"盒子"，里面装着你的应用程序和它运行所需的一切。这个盒子可以在任何电脑上运行，里面的程序不会受到外部环境影响。

Docker的核心价值

一次构建，处处运行：在开发环境构建的镜像，可以在测试、生产环境无缝运行

环境一致性：消除"在我机器上能跑"的问题

快速部署：秒级启动，支持大规模部署

Docker vs 虚拟机

虚拟机架构

应用 + 依赖库
完整操作系统 (1-2GB)
Hypervisor
宿主机操作系统

容器架构

应用 + 依赖库
Docker Engine
宿主机操作系统
核心区别
容器共享宿主机内核，无需完整操作系统，因此更轻量、更快速！

特性 容器 虚拟机

选择建议

虚拟机适用场景：强隔离需求、多操作系统、高安全性要求

Part 2.4

Docker三大核心组件

01

镜像 Image

应用的"安装包"，包含运行所需的一切

包含内容：

  • 操作系统文件系统（精简版，如Ubuntu基础镜像仅72MB）
  • 应用程序代码
  • 运行时环境（Python、Node.js、Java等）
  • 依赖库和配置文件
类比理解

镜像就像软件安装包，或者面向对象编程中的"类"。它是静态的、不可变的模板。一个镜像可以创建多个容器实例。

Part 2.4

Docker三大核心组件

02

容器 Container

镜像的运行实例，隔离的运行环境

容器特性：

  • 独立的文件系统：容器内的文件与宿主机隔离
  • 独立的网络配置：可以有自己的IP地址
  • 独立的进程空间：容器内的进程对宿主机不可见
  • 生命周期管理：可以被启动、停止、删除
类比理解

容器就像运行中的软件，或者面向对象编程中的"对象实例"。一个镜像可以创建多个容器实例，就像一个类可以创建多个对象。

Part 2.4

Docker三大核心组件

03

仓库 Repository

存储和分发镜像的地方

仓库类型：

  • Docker Hub - 官方公共仓库，类似GitHub
  • 私有仓库 - 企业内部使用，如Harbor、Nexus
  • 镜像标签 - 区分不同版本，如nginx:1.21、nginx:latest
类比理解

仓库就像应用商店。GitHub存储代码，Docker Hub存储镜像。你可以从中下载别人做好的镜像，也可以上传自己的镜像分享给他人。

Step 4

启动Docker服务

systemctl start docker

start启动服务，enable设置开机自启

名词解释：systemctl

systemctl是Linux系统服务管理命令。start启动服务，enable设置开机自启，status查看服务状态。

Part 3.3

配置镜像加速

国内访问Docker Hub较慢，配置阿里云加速器可提升下载速度10倍以上。

配置加速器 
mkdir -p /etc/docker
tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://your-id.mirror.aliyuncs.com"
  ]
}
EOF
systemctl daemon-reload
systemctl restart docker
获取加速地址

阿里云控制台 → 容器镜像服务 → 镜像加速器
