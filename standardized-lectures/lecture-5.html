Part 1

镜像仓库概述

理解镜像的存储与分发机制

Part 1.1

什么是镜像仓库？

镜像仓库（Registry）
镜像仓库是存储和分发Docker镜像的服务。类似于代码仓库（如Git），但存储的是构建好的镜像文件。
📚 名词解释：Registry vs Repository

Registry：仓库服务器，如Docker Hub、阿里云镜像服务

Repository：具体的镜像仓库，如nginx、mysql

Tag：镜像标签，如latest、1.0、alpine

Part 1.2

镜像仓库分类

  • 公有仓库：Docker Hub、Quay.io，免费使用、公开可见
  • 私有仓库：自建Registry、Harbor，安全可控、内网访问
  • 云服务商：阿里云、腾讯云、AWS ECR，稳定可靠、CDN加速
Part 1.3

Docker Hub介绍

🌐 Docker Hub

Docker官方提供的公有镜像仓库，是全球最大的容器镜像社区。

  • 官方镜像：nginx、mysql、redis、python等
  • 免费账户：1个私有仓库，无限公有仓库
  • 自动构建：关联GitHub自动构建镜像
  • Webhooks：镜像更新时触发通知
登录Docker Hub 
docker login
# 输入用户名和密码
# 登录成功后可推送镜像
Part 2

私有仓库搭建

Docker Registry部署与配置

Part 2.1

为什么需要私有仓库？

  • 安全性：敏感代码不暴露到公网
  • 速度：内网传输更快
  • 成本：节省带宽费用
  • 合规：满足数据安全要求
Part 2.2

Docker Registry部署

Docker Registry
Docker官方提供的开源镜像仓库服务，轻量级、易部署，适合小型团队使用。
启动Registry容器 
docker run -d \
  --name registry \
  -p 5000:5000 \
  -v /data/registry:/var/lib/registry \
  --restart=always \
  registry:2
-p 5000:5000 映射端口
-v /data/registry 数据持久化目录
--restart=always 自动重启
Part 2.3

推送镜像到私有仓库

Step 1

标记镜像

# 格式：docker tag 原镜像 仓库地址/镜像名:标签
docker tag nginx:latest localhost:5000/mynginx:v1
Step 2

推送镜像

docker push localhost:5000/mynginx:v1
Step 3

拉取镜像

docker pull localhost:5000/mynginx:v1
Part 2.4

查看仓库中的镜像

查看镜像列表 
# 查看所有镜像仓库
curl http://localhost:5000/v2/_catalog

# 查看某镜像的所有标签
curl http://localhost:5000/v2/mynginx/tags/list
返回示例
{
  "repositories": ["mynginx", "myapp", "mysql"]
}
Part 2.5

配置HTTPS（生产环境）

⚠️ 安全警告

生产环境必须使用HTTPS，否则密码明文传输，存在安全风险。

生成自签名证书 
mkdir -p /data/registry/certs
openssl req -newkey rsa:4096 -nodes \
  -keyout /data/registry/certs/domain.key \
  -x509 -days 365 \
  -out /data/registry/certs/domain.crt
启动带HTTPS的Registry 
docker run -d \
  --name registry \
  -p 443:443 \
  -v /data/registry:/var/lib/registry \
  -v /data/registry/certs:/certs \
  -e REGISTRY_HTTP_ADDR=0.0.0.0:443 \
  -e REGISTRY_HTTP_TLS_CERTIFICATE=/certs/domain.crt \
  -e REGISTRY_HTTP_TLS_KEY=/certs/domain.key \
  registry:2
Part 3

阿里云镜像服务

企业级镜像托管方案

Part 3.1

阿里云容器镜像服务

☁️ 阿里云容器镜像服务ACR

阿里云提供的企业级容器镜像托管服务，支持镜像安全扫描、镜像签名、镜像加速等功能。

✓ 免费额度：个人版免费使用
✓ 国内加速：镜像加速器提升下载速度
✓ 安全扫描：自动检测镜像漏洞
✓ 跨区域同步：支持多地域镜像分发
Part 3.2

配置镜像加速器

💡 为什么需要加速器？

Docker Hub服务器在国外，国内访问速度慢。阿里云提供镜像加速服务，大幅提升下载速度。

配置Docker加速器 
# 编辑daemon.json
sudo vim /etc/docker/daemon.json

# 添加加速器配置
{
  "registry-mirrors": [
    "https://xxxxxx.mirror.aliyuncs.com"
  ]
}

# 重启Docker
sudo systemctl daemon-reload
sudo systemctl restart docker

Part 4

镜像安全最佳实践

保护镜像与仓库安全

Part 4.1

镜像安全风险

🦠

漏洞风险

基础镜像可能包含已知漏洞

🔑

密钥泄露

镜像中可能包含敏感信息

🎭

镜像篡改

传输过程可能被中间人攻击

👤

权限滥用

容器以root权限运行

Part 4.2

安全最佳实践

1

使用官方基础镜像

优先选择官方镜像，定期更新基础镜像版本

2

最小化镜像体积

使用alpine版本，减少攻击面

3

不存储敏感信息

密钥、密码通过环境变量或Secret注入

4

镜像漏洞扫描

使用Trivy、Clair等工具扫描镜像

Part 4.3

镜像漏洞扫描

Trivy
一款开源的容器镜像漏洞扫描工具，可检测操作系统包和应用程序依赖中的已知漏洞。
安装并使用Trivy 
# 安装Trivy
curl -sfL https://raw.githubusercontent.com/aquasecurity/trivy/main/contrib/install.sh | sh

# 扫描镜像
trivy image nginx:latest

# 扫描本地镜像
trivy image --input myimage.tar
