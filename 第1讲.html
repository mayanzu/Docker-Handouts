Docker 容器技术竞赛培训

第一讲：环境构建与基础入门

0. 计算机基础知识速览

什么是操作系统？

操作系统是管理计算机硬件和软件的程序。就像餐厅经理：协调厨房（CPU）、仓库（内存）、服务员（进程）。

  • Windows：图形化，易用，游戏/办公首选
  • Linux：命令行为主，服务器首选，开源免费
  • macOS：Unix 核心，设计师/开发者喜爱

为什么要学 Linux？

  • 全球 90% 以上的服务器运行 Linux
  • Docker 底层基于 Linux 内核技术
  • 竞赛环境几乎都是 Linux
  • 企业开发必备技能

1. 为什么是 Docker？

传统开发的痛点

❌ "在我电脑上能跑"

开发环境、测试环境、生产环境配置不一致，导致程序在自己电脑能跑，到服务器就崩溃。

❌ 环境配置地狱

装 Python、Node.js、MySQL、Redis……版本冲突，依赖混乱，重装系统要花半天重新配置。

Docker 解决方案

核心思想：把应用和环境一起打包，像"集装箱"一样标准化运输。

  • 一致性：开发、测试、生产环境完全一致
  • 隔离性：每个容器独立运行，互不干扰
  • 便携性：一次构建，到处运行
  • 高效性：秒级启动，资源占用少

生活类比

虚拟机 = 整套房子：每个虚拟机都有完整的操作系统，像建一整栋房子，占地大、启动慢。

Docker 容器 = 集装箱：共享主机操作系统内核，只打包应用和依赖，轻量快速。

2. 安装 WSL 2 (Windows用户必做)

什么是 WSL？

Windows Subsystem for Linux（WSL）让你在 Windows 里运行一个真正的 Linux 环境，无需双系统，也不需要笨重的虚拟机。简单来说，就是在 Windows 里开了一个 Linux 的窗口。

WSL 2 基于微软的 Hyper-V 虚拟化技术，性能接近原生 Linux，与 Windows 深度整合：文件互通、网络共享、图形应用支持。

什么是虚拟机？

虚拟机（Virtual Machine）通过虚拟化技术，在真实电脑里模拟出一台完整的"第二台电脑"，包含独立的 CPU、内存、硬盘、网卡等。常见例子：VMware、VirtualBox。

缺点：启动慢、占用资源多、和主系统隔离严重，需要手动设置共享。

Hyper-V 是什么？

Hyper-V 是微软官方的一级虚拟化引擎（Hypervisor），负责管理虚拟机如何共享 CPU 和内存。Windows 10/11 专业版自带 Hyper-V，WSL 2 就是基于它运行的。

重要：如果电脑 BIOS 没开启硬件虚拟化（Intel VT-x / AMD-V），Hyper-V 和 WSL 都无法启动。

WSL 2 如何运行 Ubuntu？

WSL 2 在 Hyper-V 之上启动一个轻量级的 Linux 内核，实质是一个高度优化的虚拟机。但它与 Windows 深度整合：

  • 文件系统互通（/mnt/c 访问 C 盘）
  • 网络自动共享
  • 图形应用支持（WSLg）
  • 近乎原生的 Linux 性能

为什么选择 Ubuntu？

  • 社区庞大：资料全，遇到问题容易查到答案
  • 易用友好：包管理器 apt 简单，对新手友好
  • 稳定可靠：LTS 版本长期支持，企业级可靠
  • 预装齐全：自带 Python、Git、build-essential
  • 行业标准：云平台、竞赛环境大量采用

安装步骤

1
启用 WSL

以管理员身份打开 PowerShell，运行：

wsl --install

这会自动安装 WSL 2 和 Ubuntu。

2
重启电脑

安装完成后必须重启。

3
首次启动 Ubuntu

在开始菜单找到 "Ubuntu"，首次启动会要求设置用户名和密码。

重要：密码输入时不显示任何字符，这是正常的！

4
验证安装
wsl --version
wsl --list --verbose

确认 WSL 版本为 2，Ubuntu 状态为 Running。

5
测试 Linux 命令

在 Ubuntu 终端运行：

pwd
ls -la
uname -a

如果能看到输出，说明安装成功！

常见问题排查

❌ "需要启用虚拟机平台"

原因：BIOS 未开启虚拟化或 Windows 功能未启用

解决：

  1. 重启电脑进入 BIOS（开机按 F2/F12/Delete）
  2. 找到 "Virtualization Technology" 或 "Intel VT-x" / "AMD-V"，设置为 Enabled
  3. 保存退出，进入 Windows 后以管理员运行： 
    dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
  4. 重启电脑后再次运行 wsl --install
❌ "找不到 wsl 命令"

原因：Windows 版本过旧（需要 Windows 10 2004 以上）

解决：更新 Windows 到最新版本，或手动下载 WSL 2 Linux 内核更新包：
https://aka.ms/wsl2kernel

❌ Ubuntu 启动卡住或闪退

解决方法：

  1. 重置 WSL： 
    wsl --shutdown
wsl --unregister Ubuntu
wsl --install -d Ubuntu
  2. 检查杀毒软件是否拦截 WSL
  3. 确保 C 盘有足够空间（至少 5GB）
⚠️ 网络连接问题

症状：WSL 内无法 ping 通外网或 apt update 失败

解决：在 PowerShell（管理员）运行：

wsl --shutdown
netsh winsock reset
netsh int ip reset
ipconfig /flushdns

然后重启电脑。

性能优化建议

  • 限制内存占用：创建 C:\Users\你的用户名\.wslconfig 文件，写入： 
    [wsl2]
memory=4GB
processors=2
swap=2GB
  • 文件存储位置：尽量把项目文件放在 WSL 文件系统内（~ 目录），而不是 /mnt/c/，可获得 2-5 倍的 I/O 性能提升
  • 终端工具：推荐使用 Windows Terminal，支持多标签、主题、GPU 加速

Docker Desktop 与 WSL 的关系

Docker Desktop 启动后会做什么：

  1. 启动 WSL 2 虚拟机（如果还没启动的话）
  2. 在 WSL 2 中运行 Docker daemon（后台服务）
  3. 让你可以通过 GUI 或命令行与 Docker 交互
  4. 所有容器都运行在 WSL 2 的 Linux 内核中

文件系统访问：Docker Desktop 可以访问你的 Windows 文件系统，路径是 /mnt/c/Users/[用户名]/。你可以在 Windows 中修改文件，Docker 容器也能看到。

注意事项：

  • Docker Desktop 占用 WSL 2 的资源，会影响电脑性能
  • WSL 2 没启动的话，Docker Desktop 无法工作
  • 重启 Docker Desktop 时，所有运行中的容器会停止
  • Docker 数据存储在 WSL 2 的虚拟磁盘中，不会占用 Windows 系统盘

3. Docker Desktop 使用指南

什么是 Docker Desktop？

Docker Desktop 是 Docker 官方提供的图形化管理工具，让你在 Windows/macOS 上轻松使用 Docker。它包括：

  • Docker Engine（核心引擎）
  • Docker CLI（命令行工具）
  • Docker Compose（多容器编排）
  • Kubernetes（容器编排平台，可选）
  • 图形化管理界面

界面功能介绍

Containers 容器

查看所有容器的运行状态、日志、资源占用。可以启动/停止/删除容器，进入容器执行命令。

Images 镜像

管理本地镜像，查看镜像大小、创建时间。可以拉取、删除镜像，从镜像创建容器。

Volumes 数据卷

管理持久化存储，查看哪些容器使用了哪些卷，清理无用的卷释放空间。

Networks 网络

查看和管理 Docker 网络，创建自定义网络让容器之间通信。

重要设置项

Resources（资源限制）：

  • CPUs：分配给 Docker 的 CPU 核心数（建议：总核心数的一半）
  • Memory：内存限制（建议：4-8GB，至少 2GB）
  • Disk image size：虚拟磁盘大小（用于存储镜像和容器数据）

Docker Engine（高级配置）：

{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ],
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "10m",
    "max-file": "3"
  }
}

配置镜像加速器可大幅提升下载速度，日志限制可防止磁盘被占满。

使用注意事项

  • 资源占用：Docker Desktop 会持续占用内存，不用时可以退出释放资源
  • 自动启动：默认开机自启，可在设置中关闭 "Start Docker Desktop when you log in"
  • 磁盘清理：定期运行 docker system prune -a 清理无用镜像和容器
  • WSL 2 集成：确保在 Settings → Resources → WSL Integration 中勾选了 Ubuntu

4. VMware 安装 Ubuntu 环境（WSL 替代方案）

何时使用这个方案？

如果你的电脑无法启用 WSL 2（Windows 家庭版、学校/公司电脑限制等），使用 VMware Workstation Player 安装 Ubuntu 虚拟机是最稳定的替代方案。

安装步骤

1
下载并安装 VMware
  1. 访问 VMware Workstation Player 官网 下载 Windows 版本
  2. 右键以管理员身份运行安装程序
  3. 接受默认配置，完成后重启电脑

免费版足够：VMware Workstation Player 个人非商业使用完全免费。

2
下载 Ubuntu ISO 镜像

访问 Ubuntu 官网，下载 Ubuntu 22.04 LTS Desktop 版本（约 4GB）。

为什么选 Desktop 版？带图形界面，新手友好，方便操作。Server 版是纯命令行。

3
创建虚拟机

启动 VMware Player，选择 "Create a New Virtual Machine"：

  • 选择 "Installer disc image file (iso)"，浏览到下载的 Ubuntu ISO
  • 设置用户名和密码（会自动安装，无需手动操作）
  • 推荐配置：
    • 内存：4GB（最少 2GB）
    • 处理器：2 核
    • 硬盘：30GB 动态分配

注意：动态分配硬盘不会立即占用 30GB，而是按实际使用逐渐增长。

4
等待自动安装

VMware 会自动安装 Ubuntu，大约需要 10-15 分钟。安装完成后会自动重启进入桌面。

5
安装 VMware Tools

在 Ubuntu 桌面打开终端（Ctrl+Alt+T），运行：

sudo apt update
sudo apt install -y open-vm-tools open-vm-tools-desktop

这会启用剪贴板共享、文件拖放、自动调整分辨率等功能。

6
在 Ubuntu 中安装 Docker

运行以下命令（复制整段一起运行）：

# 更新系统并安装依赖
sudo apt update
sudo apt install -y ca-certificates curl gnupg

# 添加 Docker 官方 GPG 密钥
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg

# 添加 Docker 软件源
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu \
  $(lsb_release -cs) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 安装 Docker
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin

# 启动 Docker 服务
sudo systemctl enable --now docker

# 添加当前用户到 docker 组（免 sudo）
sudo usermod -aG docker $USER

# 应用组权限（或者重启虚拟机）
newgrp docker
7
验证安装
docker --version
docker run hello-world

如果看到 "Hello from Docker!"，说明安装成功！

使用技巧

  • 共享文件夹：在 VMware 中配置 Shared Folders，可以让 Windows 和 Ubuntu 互相访问文件
  • 快照功能：在关键节点创建快照，出问题可一键恢复
  • 网络模式：默认 NAT 模式可直接上网，桥接模式可获得独立 IP
  • 性能优化：关闭 Ubuntu 动画效果，禁用不需要的后台服务

注意事项

  • 资源占用：虚拟机运行时会占用设定的全部内存，不用时记得挂起或关闭
  • 磁盘空间：虚拟机文件通常存储在 C:\Users\用户名\Documents\Virtual Machines，注意预留空间
  • 镜像下载：国内下载 Docker 镜像较慢，建议配置镜像加速器（见下一节）

5. Linux 生存指南

必会命令 Top 10

命令 说明 示例
ls 列出文件 ls -la
cd 切换目录 cd /home
pwd 显示当前路径 pwd
mkdir 创建目录 mkdir test
rm 删除文件 rm file.txt
cp 复制文件 cp a.txt b.txt
mv 移动/重命名 mv old.txt new.txt
cat 查看文件内容 cat file.txt
sudo 以管理员权限执行 sudo apt update
clear 清屏 clear

核心概念：Linux 没有 C 盘

Windows 是分盘符的（C:\, D:\），而 Linux 是一棵树，所有文件都从根目录 / 开始。

/ (根目录) → home (用户家目录) → user (你的地盘 ~)

重要路径：

  • ~ - 你的主目录（如 /home/username）
  • /mnt/c - Windows 的 C 盘（WSL 中）
  • .. - 上级目录
  • . - 当前目录

只有高手知道的秘密：Tab 键

在终端输入文件名或命令时，输入前几个字母，然后按 Tab 键，系统会自动补全剩下的内容。如果没反应，就按两下 Tab，会列出所有可能的选项。

永远不要手动把长文件名打完！

实操：用 Nano 编辑文件

在 Linux 里没有鼠标点点点的记事本，我们用 nano 编辑器（新手推荐）。

  1. 输入 nano hello.txt 并回车
  2. 像在记事本里一样输入一段话，比如 "Hello Linux"
  3. 保存：按 Ctrl + O，然后按 Enter 确认文件名
  4. 退出：按 Ctrl + X

高级编辑器：熟练后可以学习 vim，功能更强大但学习曲线陡峭。

现在就来练习吧！

打开你的 WSL Ubuntu 终端，一个命令一个命令地输入。不要复制粘贴！手工打一遍，才能真正记住。

1
练习 1：文件夹导航
pwd                           # 看看你现在在哪儿
cd /mnt/c                     # 跳到 C 盘
pwd                           # 再看一遍，应该是 /mnt/c
ls                            # 列出 C 盘的文件
cd ~                          # 回家（主目录）
pwd                           # 应该看到 /home/你的用户名

预期结果：每条命令都会有输出，你应该看到不同的路径。

2
练习 2：创建和查看文件
mkdir test_folder             # 创建文件夹
cd test_folder                # 进入文件夹
touch hello.txt               # 创建空文件
echo "Hello Linux" > hello.txt  # 写入内容
cat hello.txt                 # 查看内容
ls -la                        # 看看文件详情

预期结果：最后应该看到 "Hello Linux" 被输出到终端。

3
练习 3：复制、移动和删除
cp hello.txt hello_backup.txt  # 复制文件
ls                             # 现在应该有两个文件
mv hello_backup.txt backup.txt # 重命名
rm hello.txt                   # 删除原文件
ls                             # 现在只剩 backup.txt

警告：Linux 的 rm 命令没有回收站，删除就是永久删除！使用 rm -rf 时要特别小心。

4
练习 4：清理环境
cd ..                          # 回到上级目录
rm -r test_folder              # 删除文件夹及其内容
ls                             # 确认已删除

快捷键技巧

  • Tab - 自动补全命令/文件名（最重要！）
  • Ctrl + C - 终止当前程序
  • Ctrl + L - 清屏（等同于 clear）
  • Ctrl + D - 退出当前 shell
  • Ctrl + R - 搜索历史命令
  • 上/下箭头 - 浏览历史命令
  • Ctrl + A - 光标移到行首
  • Ctrl + E - 光标移到行尾

常见问题

❌ "权限不够"（Permission denied）

解决：在命令前加 sudo（以管理员权限运行）。例如：sudo apt update

❌ "命令未找到"（Command not found）

原因：命令拼写错误，或者软件未安装。
解决：仔细检查拼写，或用 sudo apt install 软件名 安装。

⚠️ 如何停止一个卡住的程序？

解决：按 Ctrl + C 强制终止。如果还不行，按 Ctrl + Z 暂停程序，然后输入 kill %1 杀死进程。

7. Docker 核心概念通俗解

镜像 (Image)

类比：游戏安装包

  • 只读模板，包含运行环境
  • 可以从 Docker Hub 下载
  • 一个镜像可以创建多个容器

容器 (Container)

类比：正在运行的游戏

  • 镜像的运行实例
  • 可以启动、停止、删除
  • 相互隔离，互不影响

仓库 (Registry)

类比：应用商店

  • 存放镜像的地方
  • Docker Hub 是官方仓库
  • 可以推送自己的镜像

关系图

仓库 (Docker Hub)
    ↓ pull 拉取
镜像 (nginx:latest)
    ↓ run 运行
容器 (正在运行的 nginx)

8. 初体验：运行第一个容器

5

6
停止并删除容器
