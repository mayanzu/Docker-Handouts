0. 计算机基础知识速览

学习 Docker 前，这些概念你必须搞懂。别怕，我们用最通俗的方式讲。

什么是操作系统 (Operating System)?

简单理解：操作系统是电脑的"管家"。它管理所有的硬件资源（CPU、内存、硬盘等），让软件能够顺利运行。你现在用的 Windows 就是一个操作系统。

类比：如果你的电脑是一家餐厅，操作系统就是经理。经理负责分配厨师（CPU）、食材库存（内存）、冷冻库（硬盘）。如果没有经理，一切都乱套了。

什么是 Linux?

简单理解：Linux 是一个免费开源的操作系统，就像 Windows 一样，但它不是微软做的，而是由全世界的开发者共同维护的。

Windows
微软制造，要付费，普通用户用
macOS
苹果制造，只能装在苹果电脑
Linux
免费开源，全球 99% 的服务器都用

为什么要学 Linux？全世界的云服务器、网站、竞赛平台几乎都跑在 Linux 上。所以作为程序员，必须懂 Linux。将来工作也离不开它。

终端 (Terminal) 和 Shell 是什么?

终端 (Terminal)：就是一个黑白的窗口，你可以在里面输入命令。它只是一个"输入输出工具"，本身不执行命令。

Shell：是真正理解和执行你的命令的程序。常见的有 bash, zsh, sh 等。

类比：终端是餐厅的传菜口，Shell 是理解你点菜单语言的服务员。你说英文，服务员理解英文；你说中文，服务员理解中文。不同的 Shell 就像不同国籍的服务员。

包管理器 (Package Manager) 是什么?

简单理解：在 Windows 里，你下载软件通常是到官网下载 .exe 文件。在 Linux 里，有个"中央超市"叫包管理器，里面什么软件都有，你一条命令就能装，还自动管理依赖。

Ubuntu/Debian： sudo apt install python3

一条命令，Python3 装好

CentOS/RedHat： sudo yum install python3

类似的功能，不同的系统

对竞赛的意义：Docker 里经常要装各种软件。学会用包管理器，可以快速搭建复杂的环境。

文件权限是什么?

简单理解：每个文件都有"身份证"，规定了谁可以读它、写它、执行它。

-rw-r--r-- 1 user group 1024 Nov 19 hello.txt
r (读)
可以看文件内容
w (写)
可以修改文件
x (执行)
可以运行这个文件

为什么要学？在竞赛时，经常需要给文件加执行权限，否则程序跑不了。

1. 为什么是 Docker？

理解容器化技术在学科竞赛中的真正价值。

先说一个困扰许多人的现实问题

假如一道竞赛题的要求是：Python 3.8、Flask 2.0.1、PostgreSQL 12、Redis 6.2、特定的编译环境...

😭 悲剧1：版本冲突
你的电脑上已经有 Python 3.9（装过其他项目）、Flask 1.8、PostgreSQL 13。库版本冲突，代码跑不起来！但你不能卸载，因为其他项目依赖它。

😭😭 悲剧2：时间浪费
你花了 2 小时配置环境，下载、安装、配置依赖。最后发现系统不兼容。临近比赛时间还没开始写代码。

😭😭😭 悲剧3：冤枉啊
你的代码在你电脑跑得好好的，本地测试没问题。提交到裁判机上直接 Runtime Error。代码没错，是环境差异导致的问题。

Docker 到底是什么？（超通俗版）

Docker 就是一个"虚拟的完整电脑"。你可以在里面装任意版本的软件，这个"虚拟电脑"是完全隔离的，不会干扰你真实电脑上的任何东西。

更重要的是：你可以把这个"虚拟电脑"打包成一个文件（Docker 镜像），发给任何人、任何电脑、任何操作系统。别人打开这个文件，看到的"电脑"跟你的完全一样。没有版本差异，没有兼容性问题。

这就是 Docker 的核心价值："Write Once, Run Anywhere"（写一次，到处运行）。这也是为什么全世界的企业和竞赛都用 Docker。

没有 Docker 的痛点

  • 版本地狱："在我电脑上能跑，在裁判机上跑不起来。"
  • 时间浪费：配置环境花了 3 小时，写代码只剩半小时。
  • 系统污染：删错一个库，整个系统崩溃要重装。
  • 跨平台问题：别人用 Mac，你用 Windows，代码跑不了。
  • 隐藏错误：系统环境的差异导致 BUG，却很难查出原因。

Docker 的超能力

  • “在我电脑上能跑，在裁判机上跑不起来。”
  • 配置环境花了3小时，写代码只剩半小时。
  • 删错一个库，整个系统重装。

Docker 的超能力

  • 环境一致： 100% 保证。Mac、Linux、Windows 完全一样。
  • 极速启动： 比虚拟机快 100 倍。（1秒 vs 20秒）
  • 版本锁定： Dockerfile 精确记录所有依赖版本。
  • 完全隔离： 不影响主系统，删除时干干净净。
  • 易于分享： 一个文件就能共享整个开发环境。

通常会问的三个问题

Q1: Docker 是虚拟机吗？

不完全是。虚拟机模拟整个电脑（包括 BIOS、驱动程序、完整 OS），启动需要 20 秒，占用 2-5GB 内存。Docker 是轻量级容器技术，只封装你的应用程序和它的依赖，启动只需 1 秒，占用 100MB 内存。想象虚拟机是"复制一个工厂"，Docker 是"装集装箱"。

Q2: Docker 占用空间大吗？

非常小。一个完整的 Python + Flask + PostgreSQL 的 Docker 镜像，通常只有 300-500MB。而如果直接装在系统上，往往要 1-2GB，还会污染系统、产生依赖冲突。

Q3: 学 Docker 难吗？

初级使用非常简单。就十来个命令，一周就能掌握 Docker 的 90% 功能。这节课我们就从零开始，一步步教你。

最核心的一个比喻

❌ 没有 Docker 的做法：
你把一份"红油火锅配方"给 10 个厨师，让他们自己去做菜参加美食大赛。每个厨师在自己的厨房做菜，厨房的工具不同、调料不同、火候也不同。结果？做出来的菜肯定不一样。
✅ 有了 Docker 的做法：
你不仅给了配方，还给了一个"魔法厨房"（Docker 容器）。这个厨房里的所有工具、调料、火候都完全锁定，不能改。无论谁、在哪里、什么时候打开这个厨房，做出来的菜100% 相同。公平！🎯

数据对比一览表

对比项 虚拟机 Docker 容器
启动时间 20-30 秒 1-2 秒 ⚡
占用内存 2-5 GB 100-500 MB ⚡
镜像文件大小 3-10 GB 100-1000 MB ⚡
部署复杂度 复杂，需配置 简单，一个文件 ⚡
环境一致性 较难保证 100% 保证 ⚡

2. 安装 WSL 2 (Windows Subsystem for Linux)

在 Windows 上获得原生的 Linux 体验，是运行 Docker 的最佳伴侣。

🧐 什么是 WSL？

简单来说，就是在 Windows 里开了一个 Linux 的窗口。你不需要装双系统，也不需要装笨重的虚拟机，就能像使用普通的 Windows 软件一样使用 Linux 命令。

什么是虚拟机？

虚拟机（Virtual Machine）通过一层虚拟化技术，在真实电脑里模拟出一台完整的“第二台电脑”，包含独立的 CPU、内存、硬盘、网卡等。常见例子有 VMware、VirtualBox。

缺点：启动慢、占用资源多、和主系统隔离严重，需要手动设置共享。

Hyper-V 是什么？

Hyper-V 是微软官方的一级虚拟化引擎（Hypervisor）。它负责管理虚拟机如何共享 CPU 和内存。Windows 10/11 专业版自带 Hyper-V，WSL 2 就是基于它来运行的。

提示：如果电脑 BIOS 没开启硬件虚拟化 (Intel VT-x/AMD-V)，Hyper-V 和 WSL 都无法启动。

WSL 2 如何运行 Ubuntu？

WSL 2 会在 Hyper-V 之上启动一个轻量级的 Linux 内核，实质上是一个高度优化的虚拟机。但它与 Windows 深度整合：文件系统互通（/mnt/c）、网络共享、图形应用支持。

效果：近乎原生的 Linux 性能，命令行体验和真机一样，却能和 Windows 工具无缝协作。

为什么是 Ubuntu？

  • 社区庞大、资料全，遇到问题容易查到答案。
  • 包管理器 apt 简单易用，对新手友好。
  • LTS 版本稳定可靠，云平台、竞赛环境大量采用。
  • 预装常用工具（Python、Git、build-essential），无需二次配置。

步骤 1 开启功能并安装

请以 管理员身份 打开 PowerShell。

wsl --install
注意： 如果提示无法识别命令，请确保 Windows 更新至最新版本。安装完成后，必须重启电脑。

步骤 2 初始化 Ubuntu

  1. 重启后，Ubuntu 终端窗口会自动弹出（如果没有，在开始菜单搜索 "Ubuntu" 打开）。
  2. 等待几分钟进行初始化安装。
  3. 根据提示输入 Username (推荐全小写，如: user)。
  4. 输入 Password (注意：输入密码时屏幕不会显示任何字符，连星星*都没有，这是正常的 Linux 安全机制，输完直接回车即可)。

小白必看：WSL 使用小技巧

1. 怎么复制粘贴？

在 Ubuntu 终端里，Ctrl+C 并不是复制，而是“终止程序”。

  • 复制： 选中文字 -> 点击鼠标右键
  • 粘贴： 点击鼠标右键

2. Windows 的文件在哪里？

WSL 可以直接访问你的 C 盘、D 盘。

C盘路径: /mnt/c/
D盘路径: /mnt/d/
桌面路径: /mnt/c/Users/你的用户名/Desktop/

电脑不支持 WSL 2？可以这样解决

常见原因包括：Windows 版本过低（1903 以下）、CPU 不支持虚拟化、BIOS 虚拟化被关闭、企业电脑禁用了 Hyper-V。以下是可行的替代方案：

方案 1：升级系统或开启虚拟化
  • 将 Windows 更新至 21H2 及以上，或直接升级到 Windows 11。
  • 进入 BIOS，确认 Intel VT-x / AMD-V 已启用。
  • 在“启用或关闭 Windows 功能”中勾选 Virtual Machine Platform 与 Hyper-V。
方案 2：使用传统虚拟机跑 Ubuntu（推荐 VMware）

步骤：

  1. 访问 VMware Workstation Player 页面，下载适用于 Windows 的安装程序（免费个人用途）。
  2. 右键以管理员身份运行安装程序，保留默认选项即可，安装完成后重启电脑。
  3. 打开 Player，点击 Create a New Virtual Machine -> 选择已下载的 Ubuntu ISO。
  4. 分配虚拟机资源：推荐 4GB 内存、2 核 CPU、30GB 虚拟磁盘，启用“Unity 统一剪贴板”和“拖放文件”共享功能。
  5. 完成后启动虚拟机，按照屏幕提示安装 Ubuntu：选择语言、键盘、网络、用户名/密码，安装时勾选“安装第三方软件”，最后重启。
  6. 在 Ubuntu 内执行 sudo apt update && sudo apt install docker.io，再运行 sudo systemctl enable --now docker。

提示：VMware Player 的虚拟网卡默认桥接到宿主机，Ubuntu 可直接访问互联网；Docker 在虚拟机内运行，适合网络/硬件受限的设备。

方案 3：远程开发环境

申请学校/企业的云服务器，或使用阿里云、华为云、腾讯云等租一台按量计费的云主机，直接远程连接（SSH）进行开发与 Docker 实验。

如果担心费用，可以利用 GitHub Codespaces、容器化在线实验平台等免费额度。

方案 4：Docker Desktop + Hyper-V 模式

部分旧机器虽无法升级 WSL 2，但仍能启用 Hyper-V。安装 Docker Desktop 时选择 Hyper-V 引擎即可运行 Linux 容器。

注意：Hyper-V 与 VMware/VirtualBox 不能同时使用，切换时需启用/禁用功能或使用 bcdedit 关闭。

课堂务必让同学们提前测试：在 PowerShell 输入 wsl --status，根据输出判断是否需要采取上述方案。

3. Docker Desktop 使用指南

详细了解 Docker Desktop 的功能和与 WSL 的配合方式。

什么是 Docker Desktop?

Docker Desktop 是 Docker 官方提供的桌面应用程序。它提供了：

  • 可视化界面：用图形界面管理镜像和容器
  • 命令行工具：在终端使用 docker 命令
  • WSL 2 集成：在 Windows 上运行原生 Linux 容器
  • 资源管理：控制 Docker 使用的 CPU、内存、磁盘空间

🔗 Docker Desktop 与 WSL 2 的关系

Docker Desktop 依赖 WSL 2 作为底层引擎来运行容器。

1

WSL 2 是基础

WSL 2 提供了轻量级 Linux 环境。Docker 容器就运行在 WSL 2 的 Linux 系统中。

2

Docker Desktop 是管理器

Docker Desktop 负责启动 WSL 2，并管理所有的容器和镜像。

3

两者必须配合

如果 WSL 2 没装或有问题，Docker Desktop 无法正常运行。

功能介绍 Docker Desktop 的主要界面

Containers 标签

显示所有容器（运行中和已停止的）。

你可以： 启动、停止、删除、查看日志、进入容器

Images 标签

显示本地的所有镜像。

你可以： 运行镜像、删除镜像、查看镜像信息

Volumes 标签

管理数据卷（容器与主机共享的存储）。

用途： 保存容器数据，容器删除后数据仍保留

Settings 标签

配置 Docker Desktop 的各种设置。

常用： 内存、CPU、镜像源、启动设置

命令行 从终端使用 docker 命令

安装 Docker Desktop 后，就可以在 WSL 的终端中使用 docker 命令了。

方式 1：在 WSL Ubuntu 终端中使用（推荐）

$ docker ps
$ docker images
$ docker run -it ubuntu bash

在 WSL 中运行的命令会自动连接到 Docker Desktop。

方式 2：在 Windows PowerShell 中使用

> docker ps
> docker images
> docker run -it ubuntu bash

也可以在 PowerShell 中直接使用 docker 命令。

常见 Docker Desktop 操作

1. 启动和停止容器

在 GUI 中：

  1. 点击 "Containers" 标签
  2. 找到要操作的容器
  3. 点击播放按钮启动或停止按钮停止

在命令行中：

docker start [容器名] docker stop [容器名]

2. 查看容器日志

在 GUI 中：

  1. 在 Containers 中找到容器
  2. 点击容器名称打开详情
  3. 选择 "Logs" 标签查看输出

在命令行中：

docker logs [容器名] docker logs -f [容器名]

3. 删除容器和镜像

警告：删除操作不可撤销！

建议：先备份重要数据

docker rm [容器名] # 删除已停止的容器 docker rmi [镜像名] # 删除镜像 docker system prune # 清理未使用的资源

4. 进入运行中的容器

想在运行中的容器里执行命令？

docker exec -it [容器名] bash

这会打开一个 bash 终端，让你在容器内部操作。

配置 Docker Desktop 资源

如果电脑运行缓慢，可能是 Docker 占用了太多资源。可以在 Settings 中调整：

CPU 数量

Docker 最多可以使用的 CPU 核心数。建议设为总核心数的一半。

内存

Docker 最多可以使用的内存大小。8GB 电脑建议设为 4GB。

磁盘镜像

Docker 存储镜像和容器的大小限制。不够用会影响性能。

提示： 设置后需要重启 Docker Desktop 才能生效。点击托盘图标 -> Quit Docker Desktop -> 重新打开。

🎮 Docker Desktop GUI 实操指南

如果你更喜欢点点点而不是敲命令，Docker Desktop 的 GUI 可以帮你。这些操作会帮你更深刻理解容器的概念。

操作 1在 GUI 中查看镜像

  1. 第 1 步： 打开 Docker Desktop
  2. 第 2 步： 点击左侧菜单的 Images 标签
  3. 第 3 步： 看到 hello-world 的镜像了吗？它显示镜像大小、创建时间等信息
  4. 第 4 步： 点击镜像右边的三个点 ⋮ 按钮，可以删除或查看详情

💡 这说明了什么： GUI 只是把命令 docker images 的输出用漂亮的卡片展示出来。

操作 2在 GUI 中查看容器

  1. 第 1 步： 点击左侧菜单的 Containers 标签
  2. 第 2 步： 如果列表是空的，说明没有容器在运行。但不用急，我们接下来会创建一个
  3. 第 3 步： 返回 Images 标签，找到 hello-world
  4. 第 4 步： 点击镜像的运行按钮（播放图标），会创建并运行一个容器

💡 观察： 容器会立刻完成任务并停止。回到 Containers 标签，看到它的状态是 "Exited" 吗？

操作 3查看容器日志

  1. 第 1 步： 在 Containers 标签中，找到刚才运行的容器
  2. 第 2 步： 点击容器名称，打开详情窗口
  3. 第 3 步： 点击 Logs 标签，看到容器的输出信息
  4. 第 4 步： 这就是 docker logs [容器名] 的图形化版本！

💡 比较： 这和你在命令行看到的输出是一样的，只是显示方式不同。

操作 4清理资源

  1. 第 1 步： 在 Containers 标签中选择一个已停止的容器
  2. 第 2 步： 点击删除按钮（垃圾桶图标），会弹出确认对话框
  3. 第 3 步： 确认删除。容器就从列表中消失了
  4. 第 4 步： 同样可以在 Images 标签中删除不需要的镜像

⚠️ 注意： 删除后不可恢复。如果镜像还要用，就别删。

命令行 vs GUI：怎么选？

命令行（推荐）
  • 更快、更灵活
  • 可以写脚本自动化
  • 竞赛环境就用这个
GUI（辅助）
  • 直观，不容易出错
  • 快速浏览镜像和容器
  • 调试时很方便

理解 Docker Desktop 和 WSL 的关系

Docker Desktop 启动后会做什么

  1. 启动 WSL 2 虚拟机（如果还没启动的话）
  2. 在 WSL 2 中运行 Docker daemon（后台服务）
  3. 让你可以通过 GUI 或命令行与 Docker 交互
  4. 所有容器都运行在 WSL 2 的 Linux 中

文件系统的访问

Docker Desktop 可以访问你的 Windows 文件系统：

/mnt/c/Users/[用户名]/ # C 盘

你可以在 Windows 中修改文件，Docker 容器也能看到。

注意事项

  • Docker Desktop 占用 WSL 2 的资源，会影响电脑性能
  • WSL 2 没启动的话，Docker Desktop 无法工作
  • 重启 Docker Desktop 时，所有运行中的容器会停止
  • Docker 数据存储在 WSL 2 的虚拟磁盘中，不会占用 Windows 系统盘

4. VMware 安装 Ubuntu 环境

可替代 WSL 的传统虚拟机方案

在无法启用 WSL 的设备上，使用 VMware Workstation Player 装一台 Ubuntu 虚拟机，再在里面配置 Docker，是最稳定的备选方案。

步骤 1：下载并安装 VMware

  1. 访问 VMware Workstation Player 官网 下载 Windows 版本。
  2. 右键以管理员身份运行安装程序，接受默认配置，安装完成后重启宿主机。
  3. 启动 VMware Player，选择 Create a New Virtual Machine 并载入已下载的 Ubuntu LTS ISO。

步骤 2：虚拟机资源配置

  • 资源建议：4GB 内存、2 个逻辑核心、30GB 动态磁盘。
  • 启用 Unity 统一剪贴板与拖放共享，提升主机与虚拟机间协作。
  • 安装时勾选“安装第三方软件”以获得额外网络/显卡支持。

步骤 3：Ubuntu 内安装 Docker

在 Ubuntu 终端输入以下命令，添加官方源并安装 Docker、构建工具和 compose 插件：

sudo apt update && sudo apt install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo \
  "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \ 
  https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" \ 
  | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo systemctl enable --now docker
sudo usermod -aG docker $USER

运行 newgrp docker 或重启虚拟机以应用权限后，再执行 docker version 与 docker run hello-world 验证安装。

小贴士：虚拟机网络默认桥接，可直接上网；镜像下载慢时可通过宿主机代理或提前导入镜像 tar 包。

5. Linux 生存指南

忘掉鼠标，开始习惯键盘。掌握这 6 个最常用的竞赛命令。

🌍 核心概念：Linux 没有 C 盘

Windows 是分盘符的（C:\, D:\），而 Linux 是一棵树，所有的文件都从根目录 / 开始。

/ (根目录)
home (用户家目录)
user (你的地盘 ~)

只有高手知道的秘密：Tab 键

在终端输入文件名或命令时，输前几个字母，然后按 Tab 键，系统会自动补全剩下的内容。如果没反应，就按两下 Tab，会列出所有可能的选项。永远不要手动把长文件名打完！

ls 查看

List. 看看当前目录下有哪些文件。

ls -la # 显示详细信息(包括隐藏文件)
pwd 定位

Print Working Directory. 我现在在哪里？

pwd
cd 移动

Change Directory. 切换文件夹。

cd /mnt/c # 去C盘
cd .. # 返回上一级
mkdir 新建

Make Directory. 创建一个新的文件夹。

mkdir my_project
touch 新建

创建一个空白文件。

touch main.py
cat 阅读

Concatenate. 直接在终端显示文件内容。

cat main.py

实操：用 Nano 编辑文件

新手推荐

在 Linux 里没有鼠标点点点的记事本。我们用 nano 编辑器。

  1. 输入 nano hello.txt 并回车。
  2. 像在记事本里一样输入一段话，比如 "Hello Linux"。
  3. 保存： 按 Ctrl + O，然后按 Enter 确认文件名。
  4. 退出： 按 Ctrl + X。

6. 安装 Docker Desktop

可视化管理容器，并提供 WSL 2 核心引擎支持。

下载安装包

前往 Docker 官网下载 Docker Desktop for Windows。
安装时务必勾选 "Use WSL 2 instead of Hyper-V"。

关键配置：配置国内镜像源

如果不配置这一步，拉取镜像可能会非常慢甚至失败。

1. 打开 Docker Desktop 设置 (齿轮图标) -> Docker Engine。

2. 将下方的配置 JSON 完整覆盖编辑框内容：

{
  "builder": {
    "gc": {
      "defaultKeepStorage": "20GB",
      "enabled": true
    }
  },
  "experimental": false,
  "registry-mirrors": [
    "https://docker.1ms.run",
    "https://docker.xuanyuan.me"
  ]
}

3. 点击右下角的 Apply & restart。

7. Docker 核心概念通俗解

不要背定义，用三个比喻听懂 Docker 到底在干什么。

镜像 (Image)

相当于：游戏安装包 / 类(Class)

它是静止的、只读的。就像你的《英雄联盟》安装包，它躺在硬盘里，不会自己动，也不会自己产生数据。你不能修改镜像本身，只能利用它去“运行”出一个程序。

容器 (Container)

相当于：运行中的游戏窗口 / 对象(Object)

它是动态的、有生命的。当你双击图标，游戏启动了，这就变成了一个容器。你可以同时双开两个账号，就是用同一个“镜像”启动了两个“容器”。容器关了，数据可能就没了。

仓库 (Repository)

相当于：应用商店 / App Store

我们去哪里下载镜像？通常是 Docker Hub。它是全球最大的镜像集市，里面有官方认证的 nginx, python, mysql 等镜像，大家都可以直接拉取使用。

Docker Hub
pull (拉取)
本地镜像 (Images)
run (运行)
运行容器 (Containers)
服务上线

8. 初体验与命令详解

不要只会复制粘贴，看懂每一行输出。

WSL: Ubuntu
$ docker run hello-world

Unable to find image 'hello-world:latest' locally

// 解释：Docker 发现你本地没有这个镜像，准备去下载。

latest: Pulling from library/hello-world

719385e32844: Pull complete

// 解释：正在从 Docker Hub 拉取层 (Layer)。

Hello from Docker!

This message shows that your installation appears to be working correctly.

// 解释：容器启动成功，打印了这句话，然后立刻退出了。

常用命令速查表

docker pull [镜像名]

去超市买菜（下载镜像）。
例如：docker pull nginx

docker run [参数] [镜像名]

开始做菜（启动容器）。
例如：docker run -d nginx

docker images

看看冰箱里有哪些菜（查看本地镜像列表）。

docker ps

看看炉子上正在煮什么（查看正在运行的容器）。
加 -a 可以看包括已经关掉的容器。

🚀 Docker 初体验：跟着做

确保 Docker Desktop 已启动（任务栏看得到 Docker 图标）。然后在 WSL 终端中执行这些命令：

第 1 步检查 Docker 是否正常

docker version

💡 期望输出： 显示 Docker 客户端版本和服务器版本。如果看到版本号说明连接成功。

第 2 步运行第一个容器

docker run hello-world

💡 期望输出： 看到 "Hello from Docker!" 的欢迎信息。第一次运行时会自动下载镜像，可能需要 30 秒到 1 分钟。

⚠️ 常见问题： 如果显示 "Cannot connect to Docker daemon"，说明 Docker Desktop 没启动。请点击任务栏的 Docker 图标启动它。

第 3 步查看你的镜像

docker images

💡 期望输出： 你应该看到一个表格，至少包含一行 hello-world 的信息。这证明镜像已经下载到你的电脑里了。

第 4 步查看运行过的容器

docker ps -a              # 列出所有容器（包括已停止的）
docker ps                 # 只列出正在运行的容器

💡 期望输出： 应该看到你刚才运行的 hello-world 容器（状态是 "Exited"，因为它已经完成了任务并自动停止）。

第 5 步进入交互式容器

docker run -it ubuntu bash

💡 期望输出： 会看到一个新的命令提示符，显示 root@[某个ID]:/# 。你现在在 Ubuntu 容器内部了！

✨ 在容器内尝试这些命令：

pwd                  # 看你在哪儿
ls                   # 列出容器内的文件
cat /etc/os-release  # 查看操作系统信息（证明这真的是 Ubuntu）
echo "Hello Docker"  # 打印一条消息
exit                 # 退出容器

⚠️ 重要： 当你 exit 后，容器会停止。这很正常，容器的使命就是运行一个程序，程序结束后容器就关闭了。

✅ 完成这些练习后，你应该会：

  • 知道如何验证 Docker 的安装
  • 会下载和运行一个镜像
  • 能够列出本地的镜像和容器
  • 可以进入容器内部执行命令
  • 理解容器的生命周期（启动 → 运行 → 停止）
