热身回顾：Docker基础知识 & 互动问答

镜像、容器、仓库三大核心

  • 镜像（Image）： 类似“软件安装包”，包含运行环境和程序
  • 容器（Container）： 镜像运行后的实例，像“正在运行的软件”
  • 仓库（Registry）： 存放镜像的云端仓库（如Docker Hub）

  • 启动容器用哪个命令？
  • 如何查看本地所有镜像？
  • 镜像和容器的区别？
引出： 手动做镜像步骤繁琐，Dockerfile 能让一切自动化！

镜像脚本的魔法：Dockerfile原理

手动操作 vs 脚本自动化

  • 手动做镜像：每次都要敲命令、复制文件、装依赖，容易出错
  • Dockerfile：像写剧本一样，一次写好，自动构建，结果可复现

举例：手动做Nginx镜像要：

  1. 拉取nginx镜像
  2. 进入容器，手动复制网页
  3. commit生成新镜像
  4. 每次都要重复

用Dockerfile只需写几行，自动完成！

实战：写出你的第一个Dockerfile

最基础的Nginx Dockerfile

FROM nginx:alpine
RUN echo "hello dockerfile" > /usr/share/nginx/html/index.html
CMD ["nginx", "-g", "daemon off;"]
指令详解：
  • FROM：选择基础镜像
  • RUN：执行命令（如安装/写文件）
  • CMD：容器启动时默认执行的命令
动手练习： 跟着老师敲一遍，保存为 Dockerfile，用 docker build -t my-nginx . 构建，再 docker run -p 8080:80 my-nginx 访问测试！

build命令进阶与排错演练

镜像命名与标签

用 -t 指定镜像名和标签：

docker build -t my-nginx:v1 .

用 -f 指定Dockerfile路径：

docker build -f ./Dockerfile.dev -t dev-nginx .
排错演练： 故意写错Dockerfile（如 FRM），观察报错，学会看错误信息定位问题。

Dockerfile常用指令实战

COPY/ADD

把本地文件复制到容器。ADD 能自动解压压缩包。

COPY index.html /usr/share/nginx/html/

WORKDIR

指定后续命令的工作目录，避免写长路径。

WORKDIR /app
RUN touch test.txt

EXPOSE

声明容器要开放的端口（如 EXPOSE 8080），run时用 -p 做端口映射。

动手练习： 每讲完一个指令，立刻加进自己的Dockerfile，重新build并测试！

镜像优化与体积对比

合并RUN指令，减少层数

对比多条RUN和用&&合并的镜像体积差异：

RUN apt update
RUN apt install -y curl
# VS
RUN apt update && apt install -y curl

用alpine镜像减小体积

把 FROM nginx 改成 FROM nginx:alpine，重新build，看镜像size变化。

动手练习： build两个版本，docker images 对比体积！

常见坑点与排查练习

常见错误：
  • CMD写多个（只会执行最后一个）
  • RUN漏写&&导致分层多
  • COPY路径写错，文件找不到
练习： 给出几个错误Dockerfile，让同学们找错、改对并验证。
